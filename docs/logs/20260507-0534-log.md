## Technical surprises that shaped the work

1. **`g.json` snapshot drift**: my initial recon read 17433 nodes/108MB; planner
   subagent's read showed 3730 nodes/27MB. File had been regenerated between the
   sessions. Plan numbers updated mid-stream. Real archiver subgraph is 3730 nodes,
   not 17433.
2. **Worktree-isolation**: failed on first attempt ("not in a git repository") because
   the runtime cached the absence of `.git/` from session start, before bootstrap
   subagent ran `git init`. Fell back to serial execution per CLAUDE.md, then
   succeeded after the runtime caught up. Now used everywhere parallel editors
   are needed.
3. **PR layout shift**: user moved heavy upstream to `yatool_orig/` mid-session
   so worktrees on the new project would be cheap. All paths in plan/ledger/
   subagent briefs rewritten to use `yatool_orig/` for reference data and
   `yatool/` for our code.
4. **`throw.go` and `STYLE.md` dropped mid-stream** by user as project standard.
   Required PR-11 retrofit BEFORE continuing the comparator/gen chain. Total
   ~600 LOC changed across PR-01/02/07 source + tests in PR-11.
5. **Reference graph carries 4 build/cow/on nodes (not 2)** — 2 platforms × {CC, AR}.
   PR-10's test had to filter by `Platform == TargetCfg.Name` to keep the
   2-vs-2 comparison meaningful. Material concern for M2's host-pass design.
6. **Reference inputs are NOT all alphabetical** — 7 of 3730 nodes have
   non-alphabetical inputs (mostly multi-cmd LD nodes mixing `$(BUILD_ROOT)`/
   `$(SOURCE_ROOT)`/`$(VCS)` paths with `vcs_info.py` mid-list). Future emitter
   per-rule decision, not global sort.
