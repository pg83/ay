## L3 — 210 misses, two distinct populations

  CFLAGS emitter the BUILD_ROOT runtime_py3 include is emitted

  - 4× `-Wno-unused-variable` — bundled `-W*` flag set lost.
  - 2× `--plugin … ar.pyplugin` — openssl AR plugin invocation.
  - PY3 PR cmds (`mod=...`, `*.pyc`) on 4 nodes — `.pyc` action

## foreign_deps — 735 mismatches

These are tool-uid references (`{'tool': [<uid>...]}`). They cannot be
fixed without stable upstream action hashes. Every time an upstream
producer (e.g. archiver, py3cc) changes inputs or cmd_args, its uid
shifts and every consumer's `foreign_deps` shifts in lockstep.
This is the lever between L0=95% and L0=100%: stabilise the leaves,
and the foreign_deps wave settles by itself.

# Top 5 PRs ranked by cascade yield

## PR-C-1 — Proto + abseil header closure (PRIMARY YIELD)

Add the transitive header closure that REF includes for
`.pb.cc.o` modules and their AR archives. The fix lives in the
PEERDIR-driven include emitter for `library/cpp/{eventlog,protobuf,retry}/proto`
families and the `EmitsIncludes`/`AddIncl` registration for the
protobuf runtime+abseil-tstring chains.

- **L2 cells closed**: 14 directly (proto cluster B) + ~10 indirect
  AR/LD followers + the 4 ANTLR CC nodes share the same root → **~28 nodes**.
- **L3 cells closed**: 11+9+9 of the top REF-only `-I` args resolve →
  **~30 of 54 content-diff nodes**.
- **deps cells closed**: 11 (L2+L3+deps cluster).
- **Risk**: Medium. Touches PEERDIR propagation rules. Must not
  re-emit headers on non-pb-cc CC nodes (over-emit guard).
- **Estimated lift**: L1 stays, **L2 +28**, **L3 +30**, **deps +11**.

## PR-C-2 — CFLAGS include-slot ordering (runtime_py3 vs abseil-cpp)

In the C/C++ CFLAGS emitter, the `-I$(BUILD_ROOT)/library/python/runtime_py3`
addition is placed before the abseil-cpp PEERDIR `-I` block; REF
places it after. Re-slot in the codegen so build-root python3 include
follows the PEERDIR-`-I` chain.

- **L3 cells closed**: 156 order-only nodes (largest single L3 cascade).
- **deps cells closed**: 133 (the order-only nodes that also drift
  deps).
- **Risk**: Low. Pure positional move in the emitter; no input/output
  set changes.
- **Estimated lift**: **L3 +156**, **deps +133**, L2 unchanged.

## PR-C-3 — objcopy_* input deduplication

The codegen for `objcopy_*` action nodes emits each input twice
(every `.py` source and its `.yapyc3` byte-compiled counterpart).
REF emits each exactly once. Add a dedup pass at the end of objcopy
input emission (or, more correctly, stop adding both the `${SRC}`
and `${SRC}.yapyc3` synonyms).

- **L2 cells closed**: 41 (cluster D).
- **deps cells closed**: 56 (their downstream LD/AR uid wave).
- **Risk**: Low. Surgical edit in one emitter; the duplicates are
  exact set-equal so set-equality already holds, only the list/count
  differs.
- **Estimated lift**: **L2 +41**, **deps +56**, downstream LD uid
  hashes stabilise.

- **Estimated lift**: **L2 +10** (10 cells, but the largest by mass).
