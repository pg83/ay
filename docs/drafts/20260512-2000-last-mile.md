## Top-5 PRs by lift

1. **PR-L2-B (py3-runtime helper-closure)** — **9 L2 rows + 2 L3 rows = 11**. Single root cause: PY_REGISTER/RESOURCE/objcopy nodes don't propagate their helper-script inputs (`objcopy.py`, `__res.py`, `sitecustomize.py`, `entry_points.py`, `.pyc.inc`, `objcopy_<sha>.o`) into containing `.a` / `LD` closures. Also fixes the PY_REGISTER path-resolution defect (L3-ε) as a side-effect of touching the same composer. Most concentrated cascade-yielder.
2. **PR-L2-A+D (musl-include closure)** — **10 L2 rows**. aarch64 + x86 host-CC nodes missing musl `bits/{posix,float,setjmp}.h` + `include/{unistd,float,setjmp}.h`. Root cause is the host-musl ADDINCL chain not transitively including arch-bits headers in the scanner closure. Fixing aarch64 closes 8; fixing x86 closes 2; same code path. Also resolves `libdevtools-ymake.a` (cluster G) as a free rollup.
3. **PR-L3-α (member-order discipline)** — **0 L2 + 13 L3 = 13**. The biggest single L3 cluster. AR member listing: hand-written `.cpp.o` must precede generated `.{rl6,ev.pb,reg3,g4,objcopy}.cpp.o` even though strict alphabetic order would interleave. Pure ordering — no new emission, just a comparator change in the AR composer.
4. **PR-L2-C+L3-γ (openssl-asm-norm + ar.pyplugin)** — **2 L2 + 2 L3 = 4**. Coupled defect: ASM member paths use `crypto/../asm/<f>.s` instead of canonical `asm/<f>.s` (path normalization), and the AR command line drops `--plugin $(SOURCE_ROOT)/.../ar.pyplugin`. One composer touches both.
5. **PR-L3-β (antlr -Wno-unused-variable)** — **0 L2 + 4 L3 = 4**. Mechanical: antlr-generated `.g4.cpp.o` compile lines need `-Wno-unused-variable`. Single flag-injection rule for the `_g4_` synthetic target class.

Honorable-mention sixth: **PR-L2-F (H_DUMPER fieldcalc/autoarray)** — 2 L2 rows + 1 rollup (`libdevtools-ymake-symbols.a`), root cause is `dep_types.h_dumper.cpp` PR-node missing its scanner-resolved header inputs.

## Single most cascade-yielding move

**PR-L2-B (py3-runtime helper-closure)** is the highest-cascade single
PR by total rows closed (11), but **PR-L3-α (AR member-order
discipline)** is the highest-cascade *move* in the sense the question
asks: it closes 13 mismatched rows with **zero new emission** — purely
a sort-key change in the AR `cmd_args` composer. No risk of regression
to L0/L1/L2 because the inputs[] sets are already identical; only the
order in the command-line argv list flips. Lowest blast radius,
highest deterministic yield.

The combined recommendation is to land **PR-L3-α first** (free 13 L3
rows, no functional change, smallest diff) — this raises L3 from
99.67 % to ≈99.85 % in one commit. Then **PR-L2-B** (11 rows, fixes a
real defect) to bring L2 to ≈99.81 % and L3 to ≈99.88 %. After those
two PRs the residue is 17 L2 + 14 L3 (= 31 total), all with
single-cluster fixes (musl-include closure being the next biggest at
10 rows).
