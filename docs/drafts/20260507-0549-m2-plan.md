# M2 plan — `tools/archiver` static slice (target-only)

## 1. M2 scope summary

Target: produce the build subgraph reachable from `tools/archiver` on the **target platform only** (`default-linux-aarch64`). Verified closure (BFS from `tools/archiver` LD root via `deps`, skipping `host_platform=true` nodes) = **1,926 nodes / 34 modules / op-types {CC: 1853, AR: 33, AS: 21, JS: 16, R6: 1, CP: 1, LD: 1}, 1 multi-cmd LD (4 cmds), 1 foreign_deps `tool` entry (dangling)**. M2 stubs the host platform: every cross-platform reference (the one R6 → host LD, plus any TOOL-style indirection) emits a placeholder `foreign_deps.tool` entry and does NOT recurse into host nodes. M2 also defers (a) hand-translating the per-module flag bundles for `contrib/libs/cxxsupp/*`, `contrib/restricted/abseil-cpp`, `contrib/libs/tcmalloc/*` (those are ~500 of 1853 CC nodes — full fidelity is M5's job), (b) the streaming emitter (M3), (c) parallel parser (M3 — Q2 unlocks `errgroup` from M2 onward, but M2 ships single-threaded for correctness; parallel-parse is a one-PR bolt-on in M3 once recursion shape is stable).

## 2. Acceptance gate honesty (escalated open question Q3)

**The M1-plan acceptance "L1 ≥ 85%, L3 ≥ 50% on archiver subgraph, binary < 5 s" requires unpacking before M2 starts.**

- **L1 = 85%** on a 1,926-node closure means we must emit ~1,637 paired byte-exact nodes. To clear that, we MUST emit all of `contrib/libs/musl` (1,328 nodes alone, generated from `contrib/libs/musl/ya.make` + `contrib/libs/musl/ya.make.inc` via `INCLUDE`). Skipping musl caps L1 at ~31%. Therefore M2 MUST implement `INCLUDE(...)` (file inclusion) and the `IF/ELSE/ENDIF` evaluator.
- **L3 = 50%** means ~963 byte-exact CC bundles. Each large module uses its OWN distinct flag bundle (verified: 29 distinct bundle fingerprints across 1,853 target CC nodes; the largest single bundle is musl-style with 1,314 nodes; second-largest is cxxsupp/builtins with 161 nodes; library/cpp/* simple-libs share 1 bundle covering 23 nodes). To clear 50% in L3, M2 must hand-translate at minimum the musl bundle (then it's ≈68%).

**Recommendation (default if user doesn't override):** scope M2's L3 acceptance to the **simple-libraries bundle** (existing `noLibcUndebugBlock`-flavoured one, generalized for non-NO_LIBC modules — already deferred as PR-08-D03) PLUS the **musl bundle** (one new flag bundle in flags.go). Together cover ~1,337 of 1,853 CC nodes ≈ 72% L3 — clears 50%. **Drop "L1 ≥ 85%" to "L1 ≥ 80%"** to allow musl-only flag-bundle work in M2.

- **R2** (1-of-26 dangling tool UID) — STILL RELEVANT, but M2 only encounters foreign_deps once (the `util/_/datetime/parser.rl6.cpp` R6 node, whose `foreign_deps.tool` points at the host `ragel6` LD UID — IN-graph, not dangling). The 1 dangling external UID (`XO1d8CLk3qDKv0XQTlDKmQ`) refers to the Python interpreter / compiler binary registered outside the build graph. - **R4** (op-type coverage) — M1 had CC + AR. M2 adds **LD** (4 cmds: `vcs_info.py` → `clang` compile-vcs → `link_exe.py` → `fs_tools.py link_or_copy_to_dir`), **AS** (assembler — single 94-arg clang invocation for `.S` files), **JS** (`gen_join_srcs.py` for `JOIN_SRCS`), **R6** (`ragel6` invocation; deps include 1 host-tool foreign_dep), **CP** (single `musl.py.pyplugin` copy via `fs_tools.py copy`).
- **R5** (Go version) — STILL RELEVANT. `go.mod` says `go 1.25`.
- **R6** (multi-cmd LD shape) — CONFIRMED: 4 cmds in one Node. cmd[0] = vcs_info, cmd[1] = compile `__vcs_version__.c` (94-arg clang), cmd[2] = link_exe (73 args; libs in topological order, NOT alphabetical), cmd[3] = link_or_copy.
- **R7** (BufferedEmitter memory) — for 1,926 nodes, fine (~50 MB peak).
- **R8** (map iteration determinism) — D14 still binding.
- **R9** (`ya.make` macros beyond M1 subset) — HIT IMMEDIATELY: PEERDIR closure hits `IF/ELSE/ELSEIF/ENDIF`, `INCLUDE(...)`, `JOIN_SRCS(name srcs...)`, `ADDINCL(...)`, `CFLAGS(...)`, `LDFLAGS(...)`, `SRCDIR(...)`, `RECURSE(...) / RECURSE_FOR_TESTS(...)` (M2 IGNORES), `NO_*()` family, `LICENSE/LICENSE_TEXTS/VERSION/ORIGINAL_SOURCE/WITHOUT_LICENSE_TEXTS` (metadata, ignored). M2 PR-13 introduces a shared **macro evaluator** that resolves `IF` predicates against a fixed environment.
- **NEW R10** — naming convention for AR archives is **non-trivial**: `util` → `libyutil.a` (special), `contrib/libs/X/Y/Z` (depth ≥ 4) → `liblibs-X-Y-Z.a` (drop `contrib`), `library/cpp/X/Y` (depth ≥ 4) → `libcpp-X-Y.a` (drop `library`), depth-3 paths keep all components. Verified against 24 reference AR outputs.
- **NEW R11** — pure-x86_64 platform is HOST. M2's TargetCfg = aarch64. M2 doesn't recurse into host, but the LD node's deps closure includes one host node (the dangling `ragel6`-built host LD). Stub host strategy (Q4).
- **NEW R12** — `module_tag: global` is real (1 node — `tcmalloc/no_percpu_cache`'s `.global.a` from `GLOBAL_SRCS(...)`).
- **NEW R13** — `JOIN_SRCS(allname srcs...)` produces a JS node whose **inputs are non-alphabetical**.
- **NEW R14** — LD node's `cmd[2]` lib list is **non-alphabetical and non-sorted**: it follows ymake's link topological order.
- **NEW R15** — `target_properties` shape varies by op-type. CC nodes carry only `module_dir`. AR nodes carry `module_dir` + `module_lang` + `module_type` (and `module_tag` when `global`). LD nodes carry `module_dir` + `module_lang` + `module_type=bin`.

## 4. Cross-cutting decisions to lock for M2

- **D19 — Recursive PEERDIR walk.** Depth-first, post-order, with cycle detection. Memoized by `targetDir`. Visit order = the order of paths inside `PEERDIR(...)` macro (preserve as-declared, not sorted) — produces LD link-order R14. M2 stays single-threaded.
- **D20 — Stub host strategy.** When recursive walk encounters host-tool reference, emit `foreign_deps` entry with the **reference graph's actual host UID string** (`OsvsEu9xnOqLNXi3INZ9CQ` for ragel6, `XO1d8CLk3qDKv0XQTlDKmQ` for the dangling external). Hardcoded in `host_stubs.go` as `map[string]string` keyed by tool path. M4 replaces with real host-pass.
- **D21 — Multi-source CC emission.** One `EmitCC` call per source. Per-source flag override NOT required for M2. Emission order = source declaration order. **PR-10-D03 refactor** lands alongside: `EmitCC` returns `(ref NodeRef, outputPath string)`.
- **D22 — LD rule shape.** ONE Node with FOUR Cmds (verified shape). Single `EmitLD(cfg, moduleDir, mainObjRef, mainObjPath, peerLibRefs, peerLibPaths, vcsScriptPath, ...)` in `ld.go`.
- **D23 — Accept PROGRAM in addition to LIBRARY.** PR-12 generalizes to accept `PROGRAM(...)` and emit an LD node closing the module instead of an AR node.
- **D24 — `--target` for `gen`.** Same flag, different target value: existing flag accepts `tools/archiver`. No CLI shape change.
- **D25 — Recursive parser parallelism.** Single-threaded for M2. M3 introduces `golang.org/x/sync/errgroup`.
- **D26 — JSON-diff library for L3 humans.** Defer until M5/M6.
- **D27 — Macro evaluator.** Tiny ad-hoc evaluator. Supports: `IF(IDENT)`, `IF(IDENT AND IDENT)`, `IF(IDENT OR IDENT)`, `IF(NOT IDENT)`, `IF(IDENT AND NOT IDENT)`. Bound vars: hardcoded string-set in `macros.go` matching what produced the reference graph. Hard error on unknown identifier inside an IF expression.
- **D28 — Parser extension scope.** PR-13 extends the parser AST with `IfStmt`, `IncludeStmt`, `JoinSrcsStmt`, `AddInclStmt`, `CFlagsStmt`, `LDFlagsStmt`, `SrcDirStmt`, `GlobalSrcsStmt`. The `NO_*()` family + `LICENSE`/`VERSION`/etc. stay as `UnknownStmt`.
- **D29 — Output path mangling for non-`.cpp` sources.** `.S` files compile to `$(BUILD_ROOT)/<moduleDir>/_/<basename>.o` (note `_/` prefix). `.rl6` files produce intermediate `.cpp` at `$(BUILD_ROOT)/<moduleDir>/_/<src>.cpp`. Verified.

## 6. PR-12 detailed brief

**Title:** Generalize `Gen()` to recursively walk PEERDIR closure; accept PROGRAM modules; refactor `EmitCC` signature to return `(NodeRef, string)`.

**Goal:** Land the architectural pivot from "single-leaf LIBRARY emitter" to "recursive PEERDIR walker that emits per-module subgraphs and threads dependencies between them," without breaking M1 acceptance and without yet supporting macros (IF/INCLUDE/JOIN_SRCS/...).

PR-12 is intentionally narrow: accepts modules whose ya.make uses ONLY {LIBRARY|PROGRAM, SRCS, PEERDIR, END, NO_*, SET, LICENSE, VERSION, ORIGINAL_SOURCE, LICENSE_TEXTS, WITHOUT_LICENSE_TEXTS, RECURSE, RECURSE_FOR_TESTS, ALLOCATOR_IMPL, NEED_CHECK, IDE_FOLDER}; anything else throws `gen: PR-12 does not yet support macro %q (deferred to PR-13)`.

**Files to change:**

- **`cc.go`** — Change `EmitCC(cfg PlatformConfig, moduleDir string, srcRel string, emit Emitter) NodeRef` to `EmitCC(...) (NodeRef, string)`, returning the NodeRef AND the output-path string (`"$(BUILD_ROOT)/" + moduleDir + "/" + filepath.Base(srcRel) + ".o"`). Keep existing 101-arg byte-exact body unchanged. The TODO at lines 44-48 about no-libc gating stays — PR-14's job.
- **`cc_test.go`** — Update `TestEmitCC_BuildCowOn_ByteExact` to consume the new return. Add an assertion that `outPath == "$(BUILD_ROOT)/build/cow/on/lib.c.o"`.
- **`gen.go`** — Rewrite per the algorithm in §6 of this plan (cycle detection, memoization, recursive depth-first walk, PROGRAM accepted as placeholder treating as LIBRARY for now; PR-19 swaps in real LD).
- **`gen_test.go`** — Add `TestGen_AcceptsProgramModule_Synthetic`, `TestGen_PeerdirCycle_Throws`, `TestGen_RejectsUnsupportedMacro`. Keep existing M1 byte-exact test green.
