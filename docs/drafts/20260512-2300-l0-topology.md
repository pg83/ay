## L0 algorithm reminder

`sha1(kv.p ‖ sorted(child_fingerprints))` where children come from each
node's `deps` array (node-to-node refs). Pure topology / dep-graph.
Therefore the 721 unpaired fingerprints reduce to two strictly smaller
sets:

- **161 LEAF divergences** — nodes whose own dep-keyset differs from REF.
- **560 PROPAGATED** — own deps identical, but child fp differs because
  some transitive descendant is a leaf.

Fixing leaves cascades up through `parents`-closure; the loop terminates
once every leaf is paired.

## Root cause #1 — CC consumer nodes miss PB/EV/CF codegen deps (96 + ~21 CC leaves → 312+ fp)

### Observation

96 CC nodes (all C++ TUs in modules whose peer-closure contains a `.ev`
or proto-library that produces `events_extension.pb.h`) have the same
3-element REF-only signature in `deps`:

```
$(BUILD_ROOT)/devtools/ymake/diag/common_msg/msg.ev.pb.cc      (kv.p=EV)
$(BUILD_ROOT)/devtools/ymake/diag/trace.ev.pb.cc               (kv.p=EV)
$(BUILD_ROOT)/library/cpp/eventlog/proto/events_extension.pb.h (kv.p=PB)
```

OUR `mkcmd.cpp.o` (representative): `inputs` contains all three plus
`*.pb.h` companions — the include scanner already reached them. But the
node's `deps` array has only `[stats_enums.h_serialized.cpp (EN)]`.
REF has `[msg.ev.pb.cc, stats_enums.h_serialized.cpp, trace.ev.pb.cc,
events_extension.pb.h]`.

Smaller variants (signature class size in parens):

- (2) `__res.pyc.inc` missing from `library/python/runtime_py3/__res.cpp.{o,pic.o}`
- (2) `sitecustomize.pyc.inc` missing
- (2) `enum_options.pb.h` missing from `library/cpp/protobuf/json/{json2proto,proto2json_printer}.cpp.o`
- (2) `retry_options.pb.h` missing from retry / symbols TUs
- (1) `merge.pb.h` missing from `protobuf/util/merge.cpp.o`
- (1) `internal.pb.h` missing from `eventlog.cpp.o`
- (1) `TConfLexer.cpp` (CP→CC) missing from cmd_parser.cpp.o
- (1) `TConfLexer.cpp` (CP→CC) missing from confreader.cpp.o
- (3) `build_info.cpp` / `sandbox.cpp` / `buildinfo_data.h` (CF→CC) missing
- (1) `parsestruct.rl.tmp` (R5→CC chain) missing

All have the same shape: **a TU includes a generated header whose
producer is a PB/EV/CF/CP/R5/PYC node, but our codegen-dep-resolver
only knows about EN producers**.

### Root cause (code-level)

`gen.go:6258-6300` — `resolveCodegenDepRefs` only consults
`ctx.enOutputs`. Comment lines 6267-6272:

> "Today only EN producers register a (path → NodeRef) map. PB / EV /
> CF / etc. follow the same emission order (codegen before consumer
> CC) but their output→ref mappings live in per-file local slices, not
> genCtx; extending this helper to those families requires lifting
> those maps onto genCtx first."

This is a known TODO. Lifting EV / PB / CF / CP / R5 / PYC producer
maps onto `genCtx`, and extending `resolveCodegenDepRefs` to cover
them, eliminates **117 CC leaves** and **all 542 + 312 + 6 + 6 + 6
fp ancestors** that propagate through their `parents`-closure.

Reference upstream: `devtools/ymake/diag/common_msg/ya.make` declares
`PROTO_LIBRARY() PEERDIR(library/cpp/eventlog/proto) SRCS(msg.ev)` —
this triggers `_CPP_EVLOG_CMD` (`build/conf/proto.conf:480`) whose
hidden `.PEERDIR=library/cpp/eventlog contrib/libs/protobuf` pulls in
`library/cpp/eventlog/proto` (an EV+PB producer). The C++ scanner
follows `trace.h → trace.ev.pb.h → events_extension.pb.h` and the
generated `.pb.cc` files get listed in `inputs` but not as graph deps.

### Yield

Fixing the EV+PB family alone closes **96 CC leaves + ≤300
propagated** (the propagated 542-impact CC clusters from `__res.pyc.inc`
and `sitecustomize.pyc.inc` overlap with the EV/PB ancestor set).
Net L0 yield: **~85–90% of unpaired fingerprints** in a single PR.

≤82 propagated** (objcopy → AR → LD), eliminating both the L0 PY
cluster and the L2 AR `libpy3tools-python3-Lib.global.a` inputs delta.
