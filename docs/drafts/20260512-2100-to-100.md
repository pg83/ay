## Cluster A — openssl `AR_PLUGIN(ar)` not honoured (5 nodes)

**Affected:**
- L2/L3 [AR] `contrib/libs/openssl/libcontrib-libs-openssl.a` (aarch64, x86_64).
- L2 (consequential) all openssl AS member-inputs carry `crypto/../asm/...`
  instead of canonical `asm/...`.

**Two distinct defects in this cluster:**

1. **`AR_PLUGIN(ar)` macro unimplemented.** `contrib/libs/openssl/ya.make:54`
   declares `AR_PLUGIN(ar)`, expanded by ymake.core.conf:3396-3398 to
   `SET(_AR_PLUGIN ar.pyplugin)`. The reference AR cmd_args carry
   `--plugin $(SOURCE_ROOT)/contrib/libs/openssl/ar.pyplugin` and the .pyplugin
   file lands in `inputs[]`. Our `ar.go` ignores `_AR_PLUGIN`. **Fix:** parse
   `AR_PLUGIN(<name>)` in yamake.go (new `ArPluginStmt`), thread to
   `genCtx.moduleData.arPlugin`, and append `--plugin <abspath>` + add the
   .pyplugin to AR inputs in `emitARNode` (ar.go:126).

2. **AS member-input path not `path.Clean`'d.** `gen.go:5509` builds
   `asInputPath = "$(SOURCE_ROOT)/" + srcDir + "/" + srcRel`. When openssl
   uses `SRCDIR(crypto)` and SRCS `../asm/aarch64/...` the literal
   `crypto/../asm/...` flows into AR memberInputs, even though the AS node
   itself uses the cleaned form (`composeASPaths` in as.go:369 calls
   `path.Clean`). **Fix:** apply `path.Clean(srcDir + "/" + srcRel)` at
   gen.go:5509, mirror of as.go:369. One-line change.

## Cluster B — antlr4 `RUN_ANTLR4_CPP` single-grammar Listener default inverted (5 nodes)

**Affected:**
- L3 [JV] `TConfLexer.cpp` (aarch64): REF `-no-listener`, OUR `-listener`.
- L2 (consequential, downstream of the JV's outputs): TConfLexer.g4.cpp CP/CC,
  TConfParser.g4.cpp CP/CC (REF closures pick up libcxx/glibcasm headers our
  JV outputs don't trigger).

**Defect.** `yamake.go:1200` constructs `RunAntlr4CppStmt` with
`Listener: true`. The reference treats single-grammar antlr4 (`RUN_ANTLR4_CPP`)
the same as split (`RUN_ANTLR4_CPP_SPLIT`): default `listener=false`. ya.make
at `devtools/ymake/lang/ya.make:23-31` uses neither `LISTENER` nor
`NO_LISTENER`; reference emits `-no-listener`. **Fix:** flip
`yamake.go:1200` default to `Listener: false` and adjust the
`NO_LISTENER`/`LISTENER` arms in parseRunAntlr4Cpp accordingly.

## Cluster C — RUN_PROGRAM closure under-propagates header includes (3 nodes)

**Affected:**
- L2 [PR] `devtools/ymake/symbols/dep_types.h_dumper.cpp` and downstream
  CC + AR (`libdevtools-ymake-symbols.a`). REF inputs include
  `library/cpp/fieldcalc/{field_calc.h, field_calc_int.h, lossy_types.h}`,
  `library/cpp/deprecated/autoarray/autoarray.h`, `util/generic/cast.h` —
  headers reachable from the PR's `IN file_store.h`/`IN cmd_store.h` via
  the module's PEERDIR(library/cpp/fieldcalc).

**Defect.** `EmitPR` (m3_misc.go:664) accepts `inputClosure` from its caller,
which apparently doesn't include the transitive header closure of the `IN`
files when those headers route through peer PEERDIRs. The header `file_store.h`
includes `field_calc.h` from a peer; the scanner's PR-side walkClosure is not
threading peer-include closures through. **Fix:** the call site at
m3_misc.go:1637 must include peer-AddIncl-resolved transitive closure of
every `IN <header>` file, identical to the closure CC nodes get. (Likely
one-line fix in the caller's `walkClosure` invocation context.)

## Cluster D — runtime_py3 PR positional-arg substitution incomplete (2 nodes)

**Affected:** L3 [PR] `library/python/runtime_py3/__res.pyc` (aarch64, x86_64).

**Defect.** `EmitPR` only substitutes `${ARCADIA_ROOT}` and rewrites bare-IN
filenames. RUN_PROGRAM args for py3 yapyc3 generation include `${MODDIR}` and
bare OUT_NOAUTO basenames (`__res.pyc`, `sitecustomize.pyc`,
`mod=${MODDIR}/__res.py`). Reference expands `${MODDIR}` →
`library/python/runtime_py3` (no leading `$(BUILD_ROOT)/`) and bare-OUT
basenames → `$(BUILD_ROOT)/library/python/runtime_py3/__res.pyc`. **Fix:** in
EmitPR (m3_misc.go:693-701) add: `strings.ReplaceAll(a, "${MODDIR}", instance.Path)`;
add bare-OUT/OUT_NOAUTO/STDOUT-set rewriting symmetric to inSet.

## Cluster E — python3 Lib `.global.a` missing 535 `.py` source inputs (1 node, big)

**Affected:** L2 [AR] `contrib/tools/python3/Lib/libpy3tools-python3-Lib.global.a`
(aarch64). 535 raw `.py` paths missing from `inputs[]`. The `.yapyc3` outputs
are present; only the source `.py` paths are absent. (Mirror node on x86_64

**Defect.** PY_SRCS-resfs path likely fails to thread per-`.py` source into

`addMemberInputs` calls from `emitPySrcs` (gen.go:4679+) and ensure the raw
.py source paths join memberInputs alongside the .yapyc3 outputs.

## Cluster F — ragel6 flags hardcoded `-CT0` (3 nodes)

**Affected:**
- L3 [R6] `devtools/ymake/lang/makelists/makefile_lang.rl6.cpp`: REF `-lF1`,
  OUR `-CT0` (target-local `SET(RAGEL6_FLAGS -lF1)` not honoured).
- L3 [R6] `util/_/datetime/parser.rl6.cpp`: REF `-CG2`, OUR `-CT0` (release
  build default).

**Defect.** `r6.go:108` hardcodes `-CT0`. ymake_conf.py:2271-2277 selects
`-CG2` (release) vs `-CT0` (debug) globally; modules may override via
`SET(RAGEL6_FLAGS -lF1)`. **Fix:** thread `instance.Flags.Ragel6Flags`
through; default to `-CG2` (release matches reference), and honour
per-module `SET(RAGEL6_FLAGS ...)` (parser already exists in our SET
evaluator).
