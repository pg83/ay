## PRs ordered by cascade-yield

These 4 PRs collectively close every L2/L3 row above. Order chosen so
that earlier PRs unblock the closure-propagation that later PRs rely on.

### PR-final-1 ŌĆö codegen-helper closure registry expansion (~22 L2 + 3 L3)

**Subsumes:** classes D, E, G, J, K, L, M, N, plus L3 ╬▓.

**Defect.** `codegen_registry.go` knows about a small set of generator
helpers (proto runtime, ev compiler), but several more generator
families inject helper-script `inputs[]` and `-I` flags in upstream
ymake that we don't yet emit:

- proto runtime: `cpp_proto_wrapper.py`, `generated_message_bases.h`,
  `wire_format.h`, `map_entry.h` (D: 7 AR rollups + 3 CC consumers)
- build_info: `build_info_gen.py`, `xargs.py`, `yield_line.py`,
  `configure_file.py` + `-I$(BUILD_ROOT)/library/cpp/build_info` on
  every consumer of `build_info.h` (E + 2 Z + L3 ╬▓: 5 occurrences)
- antlr3/antlr4: `CmdLexer.cpp`, `stdout2stderr.py`, `antlr.jar` on
  parser .cpp (G: 2)
- runtime_py3: `objcopy.py`, `__res.py`, `entry_points.py` and
  `sitecustomize.py` in AR closures (J: 4)
- build/scripts helpers in lang AR (K: 2)
- py3cc/slow LD: `objcopy_<sha>.o` chunked-resource outputs + main.py
  (L: 1)
- event2cpp LD: drop `.pb.h` from LD inputs (M: 1)
- ymake-symbols PR-node misses `library/cpp/fieldcalc/*.h` closure (N: 1)

**Where.** Extend `codegen_registry.go` per-family entries, then
emit-time wire-up in `pb.go`, `ev.go`, `m3_misc.go` (build_info), `ar.go`
(member-closure roll-up), `ld.go` (event2cpp drop, py3cc chunk
spill-over), `cp.go` (antlr helper tail). Pattern is already
established (see PR-M3-F-7-D).

**Verification.** L2 drops from 50 ŌåÆ ~16; L3 from 32 ŌåÆ ~25.

### PR-final-2 ŌĆö emit-time path canonicalization + sibling-output suppression (~9 L2 + 2 L3)

**Subsumes:** classes C, H + L3 ╬┤ + Z (openssl asm).

**Defect-1.** `composeASPaths` already runs `path.Clean` on `inputPath`
when SrcDir is in use, but the resulting **output** path retains the
literal `crypto/../asm/...` shape because the .o path is computed from
`composeSrcDirOutputRel` first and the input gets re-derived without
`path.Clean` in non-srcDir branches. Probe shows openssl `.a` has 28
OUR-only entries with `crypto/../asm/...` that map 1:1 to 28 REF-only
entries with `asm/...` via `os.path.normpath` ŌĆö i.e. pure path
denormalization. Fix: `path.Clean` applies to *both* arms of
`composeASPaths`, or normalize at AR-roll-up time.

**Defect-2.** PB `.cc.o` nodes (devtools/ymake/diag/common_msg/msg.ev.pb.cc.o,
trace.ev.pb.cc.o) include their own sibling `.pb.h` as an input.
Reference treats the .pb.h as a `_/` codegen-output, not an input of its
co-generated .cc.o. Suppress this self-edge in `pb.go` / `ev.go`.

**Defect-3.** openssl AR also misses `ar.pyplugin` (2 L2 + 2 L3 = `--plugin`
+ `ar.pyplugin` argv pair). The `ar.pyplugin` path becomes a closure
input on the AR and a `--plugin` `<path>` argv-pair on cmd_args; this is
a `SET_APPEND(AR_TYPE ŌĆ”)` style emission triggered by a `USE_PLUGIN`
hint in the openssl ya.make.

**Where.** `as.go::composeASPaths`, `pb.go`, `ev.go`, `ar.go` (plugin),
`codegen_registry.go` (register openssl `ar.pyplugin`).

**Verification.** L2 drops by ~9; L3 by ~2.

### PR-final-3 ŌĆö over-emission of stats_enums into R6 + sibling-CC patches (~6 L2)

**Subsumes:** class A entirely.

**Defect.** Six R6 nodes (e.g. `go_import_parser.rl6.cpp`,
`makefile_lang.rl6.cpp`, `macro_parser.rl6.cpp`,
`nlg_includes_parser.rl6.cpp`, `cython_parser.rl6.cpp`,
`ts_import_parser.rl6.cpp`) carry OUR-only inputs:
`$(BUILD_ROOT)/devtools/ymake/diag/stats_enums.h_serialized.{cpp,h}`.

`stats_enums.h_serialized.*` is an enum-serializer codegen output owned
by `devtools/ymake/diag`. It's being leaked into R6 inputs of every
module that **PEER-depends** on `devtools/ymake/diag` (a transitive
header closure escape). R6 inputs should be limited to the ragel binary
+ `.rl6` source + its transitive `#include` closure of *headers* ŌĆö
generated `.cpp` siblings of the enum serializer must not appear.

**Where.** `r6.go::EmitR6` (or wherever the closure is composed) ŌĆö the
peer-header closure must exclude `_serialized.cpp` outputs (same way
PB rollups already exclude `.pb.cc`-self outputs).

**Verification.** L2 drops by 6.

### PR-final-4 ŌĆö per-module RAGEL6_FLAGS + per-CC SRCFLAGS + axis flags (~5 L2 + 11 L3 + 13 order-only)

**Subsumes:** classes B, F, ╬Č, ╬│, ╬Ę + L3 ╬▒ (order-only).

**Defect-1 (╬Č, 2).** `r6.go::composeRagel6CmdArgs` hard-codes `-CT0` at
line 108. Upstream selects `-CG2` for release, `-CT0` for debug
(`ymake_conf.py::set_default_flags`). Per-module `SET(RAGEL6_FLAGS ŌĆ”)`
overrides (`util/ya.make` for `util/datetime/parser.rl6` ŌåÆ `-CG2`;
`devtools/ymake/lang/makelists/ya.make` ŌåÆ `-lF1`) are also unhonored ŌĆö
the module-level SET is parsed but not threaded into the R6 emitter.
Reference `sg2.json` is a release build, ours is debug.

**Defect-2 (╬Ę, 1).** `m3_misc.go::170-272` flips `-listener` /
`-no-listener` on JV cmd_args; one `TConfLexer.cpp` node sits on the
wrong side of the axis (LISTENER override).

**Defect-3 (╬│, 4).** `-Wno-unused-variable` is missing on antlr-generated
CC nodes (`CmdParser.g4.cpp.o`, `TConfParser.g4.cpp.o`). Upstream injects
this via the antlr macro's `CFLAGS_FOR_GENERATED` pragma.

**Defect-4 (B, 4).** Antlr CC/CP nodes also miss host-CC header closure
(libcxx `inttypes.h`, glibcasm `libc-header-start.h`, `stdint-intn.h`).
Likely co-fixed with ╬│: the antlr-codegen path lacks both the
generated-CC's CFLAG bundle and its header-closure resolution.

**Defect-5 (F, 5).** musl-self host CC nodes
(`crypto/rand/rand_egd.c.pic.o` and siblings) miss their musl
`include/unistd.h` + `arch/x86_64/bits/posix.h` closure. Likely an
edge case in the CC composer where `LibcMusl=true && PIC=true` skips
the musl-self header-closure pass.

**Defect-6 (╬▒, 13).** AR/R6/PY nodes whose `cmd_args` match REF as a
multi-set but not in order. Sort the right slots (e.g. AR member order
deterministically by module-DFS rather than insertion order).

**Where.** `r6.go` (RAGEL6_FLAGS plumbing), `m3_misc.go` (JV LISTENER
axis), `cc.go` (CFLAGS_FOR_GENERATED for antlr; musl-self header
closure for PIC=true), `ar.go` (member ordering).

**Verification.** L2 drops by 5; L3 by ~11 + 13 (order).
