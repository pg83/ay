# T-58 ‚Äî default old-flex `_SRC("l"/"lex"/"lpp")` source generation

## Upstream mechanism (build/conf/bison_lex.conf)

`_SRC("l")` (and the `lex`/`lpp` aliases, same body) emit a lex/flex node:

```
.CMD=$_FLEX_TOOL $LEX_FLAGS ${SRCFLAGS} $_FLEX_HEADER ${hide:_FLEX_GEN_EXT} \
     -o${output;suf=$_FLEX_GEN_EXT:SRC} ${hide;output_include:"util/system/compiler.h"} \
     ${input:SRC} ${hide;kv:"p LX"} ${hide;kv:"pc yellow"}
.ADDINCL=$_FLEX_TOOL_DIR
```

Defaults (the `when ($_BISON_FLEX_SET_DEFAULTS == "yes")` block, which is the
active config):

- `_FLEX_TOOL = ${tool:"contrib/tools/flex-old"}` ‚Üí binary `$(B)/contrib/tools/flex-old/flex`
- `_FLEX_TOOL_DIR = contrib/tools/flex-old` (the `.ADDINCL`)
- `LEX_FLAGS =` (empty), no per-source `SRCFLAGS` for the representatives
- `_FLEX_HEADER =` (empty ‚Äî old flex emits no header)
- `_FLEX_GEN_EXT = .cpp`

So for `kernel/remorph/literal/lexer.l` the LX node is exactly:

```
cmd:  $(B)/contrib/tools/flex-old/flex -o$(B)/kernel/remorph/literal/lexer.l.cpp \
      $(S)/kernel/remorph/literal/lexer.l
env:  ARCADIA_ROOT_DISTBUILD=$(S)
kv:   p=LX pc=yellow   deps:[flex-old tool LD]   module_dir=kernel/remorph/literal
```

Verified against `/home/pg/monorepo/4/sg7.json` (normalized LX node). The node's
`inputs` are the flex tool, the `.l` source, and the **generated-output include
closure** of `lexer.l.cpp` (`generatedOutputClosure`, i.e. the same closure the
bison producer attaches for its header ‚Äî generated `.pb.h` etc. expand to their
`.proto` source + `cpp_proto_wrapper.py`). The generated `.l.cpp` is then
compiled as ordinary C++ and archived; the `.ADDINCL=contrib/tools/flex-old`
puts `-I$(S)/contrib/tools/flex-old` on every compile in the module (so
`<FlexLexer.h>`, pulled in via `lexer.h`, resolves) ‚Äî confirmed in the ref on
both `lexer.l.cpp.o` and the sibling `literal.cpp.o`.

The generated `.l.cpp`'s parsed includes = `util/system/compiler.h`
(`output_include`) ++ the `.l` source's own `#include`s (it copies its `%{‚Ä¶%}`
prologue verbatim into the output).

## Our side / seam

`emit_sources.go` already dispatches `.y`‚Üíbison, `.rl6`‚Üíragel. Flex is the
ragel-shaped case: one tool, generates a `.cpp` (no header consumed by
siblings), compiled + archived. Mirror `emitLibraryRagel6Source` (single-pass,
in the pass-1 codegen loop) rather than the bison two-phase pre-pass.

`.l` currently classifies as `srcExtRegular` and hits the unsupported-source
warning. The include parsers already know `.l`/`.lex`/`.lpp` (CIncludeParser).
