# T-54 plan: parse-time variable expansion in INCLUDE paths

## Upstream mechanism

`devtools/ymake/lang/makefile_reader.cpp:49-60` — `TMakeFileReader::Statement`
handles `INCLUDE` specially: it computes `TString name = EvalExpr(Context->Vars(),
args[0])` BEFORE calling `Include(name, ...)`. `Include` (`makefile_reader.cpp:97-108`)
then sends an *already-evaluated* name through `Conf.CanonPath` (absolute) or
`ResolveIncludePath` (relative). `lang/resolve_include.cpp:11-21` strips `${CURDIR}/`
and `${ARCADIA_ROOT}/` AFTER evaluation and asserts the remainder contains no `$`,
proving variable substitution happens before prefix stripping / source-root
resolution. `SET`/`DEFAULT` write into the same `Context->Vars()`
(`lang/eval_context.cpp:321-339`), so an earlier `SET` — including one from an
earlier `INCLUDE` — is visible to a later `INCLUDE` argument. `EvalExpr`
(`lang/eval.cpp:43-45`) is the same `$`-substitution family ay already models with
`expandStmtToken` / `expandScalarVarRef`.

ay currently only strips a literal `${ARCADIA_ROOT}/` prefix
(`yamake.go:2041`); any remaining `${VAR}` stays literal, the target file is
reported absent, and ay silently skips the include (`yamake.go:2072-2074`). The
skipped include is the only route that sets `FS_CODEGEN_FEATURE_PEERDIRS`, so the
fs_codegen feature subtree never enters the sg7 graph.

## Change

One parser-level behavior: expand `$`-references in each `INCLUDE` path argument
against a parse-time `SET` environment before `${ARCADIA_ROOT}/` stripping and
relative-path resolution. Reuse the existing `expandScalarVarRef` /
`Environment` machinery — no bespoke interpolator, no second parse, no new data
channel.

### Files touched

- `yamake.go`
  - `IncludeState` gains `env Environment`. `newIncludeState` initializes it.
    `IncludeState` is created fresh per top-level `parseFile` and threaded through
    recursive includes, so the env is module-parse scoped and shared across the
    same module's includes — exactly the upstream `Context->Vars()` lifetime.
  - `parseMacroInto`: after building an ordinary stmt, if it is a `*SetStmt`,
    fold the expanded value into `p.includes.env` immediately
    (`env.setFromString(NameEnv, expandScalarVarRef(Value, env))`). The emitted
    `SetStmt` is unchanged so gen-time collection still rebuilds `d.setVars` from
    the same statement sequence.
  - `expandOneInclude`: first `rel = expandScalarVarRef(rel, p.includes.env)`;
    every subsequent operation (`${ARCADIA_ROOT}/` strip, absolute-path reject,
    relative resolve, once/cycle keys, missing-file lookup) uses the expanded
    spelling. An unresolved `${VAR}` survives verbatim (existing
    `expandBracedVars` semantics) → the literal target is absent → existing silent
    skip. No new throw.

### Behavior preserved

Multi-arg left-to-right (`expandInclude` already iterates in order, so a `SET`
from arg `a` is visible to arg `b`); relative-from-including-dir; `${ARCADIA_ROOT}/`;
absolute-path rejection (now after expansion — `${ABS}`→`/tmp/x` still rejected);
`INCLUDE_ONCE`/cycle keyed on the resolved expanded target; missing-file silence.

### Conditional-branch note

ay inlines INCLUDEs inside IF bodies unconditionally today (parse is structural,
not evaluated). The env hook in `parseMacroInto` therefore also records SETs that
appear inside IF bodies, in source-text order — consistent with ay's existing
"parse all branches" structure. A regression test covers a SET inside an IF body
feeding a later INCLUDE. The gate (sg2–sg6 byte-exact) is the arbiter; if a gating
contour regresses from IF-body SET leakage, narrow the hook to top-level only.

## Tests (test-first)

`yamake_test.go`:
- variable-bearing INCLUDE via `INCLUDE(cfg/name.inc)` then
  `INCLUDE(${ARCADIA_ROOT}/gen/artefacts_${CONFIG_NAME}_/peers.lst)`: the peers
  file is included and its `SET(FEATURE_PEERDIRS feature/model)` appears before the
  module body; no `*IncludeStmt` survives.
- left-to-right multi-arg: `INCLUDE(a.inc b_${NAME}.inc)`, `a.inc` sets `NAME`.
- missing-include silence: `INCLUDE(${ARCADIA_ROOT}/missing_${NAME}.inc)` no error,
  no appended stmts.
- absolute-path rejection after expansion: `SET(ABS /tmp/nope.inc); INCLUDE(${ABS})`.
- cycle with expanded key: `SET(DIR .); INCLUDE(${DIR}/a.inc)` self-cycle.
- unresolved variable guard: `INCLUDE(${UNSET}/x.inc)` is silently skipped.
- IF-body SET feeding a later top-level INCLUDE.

`gen_test.go`:
- minimal graph repro reaching a leaf PY3 proto output through a peers list
  included via a variable-bearing path (plan T-53 memfs shape).

## Gate effect

- sg2–sg6 stay byte-exact OK (the risk is a gating variable-bearing INCLUDE that
  ay previously skipped; rejected if any regress).
- sg7: fs_codegen loads `feature_peerdirs.lst`, the yabs feature subtree appears,
  and the `library/cpp/protobuf/json/proto` PY3 sidecar outputs are emitted by the
  existing `emit_py_proto` path. matched must not drop.

PROTO_LIBRARY ONLY_TAGS/INCLUDE_TAGS, generic PEERDIR expansion, PY3 archive /
