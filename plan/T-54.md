# T-54 — Close the stdout stream/read-failure false-ACCEPT in ./acceptance

## Scope

This ticket carries forward the near-complete T-51 branch (commit `3a721ff`,
cherry-picked here) and closes the one remaining reviewer-blocked hole. Scope is
strictly `./acceptance` and its focused tests in `dev/test_acceptance.py`. No
graph generation, sg fixtures, or T-36/T-48/T-50/T-52 behavior is touched.

This is harness/infrastructure code, not graph reproduction — the "upstream
mechanism" framing does not apply. The contract being reproduced is the gating
policy in PROMPTS.md; the harness must never ACCEPT a candidate whose validate.py
transcript was truncated by an infrastructure failure.

## The defect (reproduced hypothesis)

`run_validate()` streams `validate.py` stdout in a `for ln in proc.stdout` loop
guarded by `except Exception`. On a pipe read/wait failure *after* some case
lines were already parsed, the handler appends a `<stream error: …>` marker and
logs it, but does **not** set `RunResult.error`. Control then falls through to
`return RunResult(label, True, returncode=proc.returncode, output=output)` —
`completed=True`, `error=""`.

`usability_reason()` therefore sees: empty `error`, a `returncode` in
`{0,1}` (the child would have exited normally), no Python traceback (the raise
is in the *orchestrator's* read, not in validate.py), and a non-empty `status`
(because `[sg2] OK` parsed). It returns `""` → the side is treated as usable →
`decide()` compares only the surviving common cases → the truncated run can
ACCEPT. This is the header-only / truncated-transcript false ACCEPT the reviewer
flagged.

## Fix

In `run_validate()`, record the stream (and wait) failure into a local
`stream_error` and return an **unusable** `RunResult` (`completed=False`,
`error=<side-specific stdout diagnostic incl. the exception repr>`), regardless
of how many case lines parsed and regardless of the child return code. The wait
in the `finally` is likewise guarded so a wait failure is recorded, not raised.

No change to `usability_reason()`/`decide()` is needed: a non-empty `res.error`
already routes the side into the `unusable` REJECT branch in `main()`, which
prints `- <SIDE>: <reason>` plus the output tail on stdout.

## Files

- `acceptance` — capture stream/wait exception into `RunResult.error`.
- `dev/test_acceptance.py` — add `test_stream_read_failure_after_case_line_marks_side_unusable`:
  monkeypatch the module's `subprocess.Popen` so `proc.stdout` yields `[sg2] OK`
  then raises `OSError`; assert `run_validate` returns `completed=False` with the
  side label NEW and the stream exception text in `error`, and that
  `usability_reason` agrees. Keep all 7 existing regressions.
