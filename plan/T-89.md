# T-89 plan: STRUCT_CODEGEN structural residual + tool closure

## Upstream mechanism

`build/internal/conf/codegen.conf` defines STRUCT_CODEGEN as a thin specialization
of BASE_CODEGEN:

```
STRUCT_CODEGEN_OUTPUT_INCLUDES=${hide;output_include:"util/generic/singleton.h"} \
${hide;output_include:"util/generic/strbuf.h"} \
${hide;output_include:"util/generic/vector.h"} \
${hide;output_include:"util/generic/ptr.h"} \
${hide;output_include:"util/generic/yexception.h"} \
${hide;output_include:"kernel/struct_codegen/reflection/reflection.h"} \
${hide;output_include:"kernel/struct_codegen/reflection/floats.h"}

macro STRUCT_CODEGEN(Prefix) {
    .CMD=$BASE_CODEGEN(kernel/struct_codegen/codegen_tool $Prefix $STRUCT_CODEGEN_OUTPUT_INCLUDES)
    .PEERDIR=kernel/struct_codegen/metadata kernel/struct_codegen/reflection
}
```

So `STRUCT_CODEGEN(prefix)` ==
1. `BASE_CODEGEN(kernel/struct_codegen/codegen_tool, prefix)` — produces
   `prefix.cpp` (noauto) + `prefix.h`, runs the fixed tool, pulls the tool's
   ordinary PEERDIR closure into the target graph (the reachability rule already
   modelled for BASE_CODEGEN);
2. seven OUTPUT_INCLUDES attached to `prefix.h` (the `${hide;output_include:...}`
   directives are NOT command-line args — they set the output_includes of the
   generated header, exactly like SPLIT_CODEGEN's `OUTPUT_INCLUDES[]`);
3. two implicit PEERDIRs at the macro's textual position:
   `kernel/struct_codegen/metadata`, `kernel/struct_codegen/reflection`.

Our gen currently lists STRUCT_CODEGEN in `acknowledgedMacros` (gen.go:146), so the
macro is a silent no-op: no producer node, no tool dep, and consumers of the
generated header (e.g. `kernel/xref/xmap.h` -> `xmap_gen.h`) never receive the
seven output_includes. The whole `kernel/struct_codegen/codegen_tool` PROGRAM
closure is absent because nothing depends on it as a tool.

sg7 evidence (`.out/baseline`): exactly two STRUCT_CODEGEN producers in the closure —
`kernel/xref/xmap_gen.{cpp,h}` and `ysite/yandex/erf_format/erf_format_gen.{cpp,h}` —
all right-only, plus the right-only `codegen_tool` program closure.
