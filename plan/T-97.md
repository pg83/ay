# T-97 plan: `ay dev dump diff --pair` duplicate-output content-equal pairing

## Problem (reproduced behavior, not a graph residual)

`dump diff --pair <output>` finds every producer of `<output>` on each side
(`findNodesByOutput`) and then picks one left/right pair to field-diff
(`findNodePairByOutput` → `findMatchingNodePair`). When two producers share the
same output path *and* the same coarse match key (`nodeKVP + platform + host +
PIC`), `findMatchingNodePair` returns the **first content-unequal same-key pair**
it encounters. With two siblings whose only difference is command mode, this
cross-pairs our mode-A node against ref's mode-B node and reports a spurious
command delta — even though exact full-node counterparts exist on both sides
(L-modeA == R-modeA, L-modeB == R-modeB).

Confirmed by the canceled T-92/T-93 investigations:

- Ragel6 `expires.rl6.cpp`: two producers `-CG2` (host) vs `-CT0` (target).
  `dump normalize` strips `host_platform`, R6 commands carry no `.pic.o` member,
  so both variants collapse to one match key. Both already match ref byte-for-byte.
- ASM `<x>.S.o`: PIC/release vs non-PIC/debug producers; asm objects are `.S.o`
  for both contours so the existing `.pic.o`-suffix PIC axis does not separate
  them. Both already match ref.

T-93 stated the generic fix directly: *pair same-key nodes by content as a
multiset — cancel content-equal pairs first.*

## Upstream / mechanism

This is diagnostic tooling, not an emitter; there is no ymake counterpart. The
correct invariant: an output produced by N sibling nodes on each side has parity
iff the two node multisets are equal. A per-output field diff is only meaningful
for producers that have **no** exact counterpart on the other side. So before
falling back to coarse-key matching, cancel every left node that equals some
right node in full (`nodeFieldHashes`). What remains are the genuine residuals.

This is strictly more general than the `.pic.o` / `-fPIC` PIC-axis special case:
it covers PIC, build-mode (`-CG2`/`-CT0`), and any future sibling-variant
collision without enumerating the axis.

## Change

`dump_diff.go`, `findNodePairByOutput` only (the `--pair` path):

1. Add `stripContentEqualPairs(leftNodes, rightNodes)` — greedily match each left
   node to an unused content-equal right node (`dumpDiffNodeContentEqual`),
   returning the residual left/right plus a sample matched pair.
2. In `findNodePairByOutput`, after the empty-side guards:
   - If no residual remains on either side → all shared producers have exact
     counterparts; return a matched pair so `--pair` prints only the header.
   - If both residual lists are non-empty → restrict the existing
     exact/axis/any/fallback search to the residuals (so a real divergence is
     reported against its closest sibling, not an already-matched one).
   - Otherwise (no content-equal pairs, or one-sided residual) → existing
     behavior is preserved unchanged.

No change to `findDumpDiff{Field,Token,Kind,Self}Match`, the aggregate
(`--by-field`/`--by-token`/`--by-kind`/`--roots`) paths, `dumpDiffNodeMatchKey`,
the PIC axis, validate normalization, graph generation, or any emitter command.
