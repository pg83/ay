## Upstream mechanism

`yabs/plutonium/libs/profiles/ya.make` declares each generator as:

```
RUN_PROGRAM(
    yabs/plutonium/libs/profiles/dsp ... --gen-cpp ...
    OUTPUT_INCLUDES ads/caesar/libs/profiles/proto/with_transitive_headers/dsp.pb.h
    OUTPUT_INCLUDES library/cpp/yaff/protobuf.h
    OUTPUT_INCLUDES library/cpp/yaff/reflect.h
    OUTPUT_INCLUDES library/cpp/yaff/yaff.h
    OUT dsp.yaff.h
)
```

`OUTPUT_INCLUDES` is upstream's `${hide;output_include:OUTPUT_INCLUDES}`: induced
include-deps recorded on the OUT file. These deps are *realized as node inputs at
the point the OUT enters an include scan*:

- An **auto cc-source OUT** (caesar `features.gen.cpp`) is C-scanned by its
  downstream CC; the closure surfaces on that **consumer**, and a no-IN producer
  lists only the tool (T-16's verified case, pinned by
  `TestGen_RunProgramOutputIncludesPbHReachConsumerNotProducer`).
- A **header-only OUT** (`dsp.yaff.h`) is never itself compiled. Its
  `output_include` closure is resolved and recorded on the **producer** command
  node — the full `$(S)` include closure of every OUTPUT_INCLUDES file (the
  codegen `.pb.h` → its proto-import graph + protobuf/libcxx C closure, and the
  source-tree `library/cpp/yaff/*.h` → their C closure), with intermediate `$(B)`
  generated `.pb.h` dropped (the proto-import graph already surfaces the `$(S)`
  `.proto` sources, and source-tree WKT `.pb.h` siblings are added explicitly).

T-16 generalized "no IN ⇒ producer carries only the tool" from the caesar
`features.gen.cpp` example. That holds only when the run emits an auto cc-source
OUT. A RUN_PROGRAM that emits **only header OUTs** (no cc-source OUT, no cc-source
STDOUT) with no IN is the dsp class: its OUTPUT_INCLUDES closure has nowhere else
to surface and rides the producer.

## Change

`emit_pr.go` / `prInputClosure`:

1. Replace the blanket `if len(stmt.INFiles) == 0 { return nil }` early return
   with a precise discriminator: compute `hasAutoCCSourceOut` (any OUT or the
   STDOUT is a cc-source ext). Return nil only when there is no IN **and** an auto
   cc-source OUT exists (caesar: closure routes to the consumer).
2. Add the `headerOnlyNoIN := len(INFiles)==0 && !hasAutoCCSourceOut` mode. In
   the OUTPUT_INCLUDES walk:
   - **headerOnlyNoIN**: resolve every OUTPUT_INCLUDES file (codegen `.pb.h` via
     the registry's `OutputPath`, else its source path), walk its full closure,
     keep every `isSource()` entry (drops `$(B)` intermediates), and add the
     source-tree `.pb.h` sibling of any `.proto` (protobuf WKT).
   - **with IN** (unchanged): codegen `.pb.h` OUTPUT_INCLUDES contribute only
     their transitive `.proto` sources (+ WKT `.pb.h` sibling); source-tree
     OUTPUT_INCLUDES are not walked (their C closure is already rooted at IN).

Keep the resolution caching intact (reuse `walkClosureTail` + the shared
`scanIn.ScanCfg`; no new context-dependent resolution).
