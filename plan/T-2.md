# T-2 — Dedup YaFF protoc-plugin tool dep on PB nodes

## Defect (reproduced hypothesis)

A `PROTO_LIBRARY` that enables two YaFF-style plugins sharing the *same* plugin
binary (e.g. `YAFF(...)` + `YAFF_SCHEMA(...)`, both `ToolPath =
library/cpp/yaff/tools/protoc_plugin`) emits its PB producer node with the
plugin's LD node listed **twice** in the graph `deps` array. Upstream lists it
**once**. PB command content (cmds/inputs/outputs/tags) is already byte-identical
— the divergence is purely dep multiplicity, surfacing as the residual sg7
deps/uid-only mismatch for `$(B)/yabs/proto/user_profile.pb.{h,cc}` and its
`.yaff.* / _tsar_vectors.yaff.*` siblings.

## Upstream mechanism

`devtools/ymake/json_visitor.cpp` `TJSONVisitor::PrepareLeaving`: a node's deps
live in `TMakeJSONVisitorNodeData::NodeDeps`, a **`TUniqVector<TNodeId>`**
(`common/uniq_vector.h` — insertion-order preserving, deduplicated). Tool deps
are added via `PrntData->NodeToolDeps.Add(tool)` and `prntDestSet.Add(tool)`
(`prntDestSet == NodeDeps`). `TUniqVector::Add` is a set insert, so adding the
same tool node id twice keeps one. The JSON `deps` array is rendered from
`NodeDeps`, hence deduplicated by node id.

The protoc **command inputs** are a separate thing (literal `--plugin=` input
files), where the binary legitimately recurs per plugin occurrence — upstream
does NOT dedup those, which is why our `inputs` field already matches byte-for-byte.

## Our mechanism / root cause

`emit_pb.go` `emitPB` builds `foreignDepRefs` as a plain `[]NodeRef`:

```
foreignDepRefs := depRefs(cppStyleguideLDRef, grpcCppLDRef, protocLDRef)
for _, plugin := range extraPlugins {
    foreignDepRefs = append(foreignDepRefs, depRefs(plugin.LDRef)...)
}
```

`ctx.tool` is memoized by tool path (`gen.go:3478`), so both YaFF plugins resolve
to the **same** `LDRef`; the loop appends it twice. `buildDeps` (node.go) and
`rewriteDeps` (dump_normalize.go) neither dedup, so the duplicate reaches `deps`.

## Fix

Mirror `TUniqVector` at the single collection site that can introduce a duplicate:
dedup `foreignDepRefs` (insertion order preserved, first occurrence kept) before
constructing the node. Command `inputs` are left untouched (they must keep the
per-plugin binary occurrence to stay byte-identical with upstream).

- `node.go`: add `dedupRefs([]NodeRef) []NodeRef` (TUniqVector analog, tiny slice,
  linear).
- `emit_pb.go`: wrap the assembled `foreignDepRefs` in `dedupRefs`.

No global hot-path change: `foreignDepRefs` is a handful of entries, deduped once
per PB node at emit; `buildDeps`/uid/json paths are untouched.

## Test (fails first)

`emit_pb_test.go`: a `PROTO_LIBRARY` with both `YAFF` and `YAFF_SCHEMA` (shared
plugin binary). Assert the PB node's `ForeignDepRefs` contains no duplicate
NodeRef (the shared yaff LD node appears exactly once). Fails on trunk (appears
twice), passes after the fix.

## Gate effect

sg7 affected `yabs/proto` PB roots stop appearing as deps/uid-only roots; sg7
`matched` rises, `our_only`/`ref_only` fall. sg2–sg6 unaffected (no shared-binary
multi-plugin protos there). No generation-time regression (O(few) dedup per PB).
