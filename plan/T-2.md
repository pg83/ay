# T-2 тАФ SPLIT_CODEGEN as a generated-output producer (SC node)

## Upstream mechanism (the thing we reproduce)

`SPLIT_CODEGEN(tool prefix opts... [OUT_NUM n] [OUTPUT_INCLUDES ...])` is a
Python plugin (`build/internal/plugins/split_codegen.py`) that lowers to the
`_SPLIT_CODEGEN_BASE` macro (`build/internal/conf/codegen.conf:13`):

```
.CMD=${tool:Tool} ${input;suf=.in:Prefix} ${hide;output:OUT} \
     ${noauto;output;suf=.cpp;nopath:Prefix} ${output;suf=.h;nopath:Prefix} \
     $OPTS ${hide;output_include:OUTPUT_INCLUDES} ${hide;kv:"p SC"} ${hide;kv:"pc yellow"}
```

- `OUT_NUM` defaults to `25` (`_DEFAULT_CPP_PARTS=20` + `_ADDITIONAL_STREAM_COUNT=5`).
- `OUT[]` = `prefix.0.cpp тАж prefix.{OUT_NUM-1}.cpp` тАФ hidden outputs, auto-compiled.
- `prefix.cpp` тАФ `noauto;output`: an output but NOT auto-compiled; the module
  re-feeds it explicitly via `SRCS(${BINDIR}/prefix.cpp)`.
- `prefix.h` тАФ `output`: generated header consumed via includes.
- `OPTS` = `--cpp-parts <OUT_NUM-5>` + positional opts.

Reference SC node (sg7, kernel/begemot_model_factors_info):
- cmd_args: `[toolbin, $(S)/<mod>/prefix.in, $(B)/<mod>/prefix.cpp, $(B)/<mod>/prefix.h, --cpp-parts, 20, <opts>]`
- inputs: `[toolbin, $(S)/<mod>/prefix.in]`  (no closure walk on the .in)
- outputs: `[prefix.0.cpp тАж prefix.24.cpp, prefix.cpp, prefix.h]`
- env: `ARCADIA_ROOT_DISTBUILD=$(SOURCE_ROOT)`; NO cwd
- kv `p=SC pc=yellow`; requirements `cpu=1 network=restricted ram=32`
- `foreign_deps.tool=[tool]`; `deps=[tool LD]` (= ForeignDepRefs only)
- target_properties: `module_dir`

The 25 numbered parts each compile to a CC node (`.pic.o`), and the
`SRCS(GLOBAL ${BINDIR}/prefix.cpp)` re-fed source compiles to its own GLOBAL CC
node. All CC nodes carry the SC producer dep (resolved generically through the
codegen registry тАФ the build-rooted source/header is a registered output).
