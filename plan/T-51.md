# T-51 — Acceptance harness robustness & diagnostics

## Symptom (from merger event log)
Repeated immediate `acceptance exit=-1` failures whose captured stdout contains
only the `OLD =/NEW =/work =` header and the two `=== <label>: validate.py … ===`
launch banners — no validator status, traceback, timeout line, or verdict reason.
The merger cannot tell *infrastructure failure* from *candidate-code rejection*.

This is acceptance/validate **orchestration** hardening, not a graph-parity ticket.
No special-casing of T-36, sg7, or any node class.

## Root causes (reproduced)
The `acceptance` orchestrator (`./acceptance OLD NEW`) runs each repo's
`dev/validate.py` in a worker thread and decides ACCEPT/REJECT from parsed
case lines. Three holes let a failure escape without an actionable stdout reason:

1. **validate.py return code is ignored.** `run_validate` returns
   `completed=True` for any non-timed-out run regardless of `proc.returncode`.
   A NEW validator that *crashes* (nonzero exit) emitting zero case lines parses
   to an empty status map; `common` is empty; `decide()` finds no reasons →
   **VERDICT: ACCEPT, exit 0** (false accept). Reproduced: NEW `dev/validate.py`
   = `sys.exit(7)` with OLD a normal `[sg2] OK` run → current acceptance ACCEPTs.

2. **Worker-thread / launch exceptions go to stderr, not stdout.** `Popen`
   (subprocess launch) is unguarded; a launch `OSError`, or any exception in
   `run_validate`, propagates out of `worker()` to the thread's default
   excepthook (**stderr**). `results[label]` is then never set, so `main()`'s
   `results["OLD"]`/`["NEW"]` raises `KeyError` (**stderr** traceback). The merger
   captures only acceptance **stdout** (via `log()`), so stdout retains just the
   header + the two banners — exactly the reported symptom.

3. **Missing worker results crash main instead of rejecting.** Same `KeyError`
   path: a side whose worker never recorded a result yields no diagnostic and a
   non-policy exit.

## Fix (generic, in `acceptance`)
Make every side's run produce a structured result, and require both sides to be
*usable* before comparing; otherwise REJECT with an actionable, stdout-logged
reason. Preserve the existing OK/XFAIL/parity policy for usable runs.

- Introduce `RunResult(label, completed, returncode, output, error)`.
- `run_validate` returns a `RunResult`:
  - missing validate.py → `error="…not found"`.
  - **wrap `Popen`** (and the stream/wait) in try/except → on failure
    `error="failed to launch/stream validate.py: …"`, logged via `log()`.
  - timeout → `error="TIMEOUT after Ns — killed"`.
  - clean end → `completed=True`, `returncode=proc.returncode`.
- `worker` wraps the call in try/except so **any** exception is recorded as a
  failed `RunResult` (with `traceback`) and logged via `log()`; `results[label]`
  is *always* set (in `finally`).
- `main` fetches results with a default failed `RunResult` (no `KeyError`).
- New per-side **usability** gate before `decide()`:
  - `error` set, or `returncode is None`, or `returncode not in {0,1}`
    (validate.py only ever returns 0/1; other codes / signal-negatives = crash),
    or **zero parsed case lines** (crashed before emitting any case) →
    side is unusable.
  - Any unusable side → log each side's reason + an output **tail** and REJECT
    (exit 1). This catches false-accept (#1), header-only launch/worker failures
    (#2), and missing results (#3), and puts the *which side & why* in stdout.
- Genuine all-SKIP runs (no data on host) stay usable: they exit 0 and emit
  `SKIP` case lines, so they keep the existing ACCEPT behavior.

## Tests (`dev/test_acceptance.py`, test-first)
Import `acceptance` by path (importlib). End-to-end via subprocess capturing
stdout/stderr **separately** to assert diagnostics land on **stdout**:
1. **Crashed NEW validator** (`sys.exit(7)`, no output) → exit ≠ 0 AND stdout
   names NEW + exit 7 / "no case results" (the false-accept repro; fails today).
2. **Launch/worker failure** → monkeypatch `subprocess.Popen` to raise `OSError`;
   assert `run_validate` returns a failed `RunResult` whose `error` carries the
   exception (no stderr-only escape); and an end-to-end run REJECTs with the
   reason on stdout, not a bare header.
3. **Normal success not regressed**: both sides `[sg2] OK` → ACCEPT, exit 0.
4. **Normal validate rejection not regressed**: OLD `[sg2] OK`, NEW `[sg2] FAIL`
   → REJECT "new FAIL appeared", exit 1.
5. **All-SKIP both sides** (exit 0, SKIP lines) → ACCEPT, exit 0 (unchanged).
