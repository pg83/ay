# T-51 plan: DESC_PROTO merge-node direct input closure (T-49 Split A)

## Upstream mechanism

Upstream-normalized DESC_PROTO submodule merge nodes (`<prj>.self.protodesc` /
`<prj>.protosrc`) carry, as reference-only direct inputs, the per-proto
descriptor producer's source/script closure in addition to the generated
`.desc`/`.rawproto` files and the merge node's own `merge_files.py` /
`collect_rawproto.py` scripts:

- `build/scripts/desc_rawproto_wrapper.py`
- every declared source `.proto`
- the parsed proto import closure (e.g. `google/protobuf/descriptor.proto` from
  protobuf / protobuf_old / protobuf_std)

The per-proto `.proto.desc` / `.rawproto` producer command inputs already match
the reference; this ticket only flattens the producer source/script closure onto
the merge node. It does NOT touch producer command include tokens (that is the
separate Split B ticket).

## Files touched

- `emit_proto_desc.go` — in `emitDescProtoSubmodule`, accumulate a deduped
  `producerSourceInputs []VFS` (`descRawprotoWrapperVFS`, the source proto VFS,
  and the `walkClosureTail` imports already computed per producer); thread it
  into `emitDescProtoMerge`, which appends it to the merge node `Node.Inputs`.
- `emit_proto_desc_test.go` — `TestEmitDescProto_MergeNodeFlattensProducerSourceInputs`:
  a builtin-style `PROTO_NAMESPACE(GLOBAL contrib/libs/protobuf/src)` + SRCDIR
  module whose source `type.proto` imports a non-source `any.proto`, asserting
  the merge node inputs include the wrapper, source proto, import-only proto,
  generated `.desc`/`.rawproto`, and merge/collect scripts.

## Expected gate effect

- The two T-49 `.self.protodesc` pair diffs stop reporting reference-only
  source/script inputs; sibling `.protosrc` nodes clear the same input class.
- Per-proto producer command include-token diffs remain (Split B).
- sg2–sg6 stay byte-exact; sg7 must not regress matched/our_only/ref_only.
