# T-76 plan: py-proto aux C++ warning-flag residual

## Observable gap

sg7 paired CC aux outputs for generated py-proto resources carry
`-Wimplicit-fallthrough` in our compile command, the normalized reference does
not. Representative pair:
`$(B)/ads/bsyeti/libs/events/proto/10f5421d07d840d1528565d43b_raw.auxcpp.py3.pic.o`.

## Upstream mechanism

`-Wimplicit-fallthrough` is not a compiler default. It enters via an
autoincluded `linters.make.inc` (AUTOINCLUDE_PATHS) gated on module language:

```
IF (MODULE_LANG == CPP)
    ... CLANG_WARNINGS(-Wimplicit-fallthrough) ...
ENDIF()
```

A `PROTO_LIBRARY` is a multimodule. Upstream evaluates the autoinclude per
submodule:

- `module _CPP_PROTO: ... SET(MODULE_LANG CPP)` — gate fires, generated `.pb.cc`
  receives `-Wimplicit-fallthrough` (matches ref).
- `module _PY3_PROTO: PY3_LIBRARY` and `PY3_LIBRARY` does `SET(MODULE_LANG PY3)`
  (build/conf/python.conf:907) — gate does NOT fire, so the optimized-py-proto
  resource aux C++ object carries no user CLANG_WARNINGS (matches ref).

The aux `.auxcpp` source compiles through `_SRC("auxcpp")` → `_SRC_cpp` →
`_CPP_ARGS_NEW`, which includes `$_USER_WARNINGS_VALUE` (= `_CLANG_USER_WARNINGS_VALUE`,
the early CLANG_WARNINGS slot). Because the `_PY3_PROTO` submodule's
`_CLANG_USER_WARNINGS_VALUE` is empty (MODULE_LANG==PY3 blocked the autoinclude),
that slot is empty in upstream's aux compile.

## Our defect

`collectStmts` (modules.go) binds `MODULE_LANG` for the autoinclude IF-evaluation
from `sbomComponentLang(d.moduleStmt.Name)`. For a `PROTO_LIBRARY` token that is
always `"CPP"`, regardless of which submodule (py vs cpp) is being collected.
`gen.go` re-collects the py side with `PY3_PROTO=true` in the env, but the
MODULE_LANG binding ignores that, so the py-side `d.clangWarnings` wrongly
inherits `-Wimplicit-fallthrough`. `emit_py_proto.go:emitPyProtoAuxChunks` then
threads `d.clangWarnings` into the aux CC compile, producing the divergent flag.

## Fix

Make the MODULE_LANG binding follow the selected submodule: for a
`PROTO_LIBRARY` collected with `PY3_PROTO` set in the env, bind `MODULE_LANG=PY3`
(mirroring `_PY3_PROTO: PY3_LIBRARY`). The CPP-proto path keeps `MODULE_LANG=CPP`.
One-line change in `collectStmts` at the existing binding site.

## Files touched

- `modules.go` — `collectStmts` MODULE_LANG binding.
- `emit_py_proto_warnings_test.go` — fail-first coverage (aux excludes the flag,
  `.pb.cc` keeps it).

- Representative `*_raw.auxcpp.py3.pic.o` pair loses the `-Wimplicit-fallthrough`

- CC by-token ours-only `-Wimplicit-fallthrough` bucket shrinks (~245).
- sg2–sg6 byte-exact, sg5 under budget; no normal `.pb.cc`/handwritten C++
