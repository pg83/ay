# T-1 — Model PROTO_DESCRIPTIONS / DESC_PROTO proto-description (PD) producers

## Upstream mechanism (build/conf/proto.conf @ /home/pg/monorepo/4)

PROTO_LIBRARY is a multimodule. Besides the CPP_PROTO submodule (the `.pb.cc`/PB
nodes we already emit) it has a **DESC_PROTO** submodule (`module _DESC_PROTO :
_BARE_UNIT`). The DESC_PROTO submodule is reached when a peer requests the
`DESC_PROTO` PEERDIR tag, which happens for **PROTO_DESCRIPTIONS** /
PROTO_REGISTRY modules (`SET(PEERDIR_TAGS DESC_PROTO DESC_PROTO_FROM_SCHEMA)`,
`.PEERDIR_POLICY=as_build_from`). In sg7 the trigger is
`ads/argus/proto/profile/generated/codegen` (a PROGRAM) doing
`BUNDLE(ads/argus/proto/profile/descriptions ...)` + `RESOURCE(...)`, and the
analogous grut path. There are exactly **2** PROTO_DESCRIPTIONS modules in the
bs_static closure; none in sg2..sg6 (verified: `grep -c desc_rawproto_wrapper`
is 0 for every other reference), so this change cannot perturb their byte-exact
gate.

Three node classes per upstream:

1. **Per-proto PD producer** (`_PROTO_DESC_RAWPROTO_CMD`, kv `p=PD`,
   `module_tag desc_proto`): one per `.proto` SRC. Runs
   `desc_rawproto_wrapper.py --desc-output X.proto.desc --rawproto-output
   X.proto.<md5(MODDIR)>.rawproto --proto-file <rootrel> -- protoc <-I set>
   --include_source_info`. Inputs: protoc binary, the proto source, the wrapper,
   and the `.proto` import closure. deps/foreign_deps.tool = [protoc].
   cwd=$(S). Outputs: `<proto>.desc` and `<proto>.<hash>.rawproto`.

2. **DESC_PROTO submodule merge** (`_PROTO_DESC_MERGE_CMD`, kv `p=PD`,
   tp{module_lang desc_proto, module_type lib, module_tag desc_proto}): merges
   the per-proto `.desc` into `<realprjname>.self.protodesc` (merge_files.py) and
   the `.rawproto` into `<realprjname>.protosrc` (collect_rawproto.py, cwd=$(B)).
   deps = the per-proto producers.

3. **PROTO_DESCRIPTIONS merge** (`_PROTO_DESC_MERGE_PEERS_CMD`, kv `p=PD`,
   tp{module_lang proto_descriptions, module_type lib}): merges peer
   `.self.protodesc` into `<realprjname>.protodesc` (merge_files.py) and
   `<realprjname>.tar` (merge_protosrc.py, cwd=$(B)). Primary output (MODULE_SUFFIX
   `.protodesc`) is what the BUNDLE moves. deps = peer DESC merge refs over the
   transitive DESC peer closure (incl. `contrib/libs/protobuf/builtin_proto/
   protos_from_protoc` injected by NEED_GOOGLE_PROTO_PEERDIRS, which pulls in
   protos_from_protobuf).

`_MODDIR_HASH = ${hash:MODDIR}` = lowercase md5 hex of MODDIR (verified:
md5("ads/argus/proto/events") == 5d7a48ee...). REALPRJNAME = last≤3 path
components joined by "-" (already computed by `archiveNameWithPrefix`).

NOTE (scope guard): the protoc `-I` set in the PD command derives from the
DESC submodule's own `_PROTO__INCLUDE` peer-addincl, which differs from the
CPP_PROTO submodule's. Per the ticket, "unrelated protoc include propagation
differences should remain separate"; sg7 is xfail (parity metrics, not
byte-exact). We reproduce the command SHAPE and the OUTPUT set (which is what
makes the PD class stop being reference-only — outputs become paired); we do
not chase byte-exact `-I` parity here.
