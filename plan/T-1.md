# Plan: T-1 — Fix ragel-native include scanner bleed-through

## Upstream mechanism identified

Upstream ymake's `TRagelIncludeProcessor::ProcessIncludes()` (in
`devtools/ymake/include_processors/ragel_processor.cpp`) treats ragel-native
`include "file.rl"` directives and C++ `#include` directives **separately**:

- **Native ragel includes** → `ResolveIncludes()` → added directly to the R6 node
  as hard deps. The scanner never recurses into these files' C headers.
- **C++ includes** → `ResolveAsUnset()` → stored as `ParsedIncls("h+cpp", ...)`
  for downstream consumption by the CC compilation node.

Our code puts **both** in `parsedIncludesLocal`. The scanner's `closureOf()`
then follows ragel-native includes to their target `.rl6` files, and from there
follows **those files' C includes** too — pulling `<numeric>`, `ymath.h`, `cfloat`,
`float.h`, and `__numeric/*` from `util/datetime/parser.rl6` into the R6 and CC
nodes for `httpfsm.rl6.cpp`.

The REF (upstream) has `parser.rl6` in both R6 and CC node inputs (correct), but
does NOT have `parser.rl6`'s C headers (correct).

## Effect on --by-token metric

- **inputs only in OURS**: `__numeric/*`, `numeric`, `cfloat`, `float.h`, `ymath.h`
  — all reachable from `parser.rl6`'s C includes.
- **inputs only in REF**: `__charconv/*`, `__chrono/*`, sanitizer headers — UNRELATED
  to this bug; these missing-in-ours tokens come from a SEPARATE mechanism (to be
  investigated if needed, but may already partly match).

## Files to change

1. **`parser_manager.go`** — add `parsedIncludesRagelNative` bucket constant.
2. **`parsers.go`** — `ragelIncludeDirectiveParser.Parse()`: move ragel-native
   includes from `parsedIncludesLocal` to `parsedIncludesRagelNative`.
3. **`scanner_test.go`**:
   - Update `TestParsedIncludes_RagelBuckets` (expects `parsedIncludesLocal` to
     have C includes only; ragel-native go to the new bucket).
   - Add `TestScanner_RagelNativeInclude_DoesNotBleedCHeaders` that creates two
     ragel files (main includes sub via native include, sub has C headers), walks
     main's closure, and asserts sub's C headers are absent.
4. **`emit_sources.go`** — After each `walkClosure` call for ragel (R6 and CC),
   additionally resolve and append the ragel-native included files using
   `parsedIncludesRagelNative`. This preserves the ragel files in the closure
   (dependency tracking) without dragging in their C headers.

## Expected metric change

For sg5: the "inputs only in OURS" cluster (`__numeric/*`, `numeric`, `cfloat`,
`float.h`, `ymath.h`) should reduce toward zero for the affected R6 and CC nodes.
The `--by-token` counts for those 4 ragel nodes should drop to 0.
Gating cases (sg2/sg2_x86_64/sg3/sg4) must remain byte-exact.
sg5 `matched=` must not decrease.
