# T-138 plan: propagate opaque FROM_SANDBOX producer script inputs through RUN_PROGRAM → ARCHIVE_ASM/RD

## Upstream mechanism

ymake uses a flat-input model: every node lists its full transitive *source*
closure. A `FROM_SANDBOX` fetch node (SB) carries exactly three explicit script
inputs on its command path (`fetch_from_sandbox.py`, `process_command_files.py`,
`fetch_from.py`). When a `RUN_PROGRAM` consumes the fetched `OUT_NOAUTO` files as
build-root `IN`, those files are opaque generated artifacts (no include parser),
but the producer's own source/script inputs are still real transitive inputs of
the consuming node — and onward to any node that consumes the RUN_PROGRAM output
(`ARCHIVE_ASM` → `.rodata` → RD compile). The data contents are never parsed as
includes.

Representative: `ads/clemmer/automorphology/uzb`:

```
FROM_SANDBOX(... OUT_NOAUTO trie|suffixes|paradigms)
RUN_PROGRAM(... morph2blob trie suffixes paradigms IN trie suffixes paradigms STDOUT ${BINDIR}/automorphpack.bin)
ARCHIVE_ASM(NAME UzbAutoMorphPack DONTCOMPRESS ${BINDIR}/automorphpack.bin)
```

`automorphpack.bin` (PR) and `UzbAutoMorphPack.rodata.asm` (RD) must each list the
three scripts; ay currently omits them.

## Existing machinery (reuse, do not duplicate)

- `GeneratedFileInfo.SourceInputs` already means "producer's $(S)-rooted leaf
  inputs, propagated to consumers". `CodegenRegistry.setSourceInputs` records it.
- `emitRunProgram` already folds each generated build-IN's `info.SourceInputs`
  into `prGeneratedFromSources` → `prSourceInputs` → recorded on each PR output
  (`setSourceInputs`) and added as output `ClosureLeaves`.
- `emitArchiveAsmNode` already reads each member's `info.SourceInputs` and
  propagates them as the `.rodata`'s `ClosureLeaves`; the RD compile walks that
  closure. (Proven by `TestEmitArchiveAsm_RunPythonOutThroughRodata`, where the
  RD node carries the RUN_PYTHON's `unzip.py`/`in.bin.gz`.)

The only gaps:
1. `emit_from_sandbox.go` never sets `SourceInputs` on the fetched outputs, so
   the three scripts never enter `prGeneratedFromSources`. → RD path stays empty.
2. `prInputClosure` walks an IN's closure only when the IN has a registered
   include parser. An opaque fetched IN has none, so its producer's source
   inputs never reach the **PR node's own** input closure.

## Changes

1. `emit_from_sandbox.go`: after registering each fetched output, record
   `fromSandboxScriptInputs` as that output's `SourceInputs`. The SB node's own
   `Inputs` list stays the three scripts (unchanged) and its command tokens are
   untouched.

2. `emit_pr.go` `prInputClosure`: in the IN loop, for an IN with no registered
   parser that resolves to a registered generated output, append that producer's
   `SourceInputs` to the closure (no parse of the fetched data). The trailing
   `dedupVFS` collapses the per-IN repetition.

This routes scripts to:
- the PR node `automorphpack.bin` via `inputClosure` (change 2);
- the RD node via the existing `SourceInputs → ClosureLeaves` chain (change 1).

## Test (fail-first)

Extend `emit_from_sandbox_test.go`: multiple `FROM_SANDBOX OUT_NOAUTO` outputs fed
to a `RUN_PROGRAM(... IN ... STDOUT pack.bin)` then `ARCHIVE_ASM`. Assert the PR
node and the RD node each list the three scripts, and the SB node still carries
exactly the three scripts (unchanged). Use `testGenX86` (rodata is x86_64-only).

## Expected gate effect

- `automorphpack.bin` and `UzbAutoMorphPack.rodata.asm` pairs print only headers.
- PR input drift −1, RD input drift −1; sg7 matched does not decrease.
- SB own inputs / command tokens unchanged; `AR inputs`/`LD inputs` stay 0.
- sg2–sg6 byte-exact, sg5 under budget.
