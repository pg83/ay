# T-50 plan: model YMAPS_SPROTO .sproto.h producers + sproto runtime propagation

## Upstream mechanism (verified)

`build/internal/conf/project_specific/maps/sproto.conf`:

```
macro _YMAPS_GENERATE_SPROTO_HEADER(File) {
    .CMD=${cwd;rootdir;input:File} ${tool:"maps/libs/sproto/sprotoc"} \
        -I=./$PROTO_NAMESPACE -I=$ARCADIA_ROOT/$PROTO_NAMESPACE -I=$ARCADIA_BUILD_ROOT \
        -I=$PROTOBUF_INCLUDE_PATH --sproto_out=$ARCADIA_BUILD_ROOT/$PROTO_NAMESPACE \
        ${rootrel;input:File} ${hide;norel;output;suf=.sproto.h;nopath;noext:File} \
        ${hide;kv:"p PB"} ${hide;kv:"pc yellow"}
    .PEERDIR=maps/libs/sproto
}
macro YMAPS_SPROTO(FILES...) {
    SET(PROTO_HEADER_EXTS .pb.h .sproto.h)
    foreach (FILE : $FILES) { _YMAPS_SPROTO_DISPATCH(${lastext:FILE} $FILE) }
}
```

`maps/libs/sproto/sprotoc/ya.make` is a `PROGRAM()` PEERDIRing `contrib/libs/protoc`
+ `maps/libs/sproto`, with `INDUCED_DEPS(h+cpp ${ARCADIA_ROOT}/maps/libs/sproto/include/sproto.h)`.

## Mechanisms reused (no new data paths)

1. Typed macro state in `ModuleData` (like `protoNamespace`): `ymapsSprotoSrcs []STR`.
2. `ctx.toolResult(argMapsLibsSprotoSprotoc)` builds the sprotoc PROGRAM + its
   `maps/libs/sproto` LIBRARY peer (sprotoc/lib/AR nodes appear for free), exposes
   `res.LDRef`/`res.LDPath`.
3. PB producer node construction mirrors `emitProtoPB`'s `.pb.h` producer (KV{P,PC},
   ForeignDepRefs, Inputs via na.inputList, walkClosureTail).
4. `registerBoundGeneratedParsedOutput(..., generatorRefs=[sprotocLDRef])` → the existing
   `scanner.resolveInducedDeps` mixes sprotoc's INDUCED_DEPS (sproto.h + closure) into
   every consumer that includes the generated `.sproto.h`. No hand-woven runtime lists.
5. PROTO_HEADER_EXTS `.sproto.h` half: `protoDirectPbHIncludes` already turns a proto's
   imports into `.pb.h` directives; for a YMAPS_SPROTO module, additionally emit the
   `.sproto.h` sibling for each import whose `.sproto.h` is module-locally produced. Done
   in the per-call transform (NOT the content-cached `ProtoIncludeDirectiveParser`), so
   resolution caching stays context-independent.
