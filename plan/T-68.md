# T-68 plan: implement `CPP_ENUMS_SERIALIZATION`

## Upstream mechanism

`build/plugins/pybuild.py:824-832`:

```python
@ymake.macro
def CPP_ENUMS_SERIALIZATION(unit, *args):
    args = iter(args)
    for arg in args:
        if arg == 'NAMESPACE':
            next(args)          # consume namespace value, emit nothing
        else:
            unit.ongenerate_enum_serialization_with_header(arg)
```

So `CPP_ENUMS_SERIALIZATION(...)` is exactly one
`GENERATE_ENUM_SERIALIZATION_WITH_HEADER(arg)` per non-`NAMESPACE` argument;
a `NAMESPACE` token consumes its following value as a control token and emits
nothing. `GENERATE_ENUM_SERIALIZATION_WITH_HEADER`
(`ymake.core.conf:4546-4550`) emits the EN producer with `--header`
(`File_serialized.cpp` + `File_serialized.h`) and
`PEERDIR(tools/enum_parser/enum_serialization_runtime)`.

The local model already implements `GENERATE_ENUM_SERIALIZATION_WITH_HEADER`
fully (`GenerateEnumSerializationStmt{Variant:"with_header"}`, handled in
`modules.go` for the peer and `emit_en.go` for the EN/CC/AR path). This ticket
only needs to route `CPP_ENUMS_SERIALIZATION` into that existing statement.
