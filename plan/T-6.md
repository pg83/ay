# T-6 — Generated `.proto` consumed by the Python protobuf emitter

## Upstream mechanism

A `SRCS(X.proto)` may name a `.proto` that is **not** an on-disk source: an
earlier `IF(GEN_PROTO)` step (`RUN_PROGRAM`/`RUN_ANTLR` STDOUT, e.g.
`irt/bannerland/proto/banner_flags/banner_flags.proto`) produces it into the
build tree. Upstream ymake wires protobuf generation for such files through the
**build output** and the **generating node**: `protoc` reads the generated
artifact from `$(B)`, the PB node takes a dependency on the producer, and the
producer's transitive `$(S)` leaf sources ride on the PB node's flat input list.

This mechanism is already reproduced for the **C++** protobuf emitter
(`emit_proto.go` / `emitPB`): it looks the proto up in the codegen registry, and
when found swaps the proto input to `build(protoRelPath)`, pins
`info.ProducerRef` as a `DepRefs` entry, folds in `info.SourceInputs`, and runs
`protoc` from `$(B)` instead of `$(S)`.

The **Python** protobuf emitter (`emitPyProtoSrc`) does not do this. It
unconditionally lists `source(protoRelPath)` as the proto input and runs from
`$(S)`. For a build-generated proto that `$(S)` path does not exist, so
sandboxing content-hashing faults opening it, and the producer node is
unreachable from the PB node (no dep edge), risking DFS-pruning at finalize.

The shared codegen registry is keyed by **platform**, not language
(`scannerForPlatform`). In `emitPyProtoSrcs` the C++ sibling is generated first
(`genModule(cppInstance)`), which registers the JV producer of the generated
proto into the shared target-platform registry; by the time the Python source
loop runs `emitPyProtoSrc`, the lookup of `build(protoRelPath)` resolves.

## Divergence (reproduced)

For a `PROTO_LIBRARY` whose `SRCS(JsonPathParser.proto)` is the `OUT` of a
`RUN_ANTLR` (no source `.proto`), consumed by a `PY3_LIBRARY`:

- C++ PB node: input `$(B)/.../JsonPathParser.proto`, producer source inputs
  present (grammar, stg, configure_file.py, stdout2stderr.py, antlr.jar).
- Python PB node: input `$(S)/.../JsonPathParser.proto` (nonexistent), no
  producer dep, no producer source inputs.

## Files touched

- `emit_py_proto.go` — in `emitPyProtoSrc`, mirror the C++ override: look up
  `build(protoRelPath)` in the codegen registry; when present swap the proto
  input VFS to the `$(B)` build path, set the protoc `Cwd` to `$(B)`, pin
  `info.ProducerRef` on `Node.DepRefs`, and append `info.SourceInputs` to the
  node inputs. Source `.proto`s keep `$(S)` + no extra dep (unchanged).
- `emit_proto_test.go` — regression: a generated `.proto` feeds Python protobuf
  generation; assert the py PB node uses the build-tree proto input, the
  producer dependency, and the producer source inputs.
