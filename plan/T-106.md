# T-106 plan: aggregate duplicate-output exact-counterpart cancellation (diagnostics only)

## Problem (reproduced)

T-97 made `ay dev dump diff --pair` reliable for duplicate-output nodes by cancelling
producers that have an exact full-node counterpart on the other side
(`stripContentEqualPairs` in `findNodePairByOutput`). The aggregate reports
(`--by-field`, `--by-kind`, `--by-token`, `--roots`) were left on the old streaming
"first record wins per (output, match-key)" pairing, so they still cross-pair our
mode-A producer against ref's mode-B producer at a shared output and report a stale
command/field delta that the direct `--pair` probe shows does not exist.

Reproduction against the available sg7 artifacts
(`.out/validate/sg7.{our,ref}.norm.jsonl`):

```
--by-kind:
  AS   paired=467  diverge=41  cmds:41   [top combo: cmds ×41]
  R6   paired=28   diverge=15  cmds:14
```

while the direct probes are header-only:

```
--pair '$(B)/contrib/libs/libunwind/_/src/UnwindRegistersRestore.S.o'   # header only
--pair '$(B)/balancer/kernel/cookie/expires.rl6.cpp'                    # header only
--pair '$(B)/contrib/libs/libmysql_r/strings/uca900_ja_tbls.cc'         # header only
--pair '$(B)/contrib/ydb/library/actors/protos/actors.pb.h'            # header only
```

The 41 AS "divergences" are the wrong-sibling cross-pairings: each AS output is
produced by debug/release (or PIC/non-PIC) sibling nodes whose only difference is a
command-mode the match key does not capture, so the streaming lookup pairs our
sibling against ref's other sibling even though exact counterparts exist on both
sides.

## Upstream mechanism / principle

This is not a graph-generation defect — upstream emits both duplicate-output
siblings; our graph already matches (the `--pair` probe is header-only, and exact
normalized-node parity counts these as matched). The defect is purely in the
diagnostic tooling's pairing heuristic. The correct principle, already established by
T-97 for `--pair`, is exact-counterpart cancellation: an output produced by several
sibling nodes per side has parity iff the two producer multisets are equal; a
per-field/per-token/per-root delta is only meaningful for producers left without an
exact full-content counterpart on the other side.

The aggregate reports must apply the same cancellation before counting any delta.

## Design

Add one shared, streaming canceler that mirrors `stripContentEqualPairs` for the
aggregate path without buffering nodes:

- `dumpDiffCanceler` is built from two cheap content-hash multisets — one per side —
  keyed by the full node content hash (`nodeFieldHashes(n)`, the same 8/10-field hash
  used everywhere else). For each content hash present on both sides it records
  `min(leftCount, rightCount)` cancellable pairs, held as two independently consumed
  budgets (`budgetLeft`, `budgetRight`).
- `cancelRight(n)` / `cancelLeft(n)` greedily consume one unit of the relevant budget
  when the node's content hash still has cancellation budget, returning whether the
  node was cancelled. File order is deterministic, so the residual (uncancelled)
  producers on each side are well-defined and pair residual-to-residual.

Each aggregate report wires the canceler at its two existing stream points:

- Right builder: `if canc.cancelRight(n) { return }` — cancelled right producers are
  excluded from the lookup maps, so a residual left producer can only match a residual
  right producer (no cross-pairing against a cancelled sibling).
- Left consumer: cancel check first. A cancelled left producer is, by definition,
  paired with an exact counterpart and carries zero delta, so it is counted in the
  paired/total denominator but contributes no field/token/kind/root difference.

This keeps the content-equal majority (sg7 `matched≈66675`) counted as paired with
zero delta — only the wrong-sibling cross-pairings stop being counted as drift.

Files touched: `dump_diff.go` (canceler + 4 wiring sites), `dump_test.go` (fixtures).
No change to validate normalization, node hashing, emitters, commands, inputs,
outputs, target properties, or the `diffSections`/`--summary` path that `validate.py`
runs. Graph parity is expected to be unchanged.

   produced by `{-CG2, -CT0}` sibling pair on both sides. Assert `--by-field`,

   only-in-ref, and `--by-field`/`--by-kind` still count one `cmds` divergence.
