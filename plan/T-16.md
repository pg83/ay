# T-16 ‚Äî YAFF / YAFF_SCHEMA as generic C++ protobuf plugin contributions

## Upstream mechanism

`build/internal/conf/project_specific/yabs.conf` defines both macros as thin
wrappers over `CPP_PROTO_PLUGIN2` (build/conf/proto.conf):

```
YAFF_PLUGIN_PATH=library/cpp/yaff/tools/protoc_plugin

macro YAFF(NAMESPACE="", FILES[], EXPERIMENTAL[]) {
    CPP_PROTO_PLUGIN2(yaff $YAFF_PLUGIN_PATH .yaff.h .yaff.cpp
        EXTRA_OUT_FLAG namespace=$NAMESPACE,${join=,;pre=file=:FILES},${join=,;pre=experimental=:EXPERIMENTAL})
}

macro YAFF_SCHEMA(SCHEMA_NAME, NAMESPACE, FILES[], EXPERIMENTAL[]) {
    CPP_PROTO_PLUGIN2(yaff_$SCHEMA_NAME $YAFF_PLUGIN_PATH _$SCHEMA_NAME.yaff.h _$SCHEMA_NAME.yaff.cpp
        EXTRA_OUT_FLAG tag=$SCHEMA_NAME,namespace=$NAMESPACE,${join=,;pre=file=:FILES},${join=,;pre=experimental=:EXPERIMENTAL})
}
```

`CPP_PROTO_PLUGIN2(Name Tool Suf1 Suf2 ... EXTRA_OUT_FLAG F)` registers a C++
protoc plugin whose command block (`_PROTO_PLUGIN_ARGS_BASE`) is:

```
--plugin=protoc-gen-${Name}=${tool:Tool} --${Name}_out=$ARCADIA_BUILD_ROOT/$PROTO_NAMESPACE ${pre=--${Name}_opt=:OutParm}
```

`${pre=--${Name}_opt=:OutParm}` prefixes `--${Name}_opt=` to **each element** of
`OutParm`. `OutParm` is the `EXTRA_OUT_FLAG` scalar interpolated as macro args,
so its **commas are argument separators**: the flag is split on `,`, empty
pieces drop out, and each surviving piece becomes its own `--${Name}_opt=<piece>`.

Reference (sg7, `yabs/proto/frequency_events.pb.cc`), `YAFF(FILES ‚Ä¶)` +
`YAFF_SCHEMA(tsar_vectors NUserProfileTsarVectors)`:

```
--plugin=protoc-gen-yaff=$(B)/library/cpp/yaff/tools/protoc_plugin/protoc_plugin
--yaff_out=$(B)/
--yaff_opt=namespace=
--yaff_opt=file=cbir_user_history.proto ‚Ä¶ --yaff_opt=file=user_profile.proto
--plugin=protoc-gen-yaff_tsar_vectors=$(B)/library/cpp/yaff/tools/protoc_plugin/protoc_plugin
--yaff_tsar_vectors_out=$(B)/
--yaff_tsar_vectors_opt=tag=tsar_vectors
--yaff_tsar_vectors_opt=namespace=NUserProfileTsarVectors
```

Outputs are added per every proto in the module (`_ADD_CPP_PROTO_OUT` appends
the suffix for each `File`), e.g. `frequency_events.yaff.{h,cpp}` and
`frequency_events_tsar_vectors.yaff.{h,cpp}`. The plugin binary
(`library/cpp/yaff/tools/protoc_plugin/protoc_plugin`) is a tool input.

## Our existing machinery (reuse, don't reinvent)

`CppProtoPlugin{Name, ToolPath, OutputSuffixes, Deps, ExtraOutFlag}` already
models exactly this. `modules.go` parses `CPP_PROTO_PLUGIN{0,,2}` into
`d.cppProtoPlugins`; `emit_proto.go`/`emit_pb.go` resolve each to a tool input +
foreign dep, add per-proto suffix outputs, and emit the plugin command block in
`composePBArgBlocks`. So YAFF/YAFF_SCHEMA only need to *produce a CppProtoPlugin*.

## Defect found in the existing emission

`composePBArgBlocks` currently emits `--${Name}_opt=:<whole ExtraOutFlag>` ‚Äî a
single token with a spurious leading colon and no comma split. No gating
reference or unit test exercises EXTRA_OUT_FLAG today (sg4/sg5 validation
plugins carry none), so the defect is latent. Upstream splits on comma, drops
empties, no colon. Fix it generically so both the new YAFF path and any future
EXTRA_OUT_FLAG plugin match upstream.

## Files touched

1. `gen.go` ‚Äî drop `YAFF` / `YAFF_SCHEMA` from `acknowledgedMacros` (they now
   contribute typed nodes).
2. `str_consts.go` ‚Äî add `kwFILES`, `kwEXPERIMENTAL` interned keywords.
3. `modules.go` ‚Äî add `tokYaff` / `tokYaffSchema` cases that parse the macro
   args (NAMESPACE/FILES/EXPERIMENTAL; YAFF_SCHEMA: positional SCHEMA_NAME,
   NAMESPACE) and build the equivalent `CppProtoPlugin`, appending to
   `d.cppProtoPlugins`. Build `ExtraOutFlag` exactly as the upstream macro
   composes it (literal commas between the namespace/file/experimental groups).
4. `emit_pb.go` ‚Äî `composePBArgBlocks`: split `ExtraOutFlag` on `,`, drop empty
   pieces, emit one `--${Name}_opt=<piece>` per surviving piece (no colon).
