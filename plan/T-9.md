# T-9 plan: transitive PROTO_NAMESPACE proto-include propagation into CPP_PROTO protoc commands

## Upstream mechanism

`build/conf/proto.conf`:

```
macro PROTO_ADDINCL(GLOBAL?"GLOBAL":"", Path, WITH_GEN?"BUILD":"") {
    _ORDER_ADDINCL($WITH_GEN $GLOBAL FOR proto ${ARCADIA_BUILD_ROOT}/$Path
                   SOURCE $GLOBAL FOR proto ${ARCADIA_ROOT}/$Path)
    ADDINCL($GLOBAL ${ARCADIA_BUILD_ROOT}/$Path)
}
macro PROTO_NAMESPACE(GLOBAL?"GLOBAL":"", WITH_GEN?"WITH_GEN":"", Namespace) {
    SET(PROTO_NAMESPACE $Namespace)
    PROTO_ADDINCL(GLOBAL $WITH_GEN $Namespace)
}
```

`PROTO_NAMESPACE(yt)` (no WITH_GEN) expands to a `GLOBAL FOR proto
${ARCADIA_ROOT}/yt` addincl (the SOURCE arm of `_ORDER_ADDINCL`; the BUILD arm
is gated on WITH_GEN and stays empty). That `GLOBAL FOR proto` addincl is added
to the declaring module's proto IncDirs AND propagated through its CPP_PROTO peer
closure (PROTO_LIBRARY's CPP_PROTO module sets `PEERDIR_TAGS CPP_PROTO`). It
therefore lands in `_PROTO__INCLUDE` of every transitive CPP_PROTO consumer and
renders as `-I=$(S)/yt` in the protoc command via `${pre=-I=:_PROTO__INCLUDE}`
(`_CPP_PROTO_CMDLINE_BASE`).

There is NO suppression for CPP_PROTO / PROTO_LIBRARY consumers. Reference sg7:
337 cpp_proto pb.h modules carry `-I=$(S)/yt`, including
`ads/autobudget/protos/brandformance.pb.h` (`module_tag=cpp_proto`).

## Our defect

`emit_pb.go:composePBArgBlocks` appends the collected `protoNamespaceTail`
(non-GLOBAL PROTO_NAMESPACE contributions, already collected transitively in
gen.go through `ProtoNamespaceTail`) only under `if moduleTag == 0`. That gate
suppresses the tail for cpp_proto modules, so transitive PROTO_LIBRARY consumers
never receive `-I=$(S)/yt`. The `moduleTag == 0` gate rested on an observation
from the narrow sg2–sg5 contours (commit 8b03291) that the broader sg7 reference
disproves.

The own declaring module already receives `-I=$(S)/yt` via its `cppOutRoot`
front block (`-I=./yt -I=$(S)/yt`), so the tail is only needed for downstream
consumers — exactly the path the gate was blocking.

## Change

- Remove the `moduleTag == 0` gate in `composePBArgBlocks`: emit the
  `protoNamespaceTail` block for all proto command lines. The tail still trails
  the `peerProtoAddIncl` chain (correct `-I` order: yt after protobuf-src).
- The tail is collected/deduped against the chain at module level already; keep
  that. No new transport, no AddInclGlobal abuse — the existing
  ProtoNamespaceTail machinery is the carrier.

Files touched: `emit_pb.go` (drop gate + `moduleTag` param if it becomes unused),
`emit_pb_test.go` (regression).

## Expected gate effect

- sg7: `ads/autobudget/protos/brandformance.pb.h` pair no longer reports
  `-I=$(S)/yt` as reference-only; cpp_proto modules reaching yt rise from 33
  toward 337.
- sg2–sg6 stay byte-exact (those contours either have no cpp_proto module
  reaching a bare PROTO_NAMESPACE through a CPP_PROTO chain, or their reference
  carries the same token — confirmed by validate.py).
