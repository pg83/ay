# T-25 plan: make_morphdict tool-subtree full-uid identity for rus.extra.dict.bin

## Reproduced divergence

After T-19, `$(B)/kernel/lemmer/new_dict/rus/extra/rus.extra.dict.bin` (PR node)
matches the reference at `self_uid` (content) level, but its **full Merkle `uid`
still differs**. The only differing normalized field is `deps`: our PR node
depends on the make_morphdict tool node `DJotZPtq…`, ref on `q-nngPsJ…` — the
same tool, distinct full uid.

Walking the make_morphdict tool subtree (matched by output path, not array
order) the divergence bottoms out in exactly **four content-diff leaf nodes**,
which are **three independent generic mechanisms** — all in modules that are
reachable ONLY through the make_morphdict tool closure in sg7, so the bugs are
invisible elsewhere:

1. `$(B)/library/cpp/build_info/build_info.cpp` (CONFIGURE_FILE)
   - ours: `configure_file.py … BUILD_TYPE=DEBUG`
   - ref : a **second** node with `BUILD_TYPE=RELEASE` (the tool/host build).
     Ref has BOTH a DEBUG (target) and a RELEASE (tool) build_info node; we emit
     only DEBUG because `buildCFGVars` hardcodes `BUILD_TYPE=DEBUG`.

2. `$(B)/library/cpp/tokenizer/nlptok_v2.rl6.cpp` and `…nlptok_v3.rl6.cpp` (R6)
   - ours: `ragel6 '-L -G2' -L …`  (RAGEL6_FLAGS as ONE joined arg)
   - ref : `ragel6 -L -G2 -L …`     (two separate argv tokens)
   - `tokenizer/ya.make` does `SET(RAGEL6_FLAGS -L -G2)`; we store the value as a
     single `ARG("-L -G2")`.

3. `$(B)/kernel/lemmer/context/default_decimator/factory.cpp.pic.o` (CC)
   - ref has input `$(S)/kernel/lemmer/context/default_decimator/default_decimator.lst`;
     we don't. `default_decimator/ya.make` does
     `ARCHIVE(NAME data.inc default_decimator.lst)` and `factory.cpp` #includes
     the generated `data.inc`. Plain `ARCHIVE` does not propagate its archived
     source members into the consumer's include closure.

`ara/builtin` uses RUN_PYTHON3(unzip.py), not make_morphdict, so it is
unaffected by all three.

## Upstream mechanisms

1. **CONFIGURE_FILE BUILD_TYPE = platform build type.** ymake substitutes the
   `BUILD_TYPE` template var from the per-platform build configuration
   (`GG_BUILD_TYPE`). Tools build on the host platform, which defaults to
   `release` (our `make.go:144` already sets `hostFlags["GG_BUILD_TYPE"]="release"`,
   and `Platform.BuildTypeUpperSTR` is the interned uppercase form used at
   `modules.go:3247`). The hardcoded `DEBUG` fallback is a shortcut that is only
   correct for a debug target instance; the host/tool instance must yield
   `RELEASE`, producing a distinct configure node — exactly what ref has.

2. **`$RAGEL6_FLAGS` expands as separate command args.** The RAGEL6 macro
   (`build/ymake.core.conf:3305`) is `… ${tool:ragel6} $RAGEL6_FLAGS ${SRCFLAGS}
   -L -I${ARCADIA_ROOT} …`. A `SET(VAR a b)` value expands in command context as
   the token sequence `a b` (split on whitespace), not one quoted blob.

3. **ARCHIVE archived source members ride into the consumer closure.** A module
   that `#include`s a generated archive header (`data.inc`) lists the archived
   member files as compile inputs (ref shows `default_decimator.lst` on
   `factory.cpp.pic.o`). Our model already does this for `ARCHIVE_BY_KEYS` and
   LJ's `LuaSources.inc` via `ArchiveEntry.PropagateSourceMembers`; plain
   `ARCHIVE` simply fails to set the flag. The fix unifies the two paths.

## Files to touch

- `emit_cf.go`: `buildCFGVars` takes the platform build-type-upper string;
  the `BUILD_TYPE` fallback uses it instead of the `buildTypeDebug` constant.
  `emitConfigureFile` passes `instance.Platform.BuildTypeUpperSTR.string()`.
- `modules.go`: `RAGEL6_FLAGS` SET handling splits the value into one `ARG` per
  whitespace token; `applyArchiveStmt` sets `entry.PropagateSourceMembers = true`.

Net: removes the `buildTypeDebug` special constant, unifies ARCHIVE with
ARCHIVE_BY_KEYS. Small, generic, no path allowlist.

## Test-first

- `emit_cf_test.go`: a CONFIGURE_FILE template referencing `@BUILD_TYPE@` built
  under a release platform must emit `BUILD_TYPE=RELEASE` (fails today: DEBUG).
- `emit_r6_test.go` (or modules test): `SET(RAGEL6_FLAGS -L -G2)` yields the
  ragel command tokens `… -L -G2 -L …` as separate args (fails today: `-L -G2`
  joined).
- Archive/CC closure test: a plain `ARCHIVE(NAME data.inc x.lst)` consumed by a
  C++ source that `#include "data.inc"` lists `$(S)/…/x.lst` among the compile
  inputs (fails today: absent).
- Integration regression: a `RUN_PROGRAM` consuming a built tool whose subtree
  contains the above, asserting the generated output's downstream ARCHIVE_ASM
  chain carries the tool identity and the output is declared once (guards T-19).

## Note to lead/reviewer

The ticket assumed a single tool-subtree cause; reconnaissance found three
independent generic mechanisms, all confined to the make_morphdict-only subtree.
All three are required for the chain full-uid to match and none is a path-specific
hack, so they are implemented together under this ticket.
