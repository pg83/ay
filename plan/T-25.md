# T-25 plan: bind MODULE_LANG so autoincluded CLANG_WARNINGS participate

## Upstream mechanism (per plan/T-24.md, verified against /home/pg/monorepo/4)

Upstream autoincludes the nearest enclosing `ads/linters.make.inc` for `ads/bsyeti`
modules (`build/internal/conf/autoincludes.json` lists `ads`). That include runs
at module finalization and contains:

```
IF (MODULE_LANG == CPP)
    IF (MODDIR STARTS_WITH ads/bsyeti OR ...)
        CLANG_WARNINGS(-Wimplicit-fallthrough)
    ENDIF()
ENDIF()
```

`CLANG_WARNINGS` (`build/ymake.core.conf`) does `SET_APPEND(_CLANG_USER_WARNINGS_VALUE ...)`;
the value lands in the `_USER_WARNINGS_VALUE` slot of `_CPP_ARGS_NEW`
(`build/conf/compilers/gnu_compiler.conf`) — after include args / `GCC_COMPILE_FLAGS`,
before `$CXXFLAGS`. So the reference command carries `-Wimplicit-fallthrough` twice:
once early (CLANG_WARNINGS), once in the later CXXFLAGS tail (from
`ads/bsyeti/ya_cpp.make_cxxflags.inc`).

`MODULE_LANG` is the upstream module-language symbol bound during module
evaluation (CPP for C++ module types, PY3 for python). ay does not bind it, so
`evalAtom` treats the bare `MODULE_LANG` ident as an implicit build-var string
`"MODULE_LANG"`, the `== CPP` comparison is false, the linter branch is skipped,
and `d.clangWarnings` stays empty.

## Files to touch

1. `env_consts.go` — add `envMODULE_LANG = internEnv("MODULE_LANG")`.
2. `modules.go` — in `collectStmts`'s `case *ModuleStmt`, after
   `d.moduleStmt = moduleStmtForKind(v, kind)`, bind
   `env.setString(envMODULE_LANG, sbomComponentLang(d.moduleStmt.Name))`.
   `sbomComponentLang` (emit_sbom.go) already maps a module TOK to the uppercase
   MODULE_LANG token (PY3 / CPP / AGNOSTIC) — reuse it (DRY). Timing is correct:
   the module statement is the first statement, the autoincluded linters.make.inc
   is appended after the module body (`moduleStmts` in gen.go), so MODULE_LANG is
   bound before `IF (MODULE_LANG == CPP)` is evaluated.

No change to emit_cc.go: `ModuleCCInputs.ClangWarnings` -> `composeCCModuleArgBlocks`
already places the value in the `_USER_WARNINGS_VALUE` slot.
