`/home/pg/monorepo/yatool/build/plugins/pybuild.py:580-583`:

```

```

`SrcX = rootrel_arc_src(path, unit) + '-'`, where `path` is the raw `PY_SRCS`
token (`__init__.py`). `rootrel_arc_src`
(`/home/pg/monorepo/yatool/build/plugins/_common.py:115`):

```
def rootrel_arc_src(src, unit):
    if src.startswith('${ARCADIA_ROOT}/'): return src[16:]
    if src.startswith('${ARCADIA_BUILD_ROOT}/'): return src[22:]
    elif src.startswith('${CURDIR}/'): return unit.path()[3:] + '/' + src[10:]
    else:
        resolved = unit.resolve_arc_path(src)
        if resolved.startswith('$S/'): return resolved[3:]
        return src  # leave as is
```

For an **ordinary** source, `resolve_arc_path('__init__.py')` →
`$S/<moddir>/__init__.py` → returns `<moddir>/__init__.py`. For a **generated**
`OUT_NOAUTO __init__.py`, the file resolves into the build root (`$B/...`), not
`$S/...`, so `rootrel_arc_src` falls through to `return src` and yields the raw
token `__init__.py`. Hence `SrcX = __init__.py-`, not `<moddir>/__init__.py-`.

`${input:Src}` declares the generated `__init__.py` as the node's input edge.
Because `Src` is build-generated, ymake's flat-input model resolves it to its
producer (the `RUN_PROGRAM`) and the bytecode node's flat input closure carries
the producer's whole closure — the `IN_NOPARSE` `.pb.h` headers and their
transitive `$(S)` proto / libcxx / protobuf / abseil includes.

## Our divergence

`emit_py_codegen.go:53-59`:

```go
srcAbs := resolveSourceVFS(ctx, instance, srcRel.string(), d.srcDirs)
if genInfo != nil {
    srcAbs = build(instance.Path.rel() + "/" + srcRel.string())
}
moduleName := srcAbs.rel() + "-"
```

For a generated source `srcAbs` is the `$(B)` path, so `srcAbs.rel()` =
`<moddir>/<token>` and `moduleName` becomes the full module path — the first-arg
drift. Upstream uses the raw token for generated sources.

For the input closure, `emit_py_codegen.go:49-51` attaches only
`genInfo.SourceInputs` — the producer's **direct** `$(S)` leaf inputs
(`reg.setSourceInputs(out, prSourceInputs)` in `emit_pr.go:180`). That set omits
the transitive include closure of the `IN_NOPARSE` headers (libcxx/protobuf/
abseil/util) that the `RUN_PROGRAM` producer node itself carries via
`prInputClosure` (`emit_pr.go:295`).

## Change

`emit_py_codegen.go`:

1. Generated-source command name: when `genInfo != nil`, set
   `moduleName = srcRel.string() + "-"` (the raw token), matching
   `rootrel_arc_src`'s `return src` branch. Ordinary sources keep
   `srcAbs.rel() + "-"`.

2. Producer-input closure: a bytecode node compiling a generated source must
   carry the producer's full input closure, not just its direct `$(S)` leaves.
   Walk the generated source's closure through the existing scanner
   (`generatedOutputClosure` / `walkClosureTail`) so the registered producer
   closure rides into the node inputs — reusing the codegen-registry + scanner
   path, no new side channel.

   The producer (`emitRunProgram`) already computes its full input closure
   (`prInputClosure`); the gap is that only the direct-source subset is stored on
   the `OUT_NOAUTO` registry entry. Confirm from the sg7 pair whether the
   walk-based closure reaches the libcxx/protobuf set; if reaching the full set
   requires propagating the producer's `inputClosure` and that proves too broad
   for this ticket, land the first-arg fix + direct-source closure and report the
   transitive-header remainder as a narrow follow-up.
