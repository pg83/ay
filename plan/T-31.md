# T-31 plan: ALL_RESOURCE_FILES / ALL_RESOURCE_FILES_FROM_DIRS expansion

## Upstream mechanism (ymake.core.conf:2847-2879)

```
macro _ARF_HELPER(Args...) { RESOURCE_FILES($Args) }

macro ALL_RESOURCE_FILES(EXT, PREFIX="", STRIP="", DIRS...) {
     _GLOB(ALL_RES_$EXT ${suf=/*.$EXT:DIRS} RESTRICTIONS ...)
     _ARF_HELPER(${pre=PREFIX :PREFIX} STRIP ${ARCADIA_ROOT}/${MODDIR}/${STRIP} ${pre=$ALL_RES_:EXT})
}

macro ALL_RESOURCE_FILES_FROM_DIRS(PREFIX="", STRIP="", DIRS...) {
     _GLOB(_ALL_RES_DIRS ${suf=/*:DIRS} RESTRICTIONS ...)
     _ARF_HELPER(${pre=PREFIX :PREFIX} STRIP ${ARCADIA_ROOT}/${MODDIR}/${STRIP} ${_ALL_RES_DIRS})
}
```

So the macro is exactly: glob `<dir>/*.<ext>` (or `<dir>/*` for the FROM_DIRS
variant) for each DIR, then call `RESOURCE_FILES` with:

- `PREFIX <prefix>` only when PREFIX is non-empty (`${pre=PREFIX :PREFIX}`),
- always `STRIP ${ARCADIA_ROOT}/${MODDIR}/${STRIP}` (note: with empty STRIP this
  is `${ARCADIA_ROOT}/<moddir>/`, trailing slash),
- the globbed file list.

Key fact verified against sg7.ref: the glob keeps the literal `${ARCADIA_ROOT}`
in the stored path (the GLOB runs before `${ARCADIA_ROOT}` is bound), so the
resfs/file key embeds the literal `${ARCADIA_ROOT}`. Decoded ref key for
models_meta:

```
resfs/file/yabs/.../models_meta/configs${ARCADIA_ROOT}/yabs/.../models_meta/configs/<file>.json
```

`expandResourceFiles` already reproduces this exactly when fed
`PREFIX <p> STRIP ${ARCADIA_ROOT}/<moddir>/ ${ARCADIA_ROOT}/<dir>/<file>.json`:
prefix is prepended to the (un-stripped, since configs != libs/cpp) path,
yielding the same key, and the existing resource machinery
(`moduleRootedVFS`/`copyFileInputVFS`/`rootrelExpand`) already resolves
`${ARCADIA_ROOT}/...` to `$(S)/...` for the `--inputs` and the resfs/src kv RHS.

## Why ARCADIA_ROOT must stay literal in the file path

During collection `${ARCADIA_ROOT}` is bound to `$(S)`. If we let the normal
expansion run over the constructed file paths the key would embed `$(S)` and the
objcopy hash would drift from REF. We therefore expand DIR tokens fully (to
resolve user SET vars like `${CONFIGS_PATH}`), then canonicalize the leading
`$(S)/` back to `${ARCADIA_ROOT}/` (and `$(B)/` to `${ARCADIA_BUILD_ROOT}/`) for
the stored path form — `${ARCADIA_ROOT}` is identically `$(S)`, so this is exact,
not a special case. The STRIP token is built directly as
`${ARCADIA_ROOT}/<moddir>/<strip>` so it stays literal too.

## Rework: relative DIR globbing (reviewer feedback)

Upstream `_GLOB` resolves a bare DIR against the module dir
(`Module.GetDir()`): `TGlobPattern` sets `RootDir = Module.GetDir()` and
`Pattern = TryCutCurdir(glob)` for any DIR that is not a known path-var prefix
(`ToYPath` returns false), and `Fixed`/`Reconstruct` cleans embedded `..`.
Every match is then stored as `${ARCADIA_ROOT}/<arc-rel>` —
`module_loader.cpp`: `TString::Join("${ARCADIA_ROOT}/", x.CutType())` — so a
source-rooted DIR and a moddir-relative DIR collapse to the identical stored
form. Real source uses both `ALL_RESOURCE_FILES(j2 templates)` (relative, under
the module) and `ALL_RESOURCE_FILES_FROM_DIRS(PREFIX adminka_projects
../../configs/adminka/projects)` (relative, parent-escaping).

`allResourceDir` now takes `modulePath` and, for a non-typed DIR, returns
`${ARCADIA_ROOT}/path.Clean(modulePath/dir)` as the stored form and the cleaned
arc-rel as the glob dir. A `$`-prefixed (non-`$(S)`) root, an absolute path, or
an out-of-tree `..` result stays ok=false (not globbable). When the relative
dir lands under the module, `STRIP=${ARCADIA_ROOT}/<moddir>/` actually strips,
yielding a moddir-relative key (the `templates/<file>` case); when it escapes to
a sibling/parent the marker survives in the key (the models_meta/idm_syncer
case). Two added tests pin both sub-cases.
