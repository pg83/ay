# T-48 тАФ with_transitive_headers wrapper-proto: producer input + module_dir attribution

## Symptom (sg7, reviewer artifact)

`$(B)/ads/caesar/libs/profiles/proto/with_transitive_headers/advm_banner.pb.h`
producer node differs from ref on TWO independent fields (confirmed by
`ay dev dump diff --pair`):

1. **inputs** тАФ ours adds 4 generated WKT headers ref does not have:
   - `$(S)/contrib/libs/protobuf/src/google/protobuf/{descriptor,timestamp}.pb.h`
   - `$(S)/contrib/libs/protobuf_old/src/google/protobuf/{descriptor,timestamp}.pb.h`
   (both sides keep the `.proto` siblings; only the `.pb.h` are extra)

2. **target_properties.module_dir** тАФ ours `ads/caesar/libs/profiles/proto/with_transitive_headers`
   (the producer module), ref `yabs/plutonium/libs/profiles` (the consumer that
   PEERDIRs the wrapper and names its headers in OUTPUT_INCLUDES).

All 69 `with_transitive_headers/*.pb.h` outputs belong to ONE producer node (the
`transitive_proto` RUN_PROGRAM); both divergences are properties of that single node.

The two divergences have DIFFERENT root causes (verified below) тАФ they are fixed
independently.

## Producer module

`ads/caesar/libs/profiles/proto/with_transitive_headers/ya.make` is a LIBRARY with
a single `RUN_PROGRAM(ads/bsyeti/tools/transitive_proto тАж IN <75 .proto> OUTPUT_INCLUDES
<the ads/caesar/libs/profiles/proto/*.pb.h> OUT <the 69 wrapper *.pb.h>)`. The wrapper
headers are consumed by `yabs/plutonium/libs/profiles` YaFF RUN_PROGRAMs via
`--includes`/`OUTPUT_INCLUDES ads/caesar/libs/profiles/proto/with_transitive_headers/<x>.pb.h`.

## Divergence 1 тАФ WKT `.pb.h` sibling over-emit

Upstream mechanism: a RUN_PROGRAM's PR-node inputs come from IN-file scanning plus
OUTPUT_INCLUDES induced-dep realization. `emit_pr.go:prInputClosure` walks each
OUTPUT_INCLUDES codegen `.pb.h` closure, keeps the transitive `.proto` sources, and
тАФ for protobuf WKTs тАФ additionally appends the checked-in `.pb.h` sibling next to
each surfaced `.proto` (commit c5549aa, added for `control_board_proto.h`).

That sibling synthesis OVER-fires when the run itself consumes `.proto` IN files.
Evidence (sg7/sg5):
- `control_board_proto.h` (IN = `.h.in`, no `.proto` IN) тЖТ ref KEEPS WKT `.pb.h`.
- `ads/caesar/libs/events/.../registrar.cpp` (no IN, OUTPUT_INCLUDES only) тЖТ ref KEEPS WKT.
- `transitive_proto` (IN = 75 `.proto`) тЖТ ref DROPS WKT тАФ the `.proto` graph is
  already rooted at the IN `.proto` files (whose scan induces `.proto` imports, not
  `.pb.h`), so the OUTPUT_INCLUDES walk must not re-synthesize the `.pb.h` sibling.

Fix: gate the WKT `.pb.h` sibling append on the run having NO `.proto` IN file.
Generic, no allowlist; keyed on the structural property (run consumes `.proto`).

## Divergence 2 тАФ producer module_dir (consumer-claim not recorded)

`attribute_generated.go:overrideGeneratedModuleDir` re-attributes a PR/CF/CP
producer to the first CONSUMER module that scan-claims one of its outputs
(`scanner.go:recordFirstClaim`, first-write-wins тАФ mirrors upstream
`Node2Module` first-DFS-leaver). The wrapper `.pb.h` are never scan-resolved: they
ride into consumers as OutTogether closure leaves (no resolve тЖТ no claim), and the
`walkClosureTail` ROOT in `prInputClosure`'s OUTPUT_INCLUDES loop is the consumed
header itself, which is added to the window directly (not via resolve) тАФ so NO
module records a claim and the node keeps its producer-time module_dir.

Upstream attributes the producer node to `yabs/plutonium/libs/profiles`: that module
processes its YaFF RUN_PROGRAM (OUTPUT_INCLUDES naming the wrapper `.pb.h`) and leaves
the producer node first. Mirror it: when `prInputClosure` resolves an OUTPUT_INCLUDES
target to a codegen output, `recordFirstClaim(info.OutputPath, instance.Path.rel())`.
Safe for the `control_board тЖТ tablet.pb.h` case: `tablet.pb.h`'s producer is a PB
node, which `overrideGeneratedModuleDir` skips (only PR/CF/CP are re-attributed).

## Files

- `emit_pr.go` тАФ `prInputClosure`: (a) gate WKT `.pb.h` sibling on no-`.proto`-IN;
  (b) record first-claim for the OUTPUT_INCLUDES codegen target.
- `emit_pr_test.go` тАФ regression: a wrapper RUN_PROGRAM (`.proto` IN + OUTPUT_INCLUDES
  codegen `.pb.h`) consumed by a profile-like module; assert (1) producer inputs carry
  the transitive `.proto` but NOT the WKT `.pb.h`, (2) producer module_dir = consumer;
  and a control RUN_PROGRAM (no `.proto` IN) still carries WKT `.pb.h`.
