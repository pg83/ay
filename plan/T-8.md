# T-8 — FROM_SANDBOX FILE/OUT outputs consumed by RESOURCE-style macros

## Upstream mechanism

`FROM_SANDBOX(FILE <id> OUT_NOAUTO <name>)` fetches a build artifact into the
module build dir, producing `$(B)/<moduledir>/<name>` (see T-4/T-6). A
`RESOURCE(<path> <key>)` in the same module then embeds that artifact. Upstream
ymake resolves the RESOURCE file argument through its standard `${input:...}`
resolution: a path may be **module-relative** (`data.json`), **`${BINDIR}`-rooted**,
or **arcadia-root-relative** (`yt/yt/library/ytprof/bundle/llvm-symbolizer`). The
resolver finds the file as a known build output (the FROM_SANDBOX node's
`$(B)/.../<name>`), so the objcopy packer reads the artifact from `$(B)` and the
objcopy node takes the producing (SB) node as a dependency. A `--global-resource`
key (`resfs/file/...` / the literal RESOURCE key) is computed from the RAW path
argument, unchanged between a source and a generated resource (upstream hashes
the raw pair; `objcopyHash` already mirrors that).

Two real sg7 cases (both `RESOURCE`, not `RESOURCE_FILES`), reached transitively
from `yabs/server/daemons/bs_static`:

- `yt/yt/library/ytprof/bundle`: `FROM_SANDBOX(FILE … OUT_NOAUTO llvm-symbolizer)`
  + `RESOURCE(yt/yt/library/ytprof/bundle/llvm-symbolizer /ytprof/llvm-symbolizer)`.
- `quality/functionality/goods/unidata/canonizer`:
  `FROM_SANDBOX(FILE … RENAME RESOURCE OUT subdomain_region_tokens.txt)` +
  `RESOURCE(quality/.../canonizer/subdomain_region_tokens.txt subdomain_region_tokens …)`.

In both the RESOURCE path is **arcadia-root-relative but rooted at the module dir**.

## Divergence (reproduced)

`emit_py_objcopy.go::emitResourceObjcopy` already probes the codegen registry for
a generated resource, but it canonicalizes the RESOURCE path with
`copyFileOutputVFS(modulePath, e.Path)`, which **always prepends the module dir**.
For an arcadia-root-relative path that is already under the module dir
(`yt/yt/library/ytprof/bundle/llvm-symbolizer`) this **doubles** the path
(`$(B)/yt/yt/library/ytprof/bundle/yt/yt/library/ytprof/bundle/llvm-symbolizer`),
so the registry lookup misses the FROM_SANDBOX output. The input then falls back
to `copyFileInputVFS`, which resolves it to a **source** VFS
(`$(S)/yt/yt/library/ytprof/bundle/llvm-symbolizer`) for a file that does not
exist on disk. Under `--sandboxing` the UID finalizer content-hashes that source
input and faults on the missing file, aborting sg7 with a 0-byte graph.

`copyFileOutputVFS` blindly prepends because producer-side OUT tokens are
module-relative by convention. The consumer (RESOURCE) side needs the
module-rooted resolution `sourceInputVFS` already encodes for the `$(S)` side.

## Files touched

- `modules.go` — add `resourceOutputVFS(modulePath, path)`: the `$(B)` analog of
  `sourceInputVFS`'s module-rooted branch. `${BINDIR}`/`$(B)`/`${ARCADIA_*}`
  prefixes resolve via `moduleRootedVFS`; a path already equal to or under
  `modulePath` is used verbatim (`build(clean)`); otherwise the module dir is
  prepended (`build(modulePath/clean)`), matching `copyFileOutputVFS`.
- `emit_py_objcopy.go` — in `emitResourceObjcopy`'s `emitEntries`, key the codegen
  registry probe and the swapped input VFS on `resourceOutputVFS` instead of
  `copyFileOutputVFS`. Source resources still miss the registry and keep their
  `copyFileInputVFS` source path; only a generated (registered) output now
  resolves to `$(B)` with the producer dep pinned.
- `emit_py_objcopy_test.go` — regression: a `FROM_SANDBOX(FILE … OUT_NOAUTO x)`
  feeding a `RESOURCE(<moduledir>/x <key>)` — assert the objcopy input is
  `$(B)/<moduledir>/x` (not `$(S)/…` and not the doubled path), the objcopy
  depends on the SB node, and the resfs key (base64 of the literal RESOURCE key)
  is unchanged.

## Expected effect on the gate

- `go test ./...` green; new regression fails before (source/doubled input),
  passes after.
- `./dev/validate.py` keeps sg2/sg2_x86_64/sg3/sg4 OK counts, XFAIL count, sg5
  matched count, and the per-case generation-time budget; sg2–sg6 stay byte-exact
  (no FROM_SANDBOX-fed RESOURCE in those cases). Direct sg7 no longer aborts on
  the ytprof/canonizer fetched RESOURCE inputs (full sg7 exit 0 stays out of
  scope; any remaining resource failure of a distinct mechanism is reported, not
  merged here).
