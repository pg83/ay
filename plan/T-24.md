# T-24 plan: restore CYTHONIZE_PY pxd Dep to the `.py` branch only

## Finding

`TestGen_CythonizePyPxdSideInputClosure` (added by T-18) failed on the trunk
state at ticket-creation time. On the current trunk HEAD it PASSES: the T-19
acceptance-gate merge (commit touching `modules.go | 2 ++`) added the missing
`Pxd:` assignment to the CYTHONIZE_PY `.py` branch, which T-18 had omitted.

Reproduction of the original failure (at T-18 commit `1e3b253`):

    go test . -run TestGen_CythonizePyPxdSideInputClosure
    emit_cython_cpp_test.go:150: CY node missing pxd-side input
        "$(S)/pkg/gevent/_helper.pxd"

Root cause: T-18 placed `CythonStmt.Pxd = <mod-as-path>.pxd` on the **`.pyx`**
branch of `applyUnknownStmt` instead of the **CYTHONIZE_PY `.py`** branch its
own plan specified. The merge fix added the field to the `.py` branch but left
the misplaced `.pyx` assignment in place. Current HEAD therefore sets `Pxd` on
**both** branches.

## Upstream mechanism (from T-18 plan, build/plugins/pybuild.py::onpy_srcs)

    dep = path
    if path.endswith('.py'):
        pxd = '/'.join(mod.split('.')) + '.pxd'
        if unit.resolve_arc_path(pxd):
            dep = pxd
    cython_args.append(dep)

`dep == path` for `.pyx` sources (the source is already a CY input → the hidden
`${hide;input:Dep}` is a dedup no-op). Only a `.py` source whose
`<mod-as-path>.pxd` resolves turns `dep` into a pxd. So the `Dep=pxd` mechanism
is `.py`-only; for `.pyx` the sibling pxd rides through the normal cimport scan
(`sourceClosure`), not through `Dep`.

The field's own doc comment already encodes this: "Pxd is the module-relative
`<mod-as-path>.pxd` candidate for a CYTHONIZE_PY `.py` source (empty otherwise)".
The `.pyx`-branch assignment contradicts both upstream and the documented
contract.

## Divergence the `.pyx` assignment causes

For a `.pyx` whose `<mod-as-path>.pxd` resolves but is NOT in the `.pyx`'s
cimport closure, `resolveCythonPxd` pulls that pxd (and its closure) onto the CY
node as a spurious `Dep` input — an input upstream does not add (`dep == path`).

## Files to touch

- `modules.go`: remove the `Pxd:` line (and its comment) from the `.pyx`
  branch. Keep it on the CYTHONIZE_PY `.py` branch.
- `emit_cython_cpp_test.go`: add a regression test pinning that a `.pyx` whose
  `<mod-as-path>.pxd` resolves but is not cimported carries NO pxd Dep input
  (upstream `dep == path`). Fails at HEAD (`.pyx` Pxd present), passes after.
