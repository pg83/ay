# T-94 plan: RUN_PROGRAM IN/OUT arg path-rooting (deep-replace)

## Divergence (representative)

`$(B)/contrib/libs/libmysql_r/strings/uca900_ja_tbls.cc` (sg7), source
`contrib/libs/libmysql_r/strings/ya.make`:

```
RUN_PROGRAM(
    contrib/libs/libmysql_r/strings/uca9dump ja --in_file=lang_data/ja_hans.txt --out_file=uca900_ja_tbls.cc
    CWD ${ARCADIA_BUILD_ROOT}/contrib/libs/libmysql_r/strings
    IN lang_data/ja_hans.txt
    OUTPUT_INCLUDES my_inttypes.h
    OUT uca900_ja_tbls.cc
)
```

Pair diff `ay dump diff --pair`:

```
[field cmds differs]
  -ours +--in_file=lang_data/ja_hans.txt
  -ours +--out_file=uca900_ja_tbls.cc
  -ref  +--in_file=$(S)/contrib/libs/libmysql_r/strings/lang_data/ja_hans.txt
  -ref  +--out_file=$(B)/contrib/libs/libmysql_r/strings/uca900_ja_tbls.cc
```

Our emitter leaves the flag-args (`--in_file=<rel>`, `--out_file=<rel>`) relative;
upstream roots the path embedded after `=` to `$(S)`/`$(B)`.

## Upstream mechanism

`devtools/ymake/args_converter.cpp` `FillTypedArgs`: for every macro keyword
carrying a deep-replace modifier (`IN{input}`, `IN_NOPARSE{input}`, `OUT{output}`,
`OUT_NOAUTO{output}`, `TOOL{tool}` in the `RUN_PROGRAM` signature, ymake.core.conf),
each value passed to that keyword is a "scriptArg". Sorted longest→shortest, each
scriptArg is searched as a **substring** of every positional `Args...` token. On a
match with valid boundaries (`IsValidSymbol`: the char immediately before and after
the match is NOT ascii-alpha / `.` / `_` / `-` / `"` / `/`) and when the path
`NPath::MustDeepReplace` (i.e. it is not already root-typed / `${ARCADIA_*}` /
absolute), the substring is rewritten to its rooted spelling (`${input"…"}` →
`$(S)/…`, `${output"…"}` → `$(B)/…`). Each arg position is replaced at most once
(longest matching scriptArg wins; the position is then consumed). The macro doc
states it directly: "Relative paths listed in TOOL, IN, OUT, STDOUT become absolute."

Our `emitPR` (emit_pr.go) only rooted args that were *exactly* an IN/OUT token (or
`head:modifier`) and explicitly **skipped** any arg containing `=` or starting with
`-`. That skip is the defect: `--in_file=lang_data/ja_hans.txt` never gets rooted.

## Change

`emit_pr.go`, `emitPR` arg loop:

- Replace the exact-match + `head:modifier` IN/OUT substitution (current lines that
  guard on `!HasPrefix("-") && !Contains("=")`) with a faithful deep-replace:
  - Build candidate list from `stmt.INFiles` (rooted via `inVFSByToken`) and
    `stmt.OUTFiles`/`stmt.OUTNoAutoFiles` (rooted via `outVFSByToken`), filtered by a
    `mustDeepReplace` predicate (drop tokens already `$(S)/`,`$(B)/`,`${ARCADIA_*}/`,
    `${CURDIR}/`,`${BINDIR}/` or absolute). STDOUT carries no `{output}` marker
    upstream, so it is excluded.
  - Sort candidates by token length descending.
  - For each positional arg, after the existing aux-TOOL substitution, apply at most
    one deep-replace: first (longest) candidate whose token occurs with valid
    boundaries wins; rewrite `head + rooted + tail`.
- Keep the existing aux-TOOL token substitution untouched (it already reproduces the
  `{tool}` deep-replace for relative TOOL dirs; rooted TOOLs are skipped, matching
  `MustDeepReplace`=false).

This both fixes the `=`/`-` flag-arg case and subsumes the old exact-match and
`head:modifier` paths (net code shrink), matching the single generic upstream
mechanism rather than special-casing libmysql.
