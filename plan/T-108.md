# T-108 — Python-protobuf `--fatal_warnings` command residual

## Observed divergence

sg7 pairs the node producing `$(B)/sprav/protos/altay_api__intpy3___pb2.py`
(and its `.pyi` / other `__intpy3___pb2.py` siblings) but the `cmds` field
differs: the upstream reference command carries `--fatal_warnings`, ours omits
it. Same ref-only flag on the paired `.pyi` outputs.

## Upstream mechanism

`sprav/protos/ya.make` is a `PROTO_LIBRARY()` that calls
`PROTOC_FATAL_WARNINGS()`. In `ydb/build/conf/proto.conf`:

- `macro PROTOC_FATAL_WARNINGS() { SET_APPEND(_PROTOC_FLAGS --fatal_warnings) }`
  (proto.conf:144) — appends `--fatal_warnings` to the `_PROTOC_FLAGS` variable.
- The python proto command base `_PY_PROTO_CMD_BASE` (proto.conf:516) renders:
  `$PY_PROTOC … --python_out=$ARCADIA_BUILD_ROOT/$PROTO_NAMESPACE $_PROTOC_FLAGS ${rootrel;input:File} …`
  i.e. `$_PROTOC_FLAGS` lands **after `--python_out` and before the trailing
  source token**. The grpc/mypy plugins ride `$Args`, appended after the source.

This is exactly how the C++ side already works: `composePBArgBlocks`
(emit_pb.go:411) does `mid = appendArgStr(mid, extraProtocFlags)` right after
`--cpp_out=…`, where `extraProtocFlags == d.protocFlags`. The macro already
populates `d.protocFlags` (modules.go:2000, both `PROTOC_FATAL_WARNINGS` and
`SET_APPEND(_PROTOC_FLAGS …)` at :2703). The py emitter
(`newPyPBModuleEmission`, emit_py_proto.go) builds the same command but never
appends `d.protocFlags`. That omission is the residual.

## Change

In `newPyPBModuleEmission` (emit_py_proto.go), append `d.protocFlags` to
`pe.mid` immediately after the `--python_out=$(B)/<protoRoot>` element, via
`appendArgStr`, mirroring the cpp side. Single-line addition; no new state,
no new data path — reuse the already-parsed `d.protocFlags`.

## Files touched

- `emit_py_proto.go` — append `d.protocFlags` after `--python_out`.
- `emit_py_proto_test.go` — fail-first test: a `PROTO_LIBRARY` +
  `PROTOC_FATAL_WARNINGS()` whose py PB node command carries `--fatal_warnings`
  positioned after `--python_out` and before the source token; a sibling
  PROTO_LIBRARY without the macro must NOT carry it.

## Expected gate effect

- sg7 `altay_api__intpy3___pb2.py` pair + ≥1 `.pyi` pair become header-only.
- T-106 aggregate `--by-token` `--fatal_warnings` only-ref count decreases
  (real command drift removed).
- sg2–sg6 stay byte-exact (none of those targets use PROTOC_FATAL_WARNINGS on a
  py-addressed PROTO_LIBRARY, so the added-only-when-present flag is inert).
- old `actors.pb.h` duplicate-output representative unaffected (cpp path).
