# T-5 ‚Äî Resolve arcadia-root-relative SRCS from the source root

## Hypothesis (reproduced before the fix)
A ya.make may list a SRCS entry as an arcadia-ROOT-relative path that does not
live under the declaring module dir. E.g. `geobase/library/abi` lists
`geobase/library/asset.cpp` (the file is at `$(S)/geobase/library/asset.cpp`,
not `$(S)/geobase/library/abi/geobase/library/asset.cpp`); likewise
`market/idx/datacamp/proto/external` lists
`market/idx/datacamp/proto/api/ExportMessage.proto`.

Our `resolveSourceVFS` (sources.go) and `resolvePySrcRel` (resource.go) search
the SRCDIR path then fall back to joining the entry under the module dir. For a
root-relative entry that exists at neither a SRCDIR nor the module dir, the
fallback DOUBLES the path: `$(S)/<moduledir>/<entry>`. The doubled path names a
nonexistent source; under sg7 the UID finalizer's content-hash of that source
input hard-aborts generation (the geobase / market doubled-path SRCS class).

## Upstream mechanism
ymake's source-resolution plan for a SRCS/source-input entry ends at the arcadia
root: after the curdir (module dir) and the SRCDIR search path, a clean entry
that exists at `$(S)/<entry>` binds there (`$(S)/<entry>`), not under the module
dir. The module dir is consulted first, so a genuinely co-located source still
binds locally (curdir wins); SRCDIR still wins where it applies. This is generic
‚Äî no per-directory special-casing.

## Change
Insert, in both resolvers, after the SRCDIR reverse search and before the
module-dir fallback, the arcadia-root binding:

- a clean entry (`pathIsClean`, non-empty)
- that does NOT exist under the module dir (`!isFile(moduleDir, entry)`)
- but DOES exist at the source root (`isFile(srcRootVFS, entry)`)

resolves to `source(entry)` (resolveSourceVFS) / `entry` (resolvePySrcRel)
instead of the doubled `$(S)/<moduledir>/<entry>`.

This mirrors verified reference work (workspace commit e832623), but lands ONLY
the source-resolution behavior ‚Äî NOT the separate py-proto generated-proto
override (`emit_py_proto.go`), which addresses the distinct
generated-file-consumed-as-input class that is out of scope here.

- `resource_test.go` ‚Äî `GenCtx` fixture needs a non-nil `fs` (resolveSourceVFS

  listing a local source and an arcadia-root-relative source; assert the emitted
  CC nodes carry the correct `$(S)` input paths (local ‚Üí module-relative,
  root-relative ‚Üí `$(S)/<entry>`, never doubled).

## Test-first
1. Graph-level regression fails before the fix: the root-relative source's CC
   node lists the doubled `$(S)/<moduledir>/<entry>` input.
2. Implement the rule until it passes (root-relative ‚Üí `$(S)/<entry>`), local
   source still `$(S)/<moduledir>/local.cpp`.
3. Gate: `./dev/validate.py` stays green (sg2/sg2_x86_64/sg3/sg4 OK counts hold,
   XFAIL not growing, sg5 matched not dropping, within time budget). Direct sg7
   no longer aborts on the geobase/market doubled-path SRCS class; if it advances
   to a generated-file-consumed-as-input failure, STOP and report ‚Äî do not widen.

## Expected effect on the gate
