# T-90 plan: SRCDIR/rooted proto enum-serialization output path residual (T-87-C)

## Upstream mechanism

The output build-directory of a command in module `M` is computed by
`TCommandInfo::InitDirs` (yatool/devtools/ymake/macro_processor.cpp:805):

```
BuildDirStr = $B/<M>
if input has a parent dir:
    relative = Relative($B/<input-dir>, $B/<M>)        # input dir relative to module
    if transform: SubstGlobal(relative, "..", "__")    # .. -> __
    if relative.StartsWith("__"):                       # ascent OR literal __-prefixed dir
        BuildDirStr = Join($B/<M>, relative)            # used directly, NO _/ localization
    else:
        BuildDirStr = Join($B/<M>, "_", relative)       # subdir → _/ localization
```

Two consequences this ticket reproduces:

1. **Object localization.** The `_/` localization segment is inserted only when the
   compiled source's directory (relative to the module) does NOT start with `__`. A
   directory literally named `__frameworks__` (donotuseme/taxi schemas) hits the same
   `StartsWith("__")` branch as a `..` ascent and is joined directly — NO `_/`.
   Our `normalizeDotDotSegments` unconditionally prepends `_/` for any subdir, so
   `__frameworks__/…` and the taxi schema proto objects get a spurious `_/`.

2. **EN serialized output location.** `GENERATE_ENUM_SERIALIZATION` emits
   `${output;suf=_serialized.cpp:File}`. For a **rooted** `File` spelling
   (`${ARCADIA_BUILD_ROOT}/m/OfferMeta.pb.h`) the output lands at the File's own
   resolved build location (`$B/m/OfferMeta.pb.h_serialized.cpp`), NOT joined under the
   module BINDIR. Our `emit_en.go` left the full root-relative `headerRel` intact and
   prepended the module dir, producing the doubled
   `$B/m/meta/m/OfferMeta.pb.h_serialized.cpp`; the downstream compile then localized
   it under `_/`. With the serialized.cpp at the header's own dir, the compile object
   falls through the existing build-source-outside-module branch
   (`composeCCPaths`/`composeSrcDirOutputRel`) and rebases under `m/meta/__/`.

   A **relative** `File` spelling keeps the module-relative (possibly doubled) output
   — preserved exactly (TestGen_EnumSerializationRootQualifiedHeaderUsesCanonicalInput).

## Files touched

- `emit_cc.go` — `normalizeDotDotSegments`: skip the `_/` localization when the
  directory (after `..`→`__`) starts with `__`.
- `emit_en.go` — rooted-spelling serialized output at the header's own build location;
  thread the serialized VFS into `emitEN` and the downstream compile.
- `emit_codegen_cc.go` — add `emitCodegenDownstreamCCFromVFS(... cppPath VFS ...)` so a
  caller with an already-computed cross-dir output VFS compiles it directly.
- `emit_en_test.go` — fail-first coverage: SRCDIR proto + rooted GENERATE_ENUM_SERIALIZATION.
