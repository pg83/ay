## Upstream mechanism

`RESOURCE(${BINDIR}/main.gzt.bin WT_GZT ${BINDIR}/grammar.bin WT_GRAMMAR)` embeds
two RUN_PROGRAM-chain outputs. Upstream's flat-input model lists on a node the
full transitive `$(S)` source closure of every input it consumes — so the objcopy
node that embeds the generated `.bin` payloads also lists the source leaves the
chain's producers consumed (`gazetteer.gzt`, the `.remorph` files,
`cascade.gztproto`), reached directly (first producer's `$(S)` INs) and
transitively through the opaque generated `.bin` inputs of the downstream
producers.

Our `emit_pr.go:emitRunProgram` already records this on each generated output:
- `GeneratedFileInfo.SourceInputs` — the producer's direct `$(S)` leaves, folded
  forward across opaque generated INs (`prGeneratedFromSources`).
- `GeneratedFileInfo.ProducerSourceClosure` — the producer's transitive parsed
  `$(S)` closure, folded across opaque generated INs by `prInputClosure`.

`emit_py_objcopy.go:resolveResourceInput` exposes only the payload path + producer
refs; `emitResourceObjcopy.flush` never appends these source sets. That is the
single gap.

The reference graph's objcopy `inputs` are pruned by `dump.go:objcopyInputKept`
(refGraph-only over-emit discount): a `$(S)` leaf is dropped when its extension is
a C/C++ compile-closure ext (`objcopyOverEmitExts`, incl. `.proto`), or it lives
under `contrib/libs/`, or it is `build/*.py`. So `base.proto` (a direct IN of the
first producer) is dropped ref-side. Our faithful side is NOT normalized, so we
must emit exactly the kept set — mirror the same predicate when appending, the
same way the existing build-root closure tail keeps only `v.isBuild()`.

## Files touched

- `dump.go`: extract the `$(S)`-leaf keep rule of `objcopyInputKept` into a shared
  predicate `objcopySourceLeafKept(rel string) bool` (DRY; no behavior change).
- `emit_py_objcopy.go`:
  - `resolveResourceInput` returns a small `resolvedResource` struct adding
    `SourceInputs` and `SourceClosure` (from `GeneratedFileInfo`).
  - `acc` gains `srcAttrInputs []VFS`; the path branch of `emitEntries` collects
    `SourceInputs`+`ProducerSourceClosure` filtered by `objcopySourceLeafKept`.
  - `flush` appends the deduped source-attribution leaves to the cache-key-only
    input tail — NOT into `cur.paths`, `--inputs`, `--keys`, `objcopyHash`, the
    command, or the output name.
