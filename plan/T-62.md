# T-62 plan: effective USE_ARCADIA_LIBM branch for normal link modules

## Upstream mechanism

`build/ymake.core.conf:902-948` gates the libm branch inside
`when ($COMMON_LINK_SETTINGS == "yes")`, i.e. only on **link modules**
(`_LINK_UNIT` enables `COMMON_LINK_SETTINGS` at line 1164; plain libraries do
not), and inside the `otherwise` of the Emscripten guard:

```
938  when ($USE_ARCADIA_LIBM == "yes") { PEERDIR += contrib/libs/libm }
941  when ($USE_ARCADIA_LIBM == "no")  { C_SYSTEM_LIBRARIES += -lm }
...
946  when ($USE_COW == "yes")          { PEERDIR += build/cow/on }
```

Two upstream facts the current ay model violates:

1. **libm PEERDIR is a COMMON_LINK_SETTINGS peer, not a language default.** It is
   appended right before `build/cow/on`. In ay, COMMON_LINK_SETTINGS-class peers
   (`build/cow/on`, allocator) live in `defaultProgramPeerdirsForWithState`
   (program path), not in `defaultPeerdirsForWithState` (language defaults). T-57
   placed libm in the language-default path, so its **own** `ADDINCL(GLOBAL
   contrib/libs/libm/include|platform)` floats into the lang-default *own*
   ADDINCL sweep (`gen.go:1782-1790`), ahead of the lang-default *transitive*
   sweep (`gen.go:1792-1800`) that carries `clang20-rt`, `zlib`,
   `double-conversion`, `libc_compat`. Reference orders libm **after** that
   transitive closure — exactly the program-default slot
   (`gen.go:1813-1822`, swept after both lang-default passes).

2. **`-lm` is the COMMON_LINK_SETTINGS `C_SYSTEM_LIBRARIES` append, not a base
   system library.** `build/conf/linkers/ld.conf:106-128` defines the base
   `_C_SYSTEM_LIBRARIES` as `-nodefaultlibs -lpthread -lc` (non-MUSL) /
   `-nostdlib` (MUSL) — **no `-lm`**. The `-lm` is added only by
   `ymake.core.conf:942`, in the `USE_ARCADIA_LIBM == "no"` arm. ay bakes `-lm`
   into `Platform.SystemLibs` (`platform.go:215,218`), so every program emits it
   even when `USE_ARCADIA_LIBM == yes`, which must instead link the libm archive
   and omit `-lm`.

## Files to touch

- `defaults.go`: remove the libm append from `defaultPeerdirsForWithState`; add
  it to `defaultProgramPeerdirsForWithState` `!postUser` branch, before
  `build/cow/on`, keeping the self/descendant guard for `contrib/libs/libm`.
- `platform.go`: drop `-lm` from both `systemLibs` branches (base
  `_C_SYSTEM_LIBRARIES`).
- `emit_ld.go`: thread a `useArcadiaLibm bool` into `emitLD` →
  `composeLDCmdLinkExe` → `composeProgramLinkTrailer`; append `-lm` right after
  `p.SystemLibs` when `!useArcadiaLibm` (reproducing `C_SYSTEM_LIBRARIES += -lm`
  in the `USE_ARCADIA_LIBM == no` arm, in the same trailing position).
- `gen.go`: pass `d.useArcadiaLibm` at the `emitLD` call site.
- `emit_libm_test.go`: keep the existing reachability guards; move the self-peer
  guard to the program path; add the ordering guard and the `-lm` link-trailer
  guards (positive + negative).

## Tests (fail-first)

1. **Ordering**: a `PROGRAM()` with `ENABLE(USE_ARCADIA_LIBM)` plus a `util`
   language-default peer whose transitive peer carries a `GLOBAL` addincl.
   Assert the program compile orders that transitive `-I` **before**
   `-I$(S)/contrib/libs/libm/include`. Fails today (libm own-addincl floats into
   the lang-default own sweep, ahead of the transitive closure).
2. **No system `-lm` with the flag**: program LD command contains the libm
   archive and **not** `-lm`.
3. **Negative guard**: a program *without* the flag keeps `-lm` and gains no
   libm archive.
4. **Self-peer guard** preserved for modules under `contrib/libs/libm`.
