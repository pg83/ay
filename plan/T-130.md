# T-130 plan: INCLUDE first-argument-only parity (caesar CC warning flags)

## Upstream mechanism

`yatool/devtools/ymake/lang/makefile_reader.cpp` handles `INCLUDE` before
normal statement dispatch. After the empty-args guard it evaluates and includes
only `args[0]`:

```cpp
if (command == TStringBuf("INCLUDE")) {
    if (args.empty()) { ... return; }
    TString name = EvalExpr(Context->Vars(), args[0]);
    this->Include(name, ctx.GetLocation().Path);
    return;
}
```

There is no loop over the remaining argument vector and no error for extra
arguments — later INCLUDE arguments are silently ignored.

The caesar representative
`$(B)/ads/caesar/libs/misc/parse_banner_text/parse_banner_text.cpp.o` writes
three paths in one `INCLUDE(...)`; upstream reads only the first
(`ads/bsyeti/ya_check_dependencies`, a `CHECK_DEPENDENT_DIRS` file, not a
warning source). Current ay loops over all three args, parsing the bsyeti/caesar
`ya_cpp.make_cxxflags.inc` files and applying their `CXXFLAGS(...)`, producing
ours-only `-Wimplicit-fallthrough`, `-Wmissing-field-initializers`,
`-Wdeprecated-this-capture` tokens across 17 caesar CC nodes.

## Files to touch

- `yamake.go` — `expandInclude`: expand only `args[0]`; keep the empty-args
  throw; do not reject `len(args) > 1`.
- `yamake_test.go` — replace `TestParseInclude_MultiArgLeftToRight` with a
  fail-first test proving args after the first are ignored; add controls for
  single-argument `.inc` and separate INCLUDE statements.

`expandOneInclude` stays unchanged (variable expansion, `${ARCADIA_ROOT}/`,
relative paths, missing-file silent skip, cycle detection, `INCLUDE_ONCE`).

- No CC inputs/outputs/target_properties drift.
