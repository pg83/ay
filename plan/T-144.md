# T-144 plan: propagate generated-IN producer source closure to downstream PR consumers

## Symptom

`$(B)/sprav/algorithms/unification/working_time/grammar/time_range.bin` (and the
cascaded `grammar.bin`) miss these PR `inputs` versus ref:

```
+$(S)/contrib/libs/protobuf/src/google/protobuf/descriptor.proto
+$(S)/contrib/libs/protobuf_old/src/google/protobuf/descriptor.proto
+$(S)/contrib/libs/protobuf_std/src/google/protobuf/descriptor.proto
+$(S)/sprav/algorithms/unification/working_time/grammar/gazetteer_articles.proto
```

## Source shape (verified, `4/sprav/.../grammar/ya.make`)

Three chained `RUN_PROGRAM`s, all `OUT_NOAUTO`:

1. `dict/gazetteer/compiler ... gazetteer.gzt main.gzt.bin`
   `IN gazetteer.gzt day_ranges.def.remorph time_range.remorph
       ${ARCADIA_ROOT}/kernel/gazetteer/proto/base.proto
       ${ARCADIA_ROOT}/kernel/remorph/cascade/cascade.gztproto`
   `OUT_NOAUTO main.gzt.bin`
2. `tools/remorphc ... -g ${BINDIR}/main.gzt.bin -o time_range.bin time_range.remorph`
   `IN ${BINDIR}/main.gzt.bin time_range.remorph OUT_NOAUTO time_range.bin`
3. `tools/remorphc ... -g ${BINDIR}/main.gzt.bin -o grammar.bin grammar.remorph`
   `IN ${BINDIR}/main.gzt.bin ${BINDIR}/time_range.bin grammar.remorph OUT_NOAUTO grammar.bin`

`main.gzt.bin`'s producer parses its `.proto`/`.gztproto` INs; the transitive
proto closure (the descriptor.proto WKT variants + `gazetteer_articles.proto`)
appears on `main.gzt.bin`'s own PR node (byte-exact in ref) and is recorded as
its `GeneratedFileInfo.ProducerSourceClosure` (= `filterSourceVFS(inputClosure)`).
The direct, *unparsed* source INs (`gazetteer.gzt`, the `.remorph` files) are NOT
in that closure — they live in `GeneratedFileInfo.SourceInputs`.

## Upstream mechanism

Upstream's flat-input model lists the full transitive **source** closure of a
producer on every node that consumes the producer's output. A node consuming a
generated, parser-less artifact (`.bin`) must therefore carry the producer's
direct source leaves AND its transitive parsed source closure — the union of
`SourceInputs` and `ProducerSourceClosure`.

## Defect

`prInputClosure` (emit_pr.go) opaque-IN branch folds only `info.SourceInputs`:

```go
if info := codegenRegForInstance(...).lookup(...); info != nil {
    out = append(out, info.SourceInputs...)
}
```

So the transitive proto closure in `ProducerSourceClosure` never reaches
`time_range.bin`. The cascade to `grammar.bin` follows: once `time_range.bin`'s
own `inputClosure` gains the protos, its own `ProducerSourceClosure` gains them
too, so the same fixed branch propagates them one hop further when `grammar.bin`
consumes both `main.gzt.bin` and `time_range.bin`.

## Change

emit_pr.go, `prInputClosure` opaque-IN branch: additionally fold
`info.ProducerSourceClosure`. Union (not replace) — `SourceInputs` carries the
unparsed direct leaves (`gazetteer.gzt`, `.remorph`) that `ProducerSourceClosure`
omits; the trailing `dedupVFS` collapses the overlap. FROM_SANDBOX producers set
no `ProducerSourceClosure`, so the dictionaries path is unaffected.

## Files

- `emit_pr_test.go`: fail-first test — chained `RUN_PROGRAM`s, first producing

## Expected gate effect

`PR inputs` divergence for the sprav bucket drops to zero; `time_range.bin` and
`grammar.bin` pairs lose their `inputs` diff; sg2–sg6 byte-exact; sg7 matched not
decreasing; no AR/LD input drift.
