# T-12: Close the final sg5 divergence — ydbd LD induced codegen dep-set

## Characterization (DEBUG.md pipeline, measured at this branch HEAD)

After incorporating the T-11 flatbuffers content fix (see below), the entire
sg5 closure is byte-exact **except one node**: `$(B)/ydb/apps/ydbd/ydbd` (the
ydbd LD linker node).

Measured with `ay dump diff --left our.jsonl --right ref.jsonl`:

- `self_uid` (intrinsic content): **0 divergent** across all 26259 nodes after
  the flatbuffers fix (before it, the ydbd LD `self_uid` diverged because
  `contrib/libs/flatbuffers/libcontrib-libs-flatbuffers.a` sat at link position
  1881 instead of 550).
- `outputs`: identical.
- Only divergence: the ydbd LD **full Merkle `uid`**, because its `deps`
  (build-order dependency set) differs:
  - ours: 1766 deps
  - ref: 2116 deps
  - ref carries **exactly 350 deps we omit; we carry 0 extra.**

The 350 missing deps are all codegen-output producer nodes:

| kind | count | example |
|------|-------|---------|
| PB (`.pb.h` proto headers)        | 338 | `$(B)/ydb/core/protos/*.pb.h` |
| PR (`.gen.h` run-program headers) | 10  | `$(B)/ydb/core/base/generated/runtime_feature_flags.h` |
| JV (antlr4 generated `.cpp`)      | 1   | `.../v1_antlr4/SQLv1Antlr4Parser.cpp` |
| PY (antlr4 generated `.pb.code0.cc`) | 1 | `.../v1_proto_split_antlr4/SQLv1Antlr4Parser.pb.code0.cc` |

These 350 are present in `deps` only — **none appear in the LD node's `inputs`
or `cmds`.** They are pure build-ordering (Merkle-uid) metadata: the actual
link command and its file inputs are byte-identical to upstream. The build the
graph describes is correct; only the cache-invalidation hash of this one node
differs.

## Part landed: the T-11 flatbuffers content fix (ce68b00)

Upstream `_CPP_FLATC_CMD` (build/conf/fbs.conf) carries
`.PEERDIR=contrib/libs/flatbuffers`, adding it as an induced PEERDIR to every
module with `.fbs` SRCS (e.g. `contrib/libs/apache/arrow`), appended after the
module's explicit PEERDIRs. Reproduced in `gen.go::genModule`: for any module
whose `d.srcs` contains a `.fbs` entry (except flatbuffers itself), append
`contrib/libs/flatbuffers` to `d.peerdirs`. Regression test
`TestGen_FbsSrcsInduceFlatbuffersLinkDep`. This makes the ydbd LD `self_uid`
byte-exact (cmds/inputs match).

## The residual upstream mechanism (PB/PR/JV/PY induced LD deps)

The 350 edges come from ymake's `TJSONVisitor` dependency-propagation pass
(`devtools/ymake/json_visitor.cpp`), which our generator does not model. The
relevant machinery:

- `AcceptDep` / `PrepareLeaving`: a module node accumulates `NodeDeps` and
  `IncludedDeps` during a DFS over the dep graph, with module attribution by
  first-leaving (`Node2Module`), `OutTogether` output grouping
  (`GetOutTogetherDependency`), `ExtraOuts` (`Module->IsExtraOut`,
  `add_to_outs`), `GlobalSrcs` (`AddGlobalSrcs`, `SRCS_GLOBAL`), `PassPeers`
  propagation, and the `JsonDepsFromMainOutput` config flag.
- The rendered JSON `deps` of the program = peer module nodes (ARs, which we
  emit correctly) **plus** the codegen producers reached through this
  traversal.

### Why the set is not reproducible via a simple closure

The 350 are a *specific* subset of the 1712 codegen outputs consumed somewhere
in the closure. Empirically, no derivable property reproduces it (each tested
on the reference graph):

| candidate rule | result vs the 350 |
|----------------|-------------------|
| codegen consumed by any non-LD node | 1712 (way over) |
| 1-hop codegen from LD's direct AR deps | 283 → overlap 195, uncovered 155 |
| transitive (all deps) from ydbd's own CC nodes | 487 → overlap 195, extra 292 |
| codegen in include-closure of own + GLOBAL-src CC nodes | 239 → overlap 222, **extra 17, uncovered 128** |

Decisive evidence that this is *not* a dir/module property: within the single
module `contrib/libs/googleapis-common-protos/google/api`,
`annotations.pb.h`/`http.pb.h` ARE LD deps while `auth.pb.h`/`backend.pb.h`
(same dir, same module) are NOT. The discriminator is the exact visitor
traversal — `OutTogether`/`ExtraOuts`/`GlobalSrcs` first-visit attribution —
not any closure over our emitted nodes. The 128-uncovered / 17-extra split of
the closest rule confirms the gap is the traversal semantics, not a missing
edge in a known closure.

Reproducing it faithfully means building ymake's full `TJSONVisitor` NodeDeps
pass as a second graph traversal in our generator — a major new subsystem. A
prior T-12 digger attempted a partial propagation and regressed 4939 sg5 nodes
(matched 26258 → 21320) before reverting. A partial implementation also
violates the "no concept without an upstream counterpart" rule: the closest
closure rule emits 17 edges upstream does **not** have.

## Decision

- Land the flatbuffers content fix (correct, makes ydbd LD `self_uid`
  byte-exact, real upstream induced-PEERDIR mechanism).
- The 350-edge induced-codegen dep set is **irreducible in our architecture**
  without a full JSON-visitor reimplementation, which would jeopardise the four
  currently byte-exact gating cases for one node's Merkle-uid metadata that has
  zero effect on the emitted link command or inputs.
- Emit a `replan` documenting the exact residual so it can be judged
  "converged по существу": 26258/26259 byte-exact; the one residual node's
  build command and inputs are correct; only its build-cache hash differs by a
  350-edge ordering set governed by an unmodelled upstream traversal.
