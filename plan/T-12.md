# T-12 — ARCHIVE member input resolution through the SRCDIR source path plan

## Upstream mechanism (verified against /home/pg/monorepo/4)

`ARCHIVE(NAME=<n> [DONTCOMPRESS] Files...)` (`build/ymake.core.conf:4163`):

    .CMD=$ARCH_TOOL -q -x $DONTCOMPRESS ${suf=\:;input:Files} \
         -o ${addincl;noauto;output:NAME} ${hide;kv:"p AR"} ${hide;kv:"pc light-red"}

The member `Files` are fed through `${input:Files}`. `${input:...}` resolves
each name through the module's *source path plan* — i.e. the cumulative SRCDIR
search path — not by blindly prefixing the module directory. So a module that
declares `SRCDIR(yweb/urlrules)` and then `ARCHIVE(NAME urlrules.inc areas.lst …)`
resolves `areas.lst` to `$(S)/yweb/urlrules/areas.lst` (the file that exists),
producing `$(B)/<module>/urlrules.inc` with ordinary source inputs.

Reference (sg7, `kernel/hosts/owner/ya.make`): `SRCDIR(yweb/urlrules)` +
`ARCHIVE(NAME urlrules.inc areas.lst 2ld.list ungrouped.list)`. The AR node must
read `$(S)/yweb/urlrules/areas.lst` etc., not the phantom
`$(S)/kernel/hosts/owner/areas.lst`.

## Divergence (reproduced)

`emit_archives.go` builds each member path as
`source(instance.Path.rel() + "/" + f)` — module dir + file name, ignoring
SRCDIR. For `kernel/hosts/owner` this yields `$(S)/kernel/hosts/owner/areas.lst`,
which does not exist; sandboxing content-hashing then faults
`open kernel/hosts/owner/areas.lst: no such file`, blocking direct sg7 before
divergence classification.

## Files touched

- `emit_archives.go` — replace the source-file branch
  `absVFS = source(rel)` with the existing SRCDIR-aware helper
  `resolveSourceVFS(ctx, instance, f, d.srcDirs)`. Thread `ctx` into
  `emitArchive`. The PR-produced ($(B)) branch is unchanged.
- `emit_ar_test.go` — regression: a fixture module with `SRCDIR(data)` and
  `ARCHIVE(NAME out.inc member.txt)`, asserting the AR cmd args and Inputs use
  `$(S)/data/member.txt` and never the phantom `$(S)/<mod>/member.txt`.

`go test ./...` and `./dev/validate.py` pass; isolated kernel/hosts/owner graph
no longer opens `kernel/hosts/owner/areas.lst`; direct sg7 no longer reports

Net change is small and reuses the existing resolution helper.
