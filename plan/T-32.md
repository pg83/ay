# T-32 plan: honor explicit PROTO_LIBRARY(name) when naming py-proto global archives

## Upstream mechanism

In ymake, a module's global archive basename is `$MODULE_PREFIX$REALPRJNAME.global.a`.
`REALPRJNAME` is set from the module statement's explicit name argument when present
(`PROTO_LIBRARY(ads-caesar-events-proto)` -> `REALPRJNAME=ads-caesar-events-proto`),
otherwise it defaults to the path-derived form (the last path components joined by `-`).
For the py side of a PROTO_LIBRARY, `MODULE_PREFIX=libpy3`, so the global archive is
`libpy3<REALPRJNAME>.global.a`.

Our code already reproduces this for:
- the regular C++ proto archive: `emit_proto.go:745` reads `d.moduleStmt.Args[0]`
  and feeds it to `archiveNameWithPrefixOrName(..., "lib", protoLibName)`.
- the objcopy/header-only global archive: `gen.go:976` reads `d.moduleStmt.Args[0]`.

The py-proto global archive in `emit_py_proto.go:98` hardcodes the explicit-name
argument to `""`, so it always falls back to the path-derived name
(`libpy3libs-events-proto.global.a`) instead of the explicit
`libpy3ads-caesar-events-proto.global.a` / `libpy3limbert-events-proto.global.a`.

## Files to touch

- `emit_py_proto.go`: in `emitPyProtoSrcs`, read the explicit module name from
  `d.moduleStmt.Args[0]` (when present) and pass it as the `name` argument of
  `globalArchiveNameWithPrefixOrName`, exactly as the C++ archive path does.
- `emit_proto_test.go`: add a focused test — a py-addressed `PROTO_LIBRARY(named)`
  produces `libpy3named.global.a`, while an unnamed `PROTO_LIBRARY()` keeps the
  path-derived `libpy3<dir-tail>.global.a`.

## Expected effect on the gate

In the fs_codegen representative pair the three `libpy3libs-...global.a` ours-only
tokens and the matching `libpy3ads-caesar...` / `libpy3limbert...` ref-only tokens
disappear. sg2..sg6 stay byte-exact; sg7 matched increases / gap shrinks.

## Scope guard

Only the py-proto global archive name source changes. No change to C++ archive
naming, feature-store resource reachability, T-29 proto root prefixes, or T-30
descriptor SBOM inputs.
