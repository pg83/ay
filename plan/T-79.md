# T-79 ŌĆö ALL_RESOURCE_FILES directory-glob (`dir/*`) residual

## Classification

**Missing expansion.** The representative right-only node
`$(B)/yabs/models_services/feature_store/lib/py3/objcopy_*.o` is missing because the
module's transitively-included `split_configs/ya.inc` declares

```
ALL_RESOURCE_FILES(json PREFIX ${MODELS_PATH} ${ARCADIA_ROOT}/${MODELS_PATH}/*)
```

i.e. the DIR token itself ends in a `*` wildcard segment. Our current
`expandAllResourceFiles` / `allResourceDir` / `globDirSuffix` treat every DIR token
as a single literal directory and list it once. A DIR ending in `/*` (or containing
any `*` segment) is not a real directory, so `isDir` returns false and **zero**
resources are emitted ŌĆö the whole `dir/*/*.json` match set (224 subdirs ├Ś ~440
`.json` files under `split_configs`) is dropped, and the objcopy chunk nodes that
would carry those resfs members never appear in our graph.

## Upstream mechanism

`build/ymake.core.conf`:

```
macro ALL_RESOURCE_FILES(EXT, PREFIX="", STRIP="", DIRS...) {
    _GLOB(ALL_RES_$EXT ${suf=/*.$EXT:DIRS} ...)
    _ARF_HELPER(... ${pre=$ALL_RES_:EXT})
}
```

So per DIR the glob pattern is literally `DIR + "/*." + EXT` (FROM_DIRS: `DIR + "/*"`).
The DIR may itself contain `*`, producing a multi-segment glob
`ads/.../split_configs/*/*.json`.

`devtools/ymake/symbols/globs.cpp` `TGlobPattern::Apply` parses the pattern into
segments (`Fixed` | `Anything`=`*` | `Pattern`=`*foo` | `Recursive`=`**`) and walks a
directory frontier segment by segment:
- `Fixed`: descend into that subdir if it exists.
- `Anything`/`Pattern`: `ListDir` each frontier dir (entries are **sorted** by
  basename via `SortedReadDir_`), keep matching subdirs for non-final segments and
  matching files for the final segment.

Match order is therefore *sorted-subdir-then-sorted-file*, i.e. nested iteration over
the sorted directory frontier. The objcopy filename hash sorts its inputs
(`objcopyHash`), so the resfs **set** is what matters for the key; order matters only
for the rendered command/archive-member sequence, which follows this traversal order.

Each match reconstructs to `${ARCADIA_ROOT}/<arc-rel>` (literal marker, as today),
fed to `RESOURCE_FILES` with the existing PREFIX/STRIP handling ŌĆö unchanged.

## Change

Generalize the single-directory glob into a generic multi-segment glob walker that
reproduces `TGlobPattern::Apply` for the segment types we encounter (`Fixed`,
`Anything`, `Pattern`). The simple `dir/*.ext` case collapses to a special case of
the same walk, so `globDirSuffix` is replaced and `allResourceDir` is simplified to
return just the cleaned source-relative DIR (which may contain `*`).

- `resource.go`:
  - `allResourceDir(modulePath, dir) -> (rel string, ok bool)` ŌĆö root/clean/reject as
    today; `*` survives `path.Clean` as an ordinary segment.
  - new `globMatch(fs, pattern) []string` ŌĆö segment walk returning matched file
    rel-paths in upstream traversal order; sorted listings via `listdir`.
  - `expandAllResourceFiles`: build `pattern = rel + "/*" + suffix` per DIR, run
    `globMatch`, append `${ARCADIA_ROOT}/<match>` to `rfArgs`.
  - delete `globDirSuffix`.
