# T-125 plan: YTPROF_BUILD_TYPE adjacent quoted-suffix flag tokenization

## Upstream mechanism

Upstream tokenizes ya.make macro arguments with the Ragel grammar
`devtools/ymake/lang/makelists/makefile_lang.rl6`:

```
str_sq_char = any - '\'' - '\\';
str_sq = '\'' str_sq_char* ('\\' any str_sq_char*)* '\'';
str = (str_dq | str_sq) >start_quoted_str %end_quoted_str;

escaped = ('\\' any);
allowed_sym = any - space - '(' - ')' - '#' - '\\';
atom = ( ( escaped | (allowed_sym - '"' - '\'') | ('$(' allowed_sym+ ')') ) (allowed_sym | escaped)* ) >start_str %end_str;

list_elem = (str | atom) %push_list_elem;
list = spaces ((list_elem | comment) spaces)** >clear_list;
```

Key fact: a list element is `str | atom`, separated by `spaces`. Only the FIRST
character of an `atom` excludes `'` and `"` (`allowed_sym - '"' - '\''`). Every
following character is `allowed_sym | escaped`, and `allowed_sym` (= any minus
space/paren/`#`/backslash) INCLUDES quotes. Therefore once an atom begins with a
non-quote byte, it greedily consumes embedded quoted segments — there is no token
boundary at an interior quote when there is no whitespace.

So `-DYTPROF_BUILD_TYPE='\"${BUILD_TYPE}\"'` is a SINGLE upstream argument: the
atom starts at `-`, then consumes `'`, `\"` (escaped), `${BUILD_TYPE}`, `\"`, `'`.
Upstream keeps the atom raw and removes the `'` delimiters and `\` escapes
downstream when forming the command, yielding `-DYTPROF_BUILD_TYPE="DEBUG"`.

## ay defect

`yamake.go` lexer splits adjacent word+quote into two tokens:
- `readWord`/`readIdentOrWord` stop at a bare `'`/`"` (not in `isWordByte`).
- `readToken` then dispatches the `'` to `readString`, producing a second token.
- `parseMacroArgs` appends each token as a separate macro argument.

For `-DYTPROF_BUILD_TYPE='\"${BUILD_TYPE}\"'` ay produces two args
`-DYTPROF_BUILD_TYPE=` and `\"${BUILD_TYPE}\"`, then `unescapeFlag` +
`expandStmtTokensSTR` yield two command tokens `-DYTPROF_BUILD_TYPE=` and
`"DEBUG"` instead of the single upstream token `-DYTPROF_BUILD_TYPE="DEBUG"`.

## Change

Reproduce the upstream atom rule: a word token continues across an adjacent
(no-whitespace) quoted segment. In `readWord` and `readIdentOrWord`, when a bare
`'`/`"` is seen with a non-empty buffer, consume the quoted segment (matching
`readString`: strip delimiters, preserve `\\`-escapes) and continue scanning the
same token. A token that STARTS with a quote still routes through `readToken` →
`readString` unchanged, so standalone quoted args are unaffected.

Parse/expand path after the change:

```
source token:   -DYTPROF_BUILD_TYPE='\"${BUILD_TYPE}\"'
parser arg:     -DYTPROF_BUILD_TYPE=\"${BUILD_TYPE}\"   (single-quote delims stripped)
unescape flag:  -DYTPROF_BUILD_TYPE="${BUILD_TYPE}"
env expansion:  -DYTPROF_BUILD_TYPE="DEBUG"
```
