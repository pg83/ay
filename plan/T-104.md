## Upstream mechanism

`_CPP_PROTO_CMDLINE_BASE` (ymake.core.conf:614):

```
$PROTOC -I=./$PROTO_NAMESPACE -I=$ARCADIA_ROOT/$PROTO_NAMESPACE ${pre=-I=:_PROTO__INCLUDE} -I=$ARCADIA_BUILD_ROOT -I=$PROTOBUF_INCLUDE_PATH ...
```

`${pre=-I=:_PROTO__INCLUDE}` renders the `_PROTO__INCLUDE` set **verbatim**, no
dedup against the structural prefix. `_PROTO__INCLUDE` is the `FOR proto` GLOBAL
ADDINCL closure (proto.conf:117-119 `PROTO_ADDINCL` → `_ORDER_ADDINCL ... FOR
proto $(S)/$Path`). A peer that re-declares the same `PROTO_NAMESPACE`
contributes its own `FOR proto $(S)/<ns>` addincl, which lands in the set at the
**peer's encounter position** — not adjacent to the module's own contribution.

## Defect in our code

`composePBArgBlocks` (emit_pb.go) renders the band with:
1. a structural `-I=$(S)/cppOutRoot` (the module's own `_PROTO__INCLUDE` entry), then
2. `if duplicateOutputRootInclude { mid = append(mid, -I=$(S)/cppOutRoot) }` — a
   second own copy hoisted **adjacent** to (1), then
3. the `protoInclude` band loop that **dedups** each token against `mid`
   (`slices.Contains`), which removes the peer-redeclared own from its natural
   encounter position.

So the peer-redeclared own is stripped from its position (3) and re-emitted in
the wrong place (2). `peerContribs.protoInclude` already carries the
peer-redeclared own at the correct position; the dedup+hoist destroys it.

## Fix

Render the band verbatim (drop the `slices.Contains` dedup and the
`duplicateOutputRootInclude` hoist). The structural `-I=$(S)/cppOutRoot` arm
keeps emitting the module's own contribution; the peer-redeclared own rides the
band at its encounter position, exactly as upstream.

For this to be correct the band input must be **peers-only** (own supplied by
the structural arm):
- PROTO_LIBRARY path (emit_proto.go:641) already passes `peerContribs.protoInclude`
  (peers-only) — no change.
- LIBRARY-hosted path (emit_proto.go:878) passes `in.ProtoInclude`
  (`effectiveProtoInclude`, own prepended). Thread the peers-only set
  (`peerProtoInclude`) to the command band instead, via a new
  `ModuleCCInputs.ProtoIncludePeers`. The reported `ModuleEmitResult.ProtoInclude`
  stays own-prepended for consumers.

EV/cfgproto/gzt (`emitProtoWrapperPBNode`) keep `effectiveProtoInclude` + their
own `evPeerProtoIncludes` dedup — they have no structural own arm
(`evProtocConstHead` uses empty-namespace `-I=./ -I=$(S)/`), so own must remain
in their band. Not touched.
