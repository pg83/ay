# T-124 plan: FROM_SANDBOX `--rename RESOURCE` command emission parity

## Upstream mechanism

`build/ymake.core.conf:4895` FROM_SANDBOX `.CMD`:

```
... --resource-id $Id $FILE $PREFIX ${pre=--rename :RENAME} $EXECUTABLE -- $OUT $OUT_NOAUTO ...
```

`${pre=--rename :RENAME}` renders, for every item in the `RENAME[]` list, the
pair `--rename <item>`, positioned after `$PREFIX` and before `$EXECUTABLE`/`--`.
ay currently parses `RENAME` into a `SKIP` section (discards it),
`FromSandboxStmt` has no rename storage, and `emit_from_sandbox.go` emits no
rename tokens. The six live sg7 SB residuals are all
`FROM_SANDBOX(FILE ... RENAME RESOURCE ...)` with ref-only `--rename RESOURCE`.

Generic `RENAME[]` list (not a `RESOURCE` special-case), matching upstream.

## Files touched

- `yamake.go`: add `Renames []STR` to `FromSandboxStmt`; in `parseFromSandbox`
  route `RENAME` to a `"RENAME"` section that appends tokens to `stmt.Renames`
  (keep `INDUCED_DEPS` skipped).
- `modules.go`: expand `expanded.Renames = expandStmtTokensSTR(v.Renames, env)`.
- `emit_from_sandbox.go`: after prefix, before `--executable`/`--`, append
  `--rename`, `<item>` for each rename token. Nothing else changes.
- `emit_from_sandbox_test.go`: fail-first `TestGen_FromSandboxRenameResourceCommand`
  covering OUT_NOAUTO and auto OUT variants, asserting the SB command contains the
  subsequence `--copy-to-dir . --rename RESOURCE -- <out>` and that inputs stay
  exactly the three explicit scripts.

## Expected gate effect

sg7: the six SB `cmds:6` divergences resolve; `--rename`/`RESOURCE` ref-only
tokens disappear from the grouped `kind=SB` token report; named representative
pairs print header-only; matched count rises by ~6; sg2–sg6 stay byte-exact; no
AR/LD residual change.
