# T-7 plan: Cython C-mode naming and extra generated headers (gevent/lxml)

## Upstream mechanism

`PY_SRCS` is processed by `build/plugins/pybuild.py::onpy_srcs`. The cython
directives select which output list a following source is appended to:

```
pyxs = pyxs_cpp        # default
CYTHON_C        -> pyxs = pyxs_c          (-> on_buildwith_cython_c_dep)
CYTHON_C_H      -> pyxs = pyxs_c_h        (-> on_buildwith_cython_c_h)
CYTHON_C_API_H  -> pyxs = pyxs_c_api_h    (-> on_buildwith_cython_c_api_h)
CYTHON_CPP      -> pyxs = pyxs_cpp        (-> on_buildwith_cython_cpp_dep)
CYTHON_CPP_H    -> pyxs = pyxs_cpp_h      (-> on_buildwith_cython_cpp_h)
CYTHONIZE_PY    -> cythonize_py = True  (does NOT change `pyxs`)
```

Crucially `CYTHONIZE_PY` only flips a flag; `.py` sources are appended to
**whatever `pyxs` is currently selected**. In gevent the order is
`CYTHON_C` … `CYTHONIZE_PY` …, so `_abstract_linkable.py` lands in `pyxs_c` and
is emitted in **C mode** as `_abstract_linkable.py.c` (not `.py.cpp`).

Output naming (`ymake.core.conf`):

- DEP variants (`_BUILDWITH_CYTHON_C_DEP`, `_..._CPP_DEP`):
  `-o ${tobindir;output;suf=${OBJ_SUF}${CYTHON_SUFFIX}.c:Src}` — **no `noext`**,
  so the `.pyx`/`.py` extension is kept: `objectify.pyx` -> `objectify.pyx.c`,
  `_abstract_linkable.py` -> `_abstract_linkable.py.c`.
- Header variants (`_BUILDWITH_CYTHON_C_H`, `_..._C_API_H`, `_..._CPP_H`):
  `-o ${...;suf=${CYTHON_SUFFIX}.c;noext:Src}` plus
  `${hide;addincl;...;suf=${CYTHON_SUFFIX}.h;noext:Src}` and (API variant)
  `${hide;addincl;...;defext=${CYTHON_SUFFIX}_api.h;noext:Src}`. `noext` strips
  the source extension, so `etree.pyx` -> `etree.c` + `etree.h` + `etree_api.h`.

For PY3_LIBRARY (gevent, lxml) `OBJ_SUF` and `CYTHON_SUFFIX` are both empty.

## Current divergence (sg7, from `.out/digger-validate`)

- gevent `_abstract_linkable.py`: we emit CY output `…/_abstract_linkable.py.cpp`
  with `--cplus`; ref emits `…/_abstract_linkable.py.c` (C mode, no `--cplus`).
  Root: our `CYTHONIZE_PY` `.py` branch creates a `CythonStmt` with `CMode=false`,
  ignoring the preceding `CYTHON_C`.
- lxml `etree.pyx` (`CYTHON_C_API_H`): we emit a single CY output
  `…/etree.pyx.c`; ref emits `…/etree.c` + `…/etree.h` + `…/etree_api.h`.
  Root: we treat `CYTHON_C_H`/`CYTHON_C_API_H` as plain C mode (a stub) — no
  `noext` naming and no companion-header outputs.
- objectify.pyx (`CYTHON_C`, DEP variant) already correct: `objectify.pyx.c`.

Object paths (`…/_/…`) and the consuming compile/archive members are derived from
the CY output name by `emitCC`, so fixing the producer name cascades correctly
(ref archive lists `…_/gevent/_abstract_linkable.py.c.pic.o`).

## Files to touch

  header, api-header) instead of just `cythonCMode`; have the `CYTHONIZE_PY`
  `.py` branch and the `.pyx` branch carry that variant onto the `CythonStmt`.
  Add `CYTHON_CPP_H` for symmetry.
- `emit_cython_cpp.go`: extend `CythonStmt` with `Header`/`ApiHeader`; compute
  `noext` generated name for header variants; emit companion `.h`/`_api.h`
  outputs and register them in the codegen registry.
- `emit_cython_cpp_test.go`: regression tests (a) `.py` CYTHONIZE_PY after
  `CYTHON_C` -> `.py.c` in C mode; (b) `.pyx` with `CYTHON_C_API_H` -> `.c` +
  `.h` + `_api.h`.
