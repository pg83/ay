# T-57 — `USE_ARCADIA_LIBM` implicit peer (`contrib/libs/libm`)

## Upstream mechanism

`build/ymake.core.conf` `module _BASE_UNIT` (lines 933-945), evaluated for every unit:

```
when ($OS_EMSCRIPTEN == "yes") {
    # Do nothing.
}
otherwise {
    when ($USE_ARCADIA_LIBM == "yes") {
        PEERDIR+=contrib/libs/libm
    }
    when ($USE_ARCADIA_LIBM == "no") {
        C_SYSTEM_LIBRARIES+=-lm
    }
}
```

`USE_ARCADIA_LIBM` defaults to `no` (`ymake.core.conf:368`, re-`SET` at `:4388`), so the
implicit `contrib/libs/libm` peer appears only when a module runs
`ENABLE(USE_ARCADIA_LIBM)` and the target platform is not Emscripten. The system
`-lm` branch is out of scope for this ticket (no Emscripten platform exists in our
build, so the `OS_EMSCRIPTEN` gate is always the `otherwise` arm).

Representatives (`/home/pg/monorepo/4`):
- `yabs/server/cs/tools/cs/ya.make:46` — `IF(NOT MUSL) ENABLE(USE_ARCADIA_LIBM) ENDIF()`
- `yabs/server/daemons/bs_static/ya.make:20` — same guard.

The `IF(NOT MUSL)` guard is already honoured by our `IfStmt`/`evalCond` machinery,
so a non-musl effective env reaches the `ENABLE`, which sets the env var to `yes`
(`modules.go` `tokEnable` → `env.setBool(internEnv("USE_ARCADIA_LIBM"), true)`).

## Current gap

`defaultPeerdirsForWithState` (defaults.go) models the other `_BASE_UNIT`/`_LINK_UNIT`
implicit peers (linux-headers, cxxsupp, util, musl, sanitizer, build/platform) but has
no `USE_ARCADIA_LIBM` rule. The effective env value computed during `collectModule`
is discarded after collection (only specific vars — `MUSL`, `NO_STRIP`, … — are
captured into `ModuleData`). Result: ref reaches
`$(B)/contrib/libs/libm/libcontrib-libs-libm.a` and its object members (99 right-only
sg7 nodes per T-53) while ay does not.

## Change

Reuse the existing env→ModuleData capture seam (the same pattern as `d.muslEnabled`):

1. `env_consts.go` — intern `envUSE_ARCADIA_LIBM` and `envOS_EMSCRIPTEN`.
2. `modules.go` `ModuleData` — add `useArcadiaLibm bool`; set it in `collectModule`
   from `env.bool(envUSE_ARCADIA_LIBM) && !env.bool(envOS_EMSCRIPTEN)`.
3. `defaults.go` `defaultPeerdirsForWithState` — when `d.useArcadiaLibm` and the
   module is not `contrib/libs/libm` (or a descendant), append `contrib/libs/libm`
   to the CPP peer list. Self-peer guarded exactly like the cxxsupp/util defaults.

No new data path: the value rides the env already threaded through `collectModule`,
captured into `ModuleData` like every other effective-env flag. Order within the peer
list is not tuned here — T-53 defers bs_static/cs link-order drift to a separate class.
