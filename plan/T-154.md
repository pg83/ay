# T-154 plan: R5 rlgen mode follows the per-platform Ragel optimized flag

## Upstream mechanism

`build/ymake_conf.py:Ragel.set_default_flags(optimized)` sets BOTH ragel flag
groups from ONE boolean:

```python
if optimized:
    self.rlgen_flags.append('-G2')   # consumed by rlgen-cd (R5)
    self.ragel6_flags.append('-CG2') # consumed by ragel6  (R6)
else:
    self.rlgen_flags.append('-T0')
    self.ragel6_flags.append('-CT0')
```

`optimized = build.is_release and not build.is_sanitized` (GnuCompiler).

So `-G2`/`-T0` (R5 rlgen mode) and `-CG2`/`-CT0` (R6 mode) derive from the SAME
`optimized` value. A module reachable in two toolchain contours appears twice:
the host/tool contour (release → optimized → `-G2`/`-CG2`) and the target
contour (here debug → `-T0`/`-CT0`). Same output paths, command differs only in
the rlgen mode.

Confirmed in the sg7 reference (`/home/pg/monorepo/4/sg7.json`):

- R5: `(-T0, tags=())` ×1 + `(-G2, tags=('tool',))` ×1 — same two outputs
  `$(B)/kernel/urlnorm/urlhashval.rl.tmp` + `.rl5.cpp`.
- R6: `(-CT0, tags=())` ×13 + `(-CG2, tags=('tool',))` ×14.

`canonContent` strips `tags`, so the two R5 producers normalize to identical
content EXCEPT the rlgen mode (`-G2` vs `-T0`).

## Current local state and the defect

- R6 (`emit_r6.go:63`) already reads `instance.Platform.Ragel6Optimized` and emits
  `-CG2`/`-CT0` — matches both contours (R6 shows zero divergence in T-153).
- R5 (`emit_r5.go:31`) HARDCODES `argG2` (`-G2`). So the host (optimized) and
  target (debug) instances of `kernel/urlnorm` BOTH emit `-G2`; being
  byte-identical they dedup to ONE node. We keep the `-G2` one (matches the ref
  tool node) and are missing the `-T0` target node → the T-153 "ref-only
  duplicate R5 producer" residual.

The codegen registry is per-scanner (`scannerFor` → host/target, two registries),
so each contour registers its own producer in its own registry — no
duplicate-owner conflict arises, exactly as for R6 today. Nothing about
registration changes.

## Change

1. `arg_consts.go`: add `argT0 = internArg("-T0")`.
2. `platform.go` + `emit_r6.go`: rename `Ragel6Optimized` → `RagelOptimized`
   (it drives both R5 and R6, matching upstream's single `optimized` boolean).
3. `emit_r5.go:emitR5`: select `argG2` when `instance.Platform.RagelOptimized`,
   else `argT0`, instead of the hardcoded `argG2`.

No change to output paths, registration, downstream `.rl5.cpp.o` compile inputs,
AR/LD ordering, or R6/proto/source behavior. The target `.o` now depends on the
target `-T0` R5 producer (same output path, same producer-ref edge mechanism);
normalized `.o` content is unchanged.
