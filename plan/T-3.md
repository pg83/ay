# T-3 ŌĆö Bison-generated CC nodes miss generated-proto producer deps

## Symptom (gate)
sg7 residual is deps/uid-only: `matched=69107, our_only=3177, ref_only=3177`.
Three bison/ypp parser compile roots show deps-only gaps vs upstream:
- `$(B)/kernel/remorph/tokenlogic/parser.y.cpp.o` (+ `.pic.o`)
- `$(B)/yt/yt/library/query/base/parser.ypp.cpp.pic.o`

Upstream records additional `deps` on generated proto (`.pb.*`) producers on
these CC nodes; ours does not.

## Upstream mechanism
In ymake every C/C++ compile node depends on the producer of any *build*
(`$(B)`) header reachable through its include closure ŌĆö the flat dep model.
A bison-generated source (`parser.y.cpp`) whose `%{ ŌĆ” %}` prologue includes a
header whose transitive closure reaches a generated `<ŌĆ”>.pb.h` therefore gains
a dep edge to that `.pb.h`'s protoc/CPP_PROTO producer node, exactly like a
hand-written `.cpp` that includes the same `.pb.h`.

## Local mechanism that already implements this
`resolveCodegenDepRefs(ctx, instance, includeInputs, excludeŌĆ”)` (gen.go:372)
maps every `$(B)` include in a CC node's closure to its registered
`GeneratedFileInfo.ProducerRef` and returns the deduped producer NodeRefs.
Every generated-source CC emitter routes its walked closure through it:
- handwritten CC: `emit_cc.go:620`
- flex `.l`: `emit_flex.go:62`
- ragel r5/r6: `emit_r5.go:77`, `emit_r6.go:134`
- `.ev`: `emit_ev.go:273`
- cython: `emit_cython_cpp.go:411`
- configure-file: `emit_cf.go:55`

`emitBisonY` (emit_bison_y.go:154) is the **sole** generated-source CC emitter
that does NOT. It sets `ccIn.ExtraDepRefs = []NodeRef{ycRef}` ŌĆö only the bison
producer itself ŌĆö and never resolves codegen dep refs from the walked closure.
So when the parser source reaches a generated `.pb.h`, the protoc producer dep
is dropped. This is the deps-only gap.

## Fix (one line, mirrors siblings)
In `emitBisonY`, after `ccIn.IncludeInputs = walkClosure(...)`:

```go
ccIn.ExtraDepRefs = append([]NodeRef{ycRef},
    resolveCodegenDepRefs(ctx, instance, ccIn.IncludeInputs, ycRef)...)
```

`ycRef` is passed as the `exclude` so the bison producer is not duplicated.
No new data path, no second parse, no context-dependent resolution ŌĆö it reuses
the same closure already walked one line above and the same registry every
other CC emitter consults.

## Files touched
- `emit_bison_y.go` ŌĆö the one-line ExtraDepRefs change.
- `emit_bison_y_test.go` ŌĆö new reproduction test.

## Test (fails before, passes after)
A LIBRARY with `SRCS(foo.proto parser.y)` where `parser.y`'s prologue includes
the generated `foo.pb.h`. Assert the generated parser CC node
(`ŌĆ”/parser.y.cpp.o`) has the foo.pb.h protoc producer node in its `graphDeps`.
Pre-fix: the dep is absent (the closure carries `foo.pb.h` as an *input* but no
producer dep edge). Post-fix: present.

## Expected gate effect
The three parser compile roots lose their deps-only gaps; sg7 `matched`
increases, `our_only`/`ref_only` decrease by the per-root proto-producer dep
count. sg2ŌĆōsg6 stay byte-exact (no bison-into-proto case there, and the change
is additive only when the closure reaches a registered `$(B)` producer).
No perf change: the closure is already walked; this adds one
`resolveCodegenDepRefs` pass over it (same as every sibling emitter).
