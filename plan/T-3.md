# T-3 — BISON .y/.ypp output shape and include propagation

## Symptom (sg7, xfail)

The yt query base lexer pair reports a missing `parser.h` closure and missing
`-I$(B)/yt/yt/library/query/base`, and the kernel qtree/remorph bison `.cpp`
outputs sit under an extra `_/` directory where upstream uses the flat module
build directory.

## Upstream mechanism (yatool/build/conf/bison_lex.conf)

`_SRC("y")` and `_SRC("ypp")` are **the same C++ bison family** — identical
`.CMD`/`.SEM`. Both emit:

- generated source: `-o ${output;suf=$_BISON_GEN_EXT;nopath:SRC}`
- generated header: `--defines=${main;addincl;norel;output;suf=.h;nopath;noext:SRC}`

ymake's auto-output placement (verified against the sg7 reference graph):

| source layout | generated `.cpp` | object | header |
|---|---|---|---|
| **flat** `req_pars.y` in module `kernel/qtree/request` | `$(B)/<mod>/req_pars.y.cpp` | `$(B)/<mod>/req_pars.y.cpp.o` | `$(B)/<mod>/req_pars.h` |
| **subdir** `pire/re_parser.y` in module `library/cpp/regex/pire` | `$(B)/<mod>/_/pire/re_parser.y.cpp` | `$(B)/<mod>/_/_/pire/re_parser.y.cpp.o` | `$(B)/<mod>/pire/re_parser.h` |

So the `_/` rebase appears **only when the source lives in a subdirectory** of
its module — exactly the rule already used by `ragel6OutVFS` and by the regular
CC object-path logic in `composeCCPaths` (`_/` only when `srcRel` contains `/`).

The current `emitBisonY` prepends `_/` **unconditionally**, so the flat case
(`req_pars.y`, `parser.ypp`) wrongly lands at `$(B)/<mod>/_/req_pars.y.cpp`
(and `$(B)/<mod>/_/_/req_pars.y.cpp.o`). The subdir case is already correct.

`.ypp` is additionally not dispatched at all: `emitOneSource` only matches
`.y`, and `classifySrcExt`/`isCodegenProducingSrc(ID)` only key on `.y`, so a
module with `parser.ypp` produces no bison node — hence the missing `parser.h`
closure and missing `-I$(B)/<mod>` on the sibling `lexer.rl6`. `.ypp` already
has a CIncludeDirective parser registered (parsers_generated.go:41).

## Changes

1. **emit_bison_y.go** — make the `_/` prefix conditional on a subdir, mirroring
   `ragel6OutVFS`: `generatedRel := srcRel + genExt`; prepend `_/` only when
   `strings.Contains(srcRel, "/")`.

2. **emit_sources.go** — dispatch `.ypp` to `emitBisonY` alongside `.y`.

3. **str_intern.go** `classifySrcExt` — map `.ypp` → `srcExtY` (so `hasBisonY`,
   the own-header addincl registration in modules.go, the codegen pass gate
   `isCodegenProducingSrcID`, and AR ordering all treat `.ypp` as bison,
   identical to `.y`).

4. **gen.go** `isCodegenProducingSrc`, **emit_copy.go**
   `isSourceEligibleForCopyAuto` — add the `.ypp` string suffix to the two
   remaining string-keyed lists for parity with the id-space class.

## Tests (test-first)

- `.ypp`: module with `parser.ypp` + sibling `lexer.rl6` including `<parser.h>`;
  assert flat `$(B)/<mod>/parser.h` & `$(B)/<mod>/parser.ypp.cpp`, flat object
  `$(B)/<mod>/parser.ypp.cpp.o`, lexer CC carries `-I$(B)/<mod>` and the
  `$(S)/<mod>/parser.ypp` source input.
- flat `.y`: module with flat `req_pars.y`; assert generated source and object
  are flat (no `_/`). Fails on current code (produces `_/req_pars.y.cpp` /
  `_/_/req_pars.y.cpp.o`).
- existing subdir tests (`genlib/pire/re_parser.y`) must stay green — the `_/`
  rebase is preserved for subdir sources.

## Out of scope

`BISON_FLAGS(-Wcounterexamples)` (yt query base ya.make) is a separate
macro→flag plumbing concern, not output shape / include propagation. The flat
`.y` cases use plain `-v`, matching our emitter. Flagged to the team, not
implemented here.

## Expected gate effect

sg7 (xfail): bison `.cpp`/object outputs move from `_/`-nested to flat for the
flat sources; `parser.ypp` node now exists; `lexer.rl6.cpp` CC gains the
`parser.h` closure and `-I$(B)/yt/yt/library/query/base`. sg2–sg6 contain no
bison sources → byte-exact unchanged. No hot-path change → no perf regression.
