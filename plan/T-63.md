# T-63 plan: uppercase `.C` source = C++ source emission

## Upstream mechanism

`/home/pg/monorepo/yatool/build/ymake.core.conf:3466-3470` defines

```
macro _SRC("C", SRC, SRCFLAGS...) {
    .CMD=$_SRC_cpp($SRC $SRCFLAGS)
    ...
}
```

i.e. uppercase `.C` routes through `$_SRC_cpp`, identical to `_SRC("cpp")`,
`_SRC("cc")`, `_SRC("cxx")`. Lowercase `_SRC("c")` (3473) uses `$_SRC_c` with
`USER_CONLYFLAGS`. So uppercase `.C` is a C++ source: compiled with the C++
compiler, with the C++ standard tail, and participating in ordinary and `GLOBAL`
archive membership like every other C++ extension.

This codebase already honors the same case distinction for headers:
`gen.go:isHeaderSource` lists both `.h` and uppercase `.H`. Source dispatch only
lists lowercase `.c`/`.cpp`/`.cc`/`.cxx`, so `.C` falls through to
`WarnUnsupportedSource` and the object/archive nodes are never emitted.

## Local cause

- `emit_sources.go:35-38` dispatches `.c`, `.cpp`, `.cc`, `.cxx` to
  `emitLibraryCSource`, but not `.C`.
- `emit_cc.go:isCxxSource` (371) recognizes `.cpp`, `.cc`, `.cxx`, but not `.C`,
  so even if dispatched it would pick the C compiler / C tail.

Effect: `yabs/server/phantom/pd/base` and `.../pd/pi` (uppercase `.C`, mostly
`GLOBAL`) emit no object/archive nodes; downstream
`$(B)/yabs/server/daemons/bs_static/yabs-server` misses those phantom archive
inputs.

## Change

1. `emit_sources.go`: add `strings.HasSuffix(srcRel, ".C")` to the
   `emitLibraryCSource` dispatch case.
2. `emit_cc.go:isCxxSource`: add `strings.HasSuffix(srcRel, ".C")`.

`strings.HasSuffix` is case-sensitive, so `.C` does not collide with `.c`,
and `.cc`/`.cpp`/`.cxx` paths are untouched.

## Tests (fail-first)

- `LIBRARY()` with `SRCS(regular.C)` and `GLOBAL_SRCS(global.C)`:
  - a non-global `.C` object + regular archive member are emitted (CC=2, AR=2);
  - the global `.C` member contributes to the module global archive
    (one `tagGlobal` AR, one no-tag AR);
  - the `.C` compile command uses the C++ compiler and includes the C++
    standard flag.
- Negative guard: `.c`, `.cpp`, `.cc`, `.cxx` keep their current
  classification (`isCxxSource` true for cpp/cc/cxx and false for c; `.C` true).
