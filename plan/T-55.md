# T-55 — Emit `.cfgproto` PB producers and downstream compiles

## Upstream mechanism (verified)

`_SRC("cfgproto")` → `_CPP_CFGPROTO_CMD($SRC)` (ymake.core.conf:3333, proto.conf:494-497):

```
.CMD=$CPP_EV_CMDLINE --plugin=protoc-gen-config=${tool:"library/cpp/proto_config/plugin"} \
     --config_out=$ARCADIA_BUILD_ROOT/$PROTO_NAMESPACE ${hide;kv:"p PB"} ${hide;kv:"pc yellow"}
.PEERDIR=library/cpp/proto_config/codegen library/cpp/proto_config/protos contrib/libs/protobuf
```

`CPP_EV_CMDLINE = $_CPP_PROTO_WRAPPER_BASE --outputs $CPP_EV_OUTS -- $_CPP_PROTO_CMDLINE_BASE`
(ymake.core.conf:616). `CPP_EV_OUTS = ${norel;output;suf=.pb.cc:File} ${norel;output;suf=.pb.h:File}`
— **keeps the source extension** (`backend_config.cfgproto.pb.{cc,h}`), exactly like `.ev`.
The difference from `.ev`: the trailing opts are the proto_config plugin
(`--plugin=protoc-gen-config=…/plugin --config_out=$(B)/`) instead of
`$CPP_EV_OPTS` (event2cpp); KV.P is `PB` not `EV`. The `--cpp_out`/styleguide
base is identical.

Verified against the sg7 reference producer node for
`$(B)/balancer/kernel/client_request/backend_config.cfgproto.pb.h`:
command head/tail are **byte-identical to our existing EV emitter** (emit_ev.go
`evProtocConstHead`/`evProtocConstTail`), then `[styleguide-plugin, <src>,
--plugin=protoc-gen-config=$(B)/library/cpp/proto_config/plugin/plugin,
--config_out=$(B)/]`. Inputs: the `.cfgproto` source, cpp_proto_wrapper.py,
three `descriptor.proto` variants (closure), `extensions.proto` (import),
protoc, cpp_styleguide, proto_config/plugin/plugin. 3 ForeignDepRefs.

Generated `.cfgproto.pb.h` is a standard protoc `--cpp_out` header (NOT
event2cpp): its include closure matches an ordinary `.pb.h` (extensions.pb.h
from the import, protobuf runtime via protoc GeneratorRef) **plus** the
file-level `option (NProtoConfig.Include) = "<h>"` headers, which the
proto_config plugin inserts at the `"includes"` insertion point
(library/cpp/proto_config/plugin/main.cpp:70-77, `#include <X>`). Verified in
the reference `.cfgproto.pb.cc.o` closure: `util/datetime/base.h`,
`balancer/kernel/client_request/port.h`, `balancer/kernel/ssl/ssl.h`.

   (shares the EV head/tail via a small extracted helper), ordinary-proto-style

   and the downstream `.pb.cc` compile.

   and cfgproto share the head/tail composition (EV output unchanged — verified
   byte-exact by sg2-sg6).

   the skip-with-no-warn stub).
7. `gen.go` — add `.cfgproto` to `isCodegenProducingSrc` so the compiled
   `.pb.cc.o` archives after the regular objects (codegen phase), matching the

8. `arg_consts.go` — `--config_out=$(B)/` and `library/cpp/proto_config/plugin`
