# T-22 — implement BASE_CODEGEN so sg7 reaches toponym_classifier/factors

## Upstream mechanism (source of truth: plan/T-21.md)

`extsearch/geo/meta/rearrs/middle/toponym_classifier/factors` is reference-only
in our sg7 closure not because of `SPLIT_CODEGEN` lowering (T-2 works for reached
modules) but because the *module is never reached*. The reachability edge is:

```
BASE_CODEGEN(kernel/fill_factors_codegen fill_factors)   (in a BC user module)
  -> tool LD $(B)/kernel/fill_factors_codegen/fill_factors_codegen
  -> ordinary PEERDIR closure of kernel/fill_factors_codegen/ya.make (a PROGRAM)
  -> extsearch/geo/meta/rearrs/middle/toponym_classifier/factors
  -> SPLIT_CODEGEN(...) in that factors module  (already implemented in T-2)
```

`gen.go` currently lists `BASE_CODEGEN` in `acknowledgedMacros` as a no-op, so we
never emit the BC producer node, never depend on the tool program's LD, and never
traverse that program's PEERDIR closure. Implementing `BASE_CODEGEN` as a
first-class codegen statement — exactly like the existing `SPLIT_CODEGEN` — emits
the BC node, takes a tool dependency on the tool's LD (via `ctx.toolResult`, which
runs `genModule` on the tool and pulls its whole PEERDIR closure into the graph),
and thereby brings the factors module into the target closure through ordinary
dependency traversal.

Upstream macro (`/home/pg/3/build/internal/conf/codegen.conf`):

```
macro BASE_CODEGEN(Tool, Prefix, Opts...) {
    .CMD=${tool:Tool} ${input;suf=.in:Prefix} ${noauto;output;suf=.cpp;nopath:Prefix} ${output;suf=.h;nopath:Prefix} $Opts ${hide;kv:"p BC"} ${hide;kv:"pc yellow"}
}
```

Command: tool binary, `$(S)/<mod>/<prefix>.in`, `$(B)/<mod>/<prefix>.cpp`
(noauto), `$(B)/<mod>/<prefix>.h`, then opts. kv `p=BC`, `pc=yellow`. No
`--cpp-parts`, no numbered parts (that is SC-specific). The `.cpp` is noauto: the
module compiles it only when it declares `SRCS(GLOBAL ${BINDIR}/<prefix>.cpp)`,
through the regular source path picking the BC producer from the codegen registry.

## Files to touch

1. `yamake.go` — `BaseCodegenStmt` type + `stmtMarker` + `parseBaseCodegen`
   (positional `Tool Prefix Opts...`); `case "BASE_CODEGEN"` in `buildStmtFor`.
2. `modules.go` — `baseCodegens []*BaseCodegenStmt` field; collect/expand the
   stmt next to `*SplitCodegenStmt`.
3. `gen.go` — remove `BASE_CODEGEN` from `acknowledgedMacros`; call
   `emitBaseCodegensForAR` next to `emitSplitCodegensForAR`.
4. `emit_base_codegen.go` (new) — emit the BC node (mirrors `emit_split_codegen.go`
   minus numbered parts); register both outputs in the codegen registry with the
   tool LD as generator dep. No auto-compiled outputs are returned.
5. `emit_base_codegen_test.go` (new) — synthetic regression on `MemFS`.

## Test (test-first)

Synthetic fixture per plan/T-21.md: a `consumer` LIBRARY with
`BASE_CODEGEN(tool fill_factors)` + `SRCS(GLOBAL ${BINDIR}/fill_factors.cpp)`;
`tool` PROGRAM `PEERDIR(hidden/factors)`; `hidden/factors` LIBRARY with
`SPLIT_CODEGEN(tool2 factors_gen NHidden)`. Pre-fix: no BC node, `hidden/factors`
absent. Post-fix: BC node for `$(B)/consumer/fill_factors.h` with the right
command + tool LD dep, and — because the tool PEERDIR closure now reaches
`hidden/factors` — the SC producer `$(B)/hidden/factors/factors_gen.h` and its
generated CC consumers exist.

- sg7 gains the `kernel/fill_factors_codegen` LD closure and the

  generated CC, removing them from the reference-only set.
