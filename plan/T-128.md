# T-128 plan: generated-header owner attribution for PY / SC / non-library ARCHIVE

## Upstream mechanism

ymake's `Node2Module` (devtools/ymake/json_visitor.cpp:638-645) attributes a
generated file to the FIRST consumer module that resolves it during include
scanning, not to its producer. ay already reproduces this with
`overrideGeneratedModuleDir` (attribute_generated.go), fed by
`generatedFirstClaim` / `generatedNodeClaim` recorded in the scanner
(scanner.go: `recordFirstClaim` / `recordNodeClaim`). The scanner records these
claims for ANY `CodegenRegistry` output it resolves, regardless of producer node
kind — the claim machinery is already generic.

The only thing that restricts attribution today is the `switch` in
`overrideGeneratedModuleDir`, which whitelists `pkPR`, `pkCF`, `pkCP`. The three
T-119 residuals are produced by node kinds outside that set:

- `pkPY` — `RUN_PYTHON3(... OUT header)` (emit_py_run.go).
- `pkSC` — `*.sc -> *.sc.h` (emit_sc.go).
- non-library `pkAR` — `ARCHIVE(NAME header ...)` generated header product
  (emit_archives.go), distinct from the `lib*.a` library archive (emit_ar.go,
  which carries `module_type=lib`, `module_lang=cpp`).

These producers register their outputs in the `CodegenRegistry`, so consumers
already record first-claims for them; only the final override skips them.

## Change

`attribute_generated.go`: replace the `switch kind { case pkPR, pkCF, pkCP }`
gate with a `generatedOwnerAttributable(node)` predicate that additionally
admits `pkPY`, `pkSC`, and the non-library `pkAR` shape:

```go
func generatedOwnerAttributable(node *Node) bool {
    switch node.KV.P {
    case pkPR, pkCF, pkCP, pkPY, pkSC:
        return true
    case pkAR:
        return node.TargetProperties.ModuleType == mtNone &&
            node.TargetProperties.ModuleLang == mlNone &&
            len(node.Outputs) == 1 &&
            !strings.HasSuffix(node.Outputs[0].rel(), ".a")
    default:
        return false
    }
}
```

All existing consensus / self-owned / node-claim / tag-inheritance logic stays
unchanged. The library archive guard rests on `module_type=lib` (set in
emit_ar.go) keeping `pkAR` library nodes out of the predicate.

## Files touched

- `attribute_generated.go` — predicate.
- `attribute_generated_test.go` — three fail-first graph-level tests + a library
  archive guard.

## Why this is safe

- A producer with no foreign consumer has no `generatedFirstClaim` entry (or it
  points at itself via the per-output consensus), so the override is a no-op:
  the predicate widening only re-attributes nodes that a different module already
  claimed by include resolution.
- Over-matching a non-header `pkAR` (e.g. ARCHIVE_ASM rodata) is harmless: those
  outputs are never `#include`d, so no foreign first-claim exists for them.
- Library archives (`module_type=lib`) are excluded by the predicate.

## Expected gate effect

- sg7 `PY`, `SC`, `AR` `target_properties` residuals (1 each) drop to 0.
- `AR inputs` stays 0 (no input filtering touched).
- sg2–sg6 byte-exact, sg5 under budget; T-109 library/LD pairs stay header-only.
