# T-85 plan: generated `.proto` PY3_PROTO bare-token resource routing (`banner_flags`)

## Upstream mechanism (verified)

`yatool/build/plugins/pybuild.py` packages every PROTO_LIBRARY's python output
by feeding the generated `_pb2.py` back through `onpy_srcs`:

```python
unit.on_generate_py_protos_internal(proto_paths)     # protoc -> _pb2.py
unit.onpy_srcs([pb2_arg(py_suf, path, mod, py_ver, unit) for ...])
```

`pb2_arg` builds the PY_SRCS token from `stripext(to_build_root(path, unit))`:

```python
def to_build_root(path, unit):
    if is_arc_src(path, unit):
        return '${ARCADIA_BUILD_ROOT}/' + rootrel_arc_src(path, unit)
    return path                       # <- generated proto: bare SRCS arg

def uniq_suffix(path, unit):
    if '/' not in path: return ''     # <- bare token has no '/', no .kqhj
    return '.{}'.format(pathid(unit.path())[:4])
```

A `RUN_PROGRAM(... STDOUT_NOAUTO banner_flags.proto ...)` proto is NOT
`is_arc_src` (it resolves to the build root), so `to_build_root` returns the
bare arg `banner_flags.proto`. Everything else follows from that ONE decision:

1. **py3cc token** — `on_py3_compile_bytecode([rootrel_arc_src(pb2_path)+'-', pb2_path, dst])`
   with `pb2_path = banner_flags__intpy3___pb2.py` (bare). First arg
   `banner_flags__intpy3___pb2.py-`, not the module-rel path.
2. **yapyc3 suffix** — `dst = pb2_path + uniq_suffix(pb2_path)`; bare token has
   no `/` so `uniq_suffix == ''`. Output `…__intpy3___pb2.py.yapyc3`, no `.kqhj`.
3. **resource backend** — `onresource_files` DEST source is the bare token
   `banner_flags__intpy3___pb2.py`. With no `${ARCADIA_BUILD_ROOT}` literal in the
   resource path, ymake embeds it via **objcopy** (`YMAKE_USE_OBJCOPY=yes`), not
   the rescompiler `_raw.auxcpp` path. A checked-in proto's token carries the
   `${ARCADIA_BUILD_ROOT}/<rootrel>` form → that literal forces the rescompiler
   `_raw.auxcpp` route (our `emitResourceObjcopy` `bad` list already encodes this
   rule; ordinary protos stay on auxcpp, confirmed: REF emits 1680 proto
   `_raw.auxcpp` nodes and reserves objcopy for the bare generated token).
4. **producer closure** — the PB (protoc) and py3cc nodes carry the generated
   `.proto` producer's full transitive `$(S)` closure (the `OUTPUT_INCLUDES`
   protos and their imports). This is `GeneratedFileInfo.ProducerSourceClosure`,
   already recorded by `emit_pr.go` (`setProducerSourceClosure`); our code reads
   only `SourceInputs` today and under-carries it.

Objcopy hash verified byte-exact against REF
`objcopy_a8b61b7e5cb098f9420299bf89.o`:
`objcopyHash([bareToken_pb2, bareToken_yapyc3], [b64(key_pb2), b64(key_yap)],
[kvHash_pb2(bare), kvHash_yap(bare)], "irt/bannerland/proto/banner_flags",
"PY3_PROTO")` — paths and kvHash inputs use the BARE token, tag is `PY3_PROTO`.

## Files to touch

- `emit_py_proto.go`
  - `emitPyProtoSrc`: detect generated proto (codegen registry hit, already
    present); thread the bare pybuild token + generated flag down; prefer
    `info.ProducerSourceClosure` over `info.SourceInputs` for PB/py3cc inputs.
  - `emitGeneratedPyProtoYapyc`: accept the bare pybuild token; emit bare first
    arg + unsuffixed yapyc3 output for generated, keep full-rel + path-id suffix
    for checked-in.
  - new `emitGeneratedPyProtoObjcopy`: package the generated `.py`/`.yapyc3` via
    an objcopy resfs node (reusing `objcopyHash`, `composeObjcopyArgBlocks`,
    `objcopyCmdArgs`, rescompiler/rescompressor refs) with tag `PY3_PROTO`; route
    generated-proto resources here, keep ordinary protos on
    `emitPyProtoAuxChunks`.
- `emit_py_proto_test.go` (new): fail-first coverage + checked-in guard.

The producer `cpp_proto` target_property tag on `banner_flags.proto` (REF tags
the producer PR node `module_tag=cpp_proto`) is a separate concern; fix locally
