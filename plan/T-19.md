# T-19 plan: RUN_PROGRAM output-set dedup for STDOUT == OUT_NOAUTO

## Representative divergence (reproduced)

`$(B)/kernel/lemmer/new_dict/rus/extra/rus.extra.dict.bin` (PR node) — the only
field that differs in the normalized pair is `outputs`:

```
ours: outputs = [ ".../rus.extra.dict.bin", ".../rus.extra.dict.bin" ]   (listed twice)
ref:  outputs = [ ".../rus.extra.dict.bin" ]                              (listed once)
```

The upstream ya.make declares the same physical file in two output roles:

```
RUN_PROGRAM(
    dict/tools/make_morphdict/main make-all --language rus --bastard-freq 0 --input lister.txt
    STDOUT rus.extra.dict.bin
    IN lister.txt
    OUT_NOAUTO rus.extra.dict.bin
)
```

The program's stdout *is* the declared output; `STDOUT` and `OUT_NOAUTO` name one
artifact. Everything else in the node (cmds, inputs, env, stdout redirect) is
already byte-identical to the reference. The self_uid differs only because of the
duplicate, which cascades a differing Merkle uid into the whole downstream
`RusExtraDict.rodata{,.asm,.o,.pic.o}` and `libnew_dict-rus-extra.a` chain — those
nodes' content already matches (empty `--pair` diffs); they are pure cascade.

In sg7, this is the ONLY PR node (of 958) whose outputs list contains a duplicate.

## Upstream mechanism

`RUN_PROGRAM` (`build/ymake.core.conf:4782`) spells outputs as:

```
${hide;output:OUT} ${hide;noauto;output:OUT_NOAUTO} ${stdout;output:STDOUT} ${stdout;noauto;output:STDOUT_NOAUTO}
```

Each modifier registers an output *file node*. ymake's output set is keyed by
file path: declaring one path through multiple output modifiers (here STDOUT +
OUT_NOAUTO) yields a single output entry — the `stdout` modifier sets the redirect
and `noauto` sets a property on the same file node, but the file is listed once.
Our `emitPR` instead appends STDOUT, then OUT, then OUT_NOAUTO without collapsing
equal paths, emitting the file twice.

Our codegen registration already models this: `emitRunProgram`'s
`registeredPROut` map registers each distinct output VFS exactly once (with the
comment that STDOUT and OUT_NOAUTO "denote one physical output"). The node's
`outputs` array in `emitPR` simply fails to apply the same set semantics.

## Change

`emit_pr.go` `emitPR`: build the `outputs` slice as a path-deduplicated set in
declaration order (STDOUT first, then OUT, then OUT_NOAUTO — the order already
correct for the 957 other PR nodes), dropping a second occurrence of a VFS already
emitted. Minimal, generic, no special-casing of the dict path.
