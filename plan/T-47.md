# T-47 plan: model CPP_EVLOG event2cpp induced-deps on generated PB outputs

## Upstream mechanism

`build/conf/proto.conf`:

```conf
macro CPP_EVLOG() {
    CPP_PROTO_PLUGIN0(event2cpp tools/event2cpp DEPS library/cpp/eventlog)
    ENABLE(_BUILD_PROTO_AS_EVLOG)
}
```

`CPP_PROTO_PLUGIN0(event2cpp tools/event2cpp ...)` registers `tools/event2cpp`
as one of the protoc plugins producing the module's `.pb.h`/`.pb.cc`. That tool
declares (`build/prebuilt/tools/event2cpp/ya.make.induced_deps`):

```conf
INDUCED_DEPS(h+cpp
    ${ARCADIA_ROOT}/contrib/libs/protobuf/src/google/protobuf/io/printer.h
    ${ARCADIA_ROOT}/contrib/libs/protobuf/src/google/protobuf/io/zero_copy_stream_impl_lite.h
    ${ARCADIA_ROOT}/contrib/libs/protobuf/src/google/protobuf/stubs/strutil.h
    ${ARCADIA_ROOT}/library/cpp/eventlog/event_field_output.h
    ${ARCADIA_ROOT}/library/cpp/eventlog/event_field_printer.h
    ${ARCADIA_ROOT}/library/cpp/eventlog/events_extension.h
    ${ARCADIA_ROOT}/util/generic/cast.h
    ${ARCADIA_ROOT}/util/stream/output.h
)
```

These (h+cpp) induced headers therefore attach to every PB output produced under
`CPP_EVLOG()`, and their transitive include closure (zero_copy_sink.h,
stubs/hash.h, abseil cleanup, base64.h, util/* …) follows from scanning them.
The true `.ev` path already wires this via `event2cppLDRef` in the registered
generated outputs' GeneratorRefs; ordinary `.proto` outputs under `CPP_EVLOG()`
do not, so their downstream CC compile inputs miss the family. This is the T-40
residual.

## Local mechanism

`scanner.go:resolveInducedDeps` reads each producing tool's `InducedDeps` bucket
(Header for `.pb.h`, Cpp for `.pb.cc`; `h+cpp` lands in both) from
`GeneratedFileInfo.GeneratorRefs` and resolves them as children of the generated
output. The defect: `emitProtoPB` builds `pbGenRefs` from
`protoc`/`cpp_styleguide`/`grpc`/extra-plugins but never `event2cppLDRef`,
because `case tokCppEvlog` in `modules.go` only models the `library/cpp/eventlog`
peer edge and leaves `_BUILD_PROTO_AS_EVLOG` output modeling out of scope.

## Change

1. `modules.go`: add `buildProtoAsEvlog bool` to `ModuleData`; set it in
   `case tokCppEvlog` (keep the existing eventlog peer addition).
2. `emit_proto.go`:
   - add `event2cppLDRef NodeRef` to `PbModuleEmission`;
   - in `newPBModuleEmission`, when `d.buildProtoAsEvlog`, resolve it via
     `ctx.tool(argToolsEvent2cpp)` (registers the tool in `moduleByRef` so the
     scanner can read its `InducedDeps`);
   - in `emitProtoPB`, append `pe.event2cppLDRef` to `pbGenRefs` (the refs used
     for `pbH`/`pbDepsH`/`pbCC`) when set.

No command-token work: scope is input closure only (ticket DESCR / T-40 done
state). grpc output registrations are left untouched (event2cpp does not feed
grpc outputs).

## Test-first

A `PROTO_LIBRARY()` + `CPP_EVLOG()` + `SRCS(foo.proto)` with a stub
`tools/event2cpp` declaring `INDUCED_DEPS(h+cpp ${ARCADIA_ROOT}/runtime/eventlog_runtime.h)`:
assert `$(B)/protos/foo.pb.cc.o` inputs contain `$(S)/runtime/eventlog_runtime.h`,
and an otherwise identical non-CPP_EVLOG `PROTO_LIBRARY()` does NOT.
