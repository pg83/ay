# T-10 — RUN_PROGRAM STDOUT outputs consumed by RESOURCE-style embedding

## Upstream mechanism

A `RUN_PROGRAM(... STDOUT/STDOUT_NOAUTO common.json)` declares `common.json`
as a **build output** with a producer node (the PR node). When that file is
later embedded with `RESOURCE` / `RESOURCE_FILES`, upstream ymake wires the
resource-objcopy node's input to the **build artifact** (`$(B)/.../common.json`)
and adds the dependency on the producing PR node — exactly as it does for any
generated file. The resource node never lists the `$(S)` source (which does not
exist for a generated file).

Our codegen registry already models this: `emit_pr.go` registers every PR
output (`OUT`, `OUT_NOAUTO`, **STDOUT/STDOUT_NOAUTO**) via
`registerBoundGeneratedParsedOutput` keyed by `copyFileOutputVFS`, recording the
producer ref. The consumer side (`emit_py_objcopy.go`) already looks the
RESOURCE *path* entry up in that registry (line ~271): when found it swaps the
input to `$(B)` and pins `info.ProducerRef`.

## Divergence (reproduced)

The real sg7 unit
`yabs/models_services/feature_store/mappers/common/catalog/ya.make` uses
`RESOURCE_FILES`, which `expandResourceFiles` turns into **two** entries per
file:

1. `ResourceEntry{Path: "common.json", Key: "resfs/file/..."}` — the payload
   input. The existing registry lookup resolves this to `$(B)` + producer dep.
2. `ResourceEntry{Path: "-", Key: "resfs/src/...=${rootrel;...input=TEXT:\"common.json\"}"}`
   — the `resfs/src` key. `emitResourceObjcopy`'s `e.Path == "-"` branch
   resolves this rootrel input **unconditionally** as
   `source(modulePath + "/" + inner)`.

For a build-generated `common.json` that produces a **phantom**
`$(S)/.../common.json` input on the objcopy node — a source file that does not
exist on disk and carries no producer edge. Reproduction
(`emit_py_objcopy_test.go`): a `PY3_LIBRARY` with
`RUN_PROGRAM(... STDOUT_NOAUTO common.json)` + `RESOURCE_FILES(common.json)`
emits an objcopy node whose `flatInputs()` contains **both**
`$(B)/db/common.json` and `$(S)/db/common.json`; the latter is the divergence.

`RESOURCE` (not `_FILES`) has no `resfs/src` `-` entry, so the simple
`RESOURCE` path was already correct — only the `RESOURCE_FILES` `resfs/src`
input regressed.

## Files touched

- `emit_py_objcopy.go` — route the `e.Path == "-"` rootrel input through the
  codegen registry, mirroring the existing path-entry lookup: when the inner
  path resolves to a registered generated output, use the `$(B)` output VFS and
  add the producer dep ref; otherwise keep the `$(S)` source (ordinary source
  JSON unchanged). Factor the shared lookup into one helper used by both the
  path entry and the kv entry (net DRY).
- `emit_py_objcopy_test.go` — regression: `RUN_PROGRAM STDOUT_NOAUTO` output
  consumed by `RESOURCE_FILES`; assert the objcopy node points at `$(B)`,
  carries the producer dependency, keeps the expected resource key, and has no
  phantom `$(S)` input. Use the yabs `common.json` shape generically (a small
  synthetic module, not a special-cased path).
