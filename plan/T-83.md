PROTO_LIBRARY compiles to `$(B)/quality/const_options.pb.h`; that generated header
uses the YaFF protoc plugin, so its include closure carries the
`library/cpp/yaff/*.h` runtime headers.

## Upstream mechanism

`sys_const.h` is generated from `sys_const.proto` and the custom generator emits
`#include "yabs/server/proto/quality/const_options.pb.h"`. Upstream's flat-input
model therefore lists `const_options.pb.h` + its source closure on every unit that
includes `sys_const.h` — confirmed: REF `sys_const.cpp.o` (1300 inputs) carries
`const_options.pb.h` + yaff, and REF `by_name.cpp` (which OUTPUT_INCLUDES
`sys_const.h`) does too. We cannot scan the custom header's generated body, so we
reconstruct it: a pkPR header reached through OUTPUT_INCLUDES re-exports the
**generated `.pb.h` sibling** of each `.proto` in its proto-import closure.

The `protobuf_old/descriptor.pb.h` false add is the existing checked-in WKT
`.pb.h` synthesis (emit_pr.go ~595) firing on `protobuf_old/descriptor.proto`.
`#include "google/protobuf/descriptor.proto"` resolves to 3 variant `.proto`
(protobuf, protobuf_old, protobuf_std — all present in REF too); only the
canonical `protobuf/descriptor.pb.h` is genuinely `#include`d (it is already in
`out` via the IN-walk). Synthesising a same-named sibling for a non-canonical
variant is the bug.

## Change (emit_pr.go `prInputClosure`, OUTPUT_INCLUDES block only)

1. **Generated `.pb.h` sibling for a pkPR custom-header target.** When the
   OUTPUT_INCLUDES target resolves to a registered codegen output whose producer
   is `pkPR` (`info.ProducerKvP == pkPR`), for each kept `$(S)/x/y.proto` derive
   `$(B)/x/y.pb.h`; if `reg.lookup` finds it registered, `walkClosure` it and add
   the sibling `.pb.h` itself plus its `$(S)` source entries (dropping the
   intermediate `$(B)` yaff/deps headers). `resolveCodegenDepRefs` then adds the
   `.pb.h` producer dep (deps 4→5). Gated to `!fullSourceClosure && !hasProtoIN`
   (T-72 no-IN rule and T-48 proto-IN guard untouched). NOT applied to a pkPB
   target (control_board's `tablet.pb.h` lists only `.proto` sources — no deep
   generated `.pb.h`).

2. **Basename guard on the checked-in WKT synthesis.** Skip `source(sibling)`
   when a `.pb.h` of the same basename is already present in `out` (the canonical
   variant is already resolved). Removes `protobuf_old/descriptor.pb.h` while
   keeping genuine single-variant WKTs (duration/empty/struct/timestamp — unique
   basenames, not otherwise present). Can only *remove* same-basename duplicate
   synths, never add, so control_board / sg5 stay byte-exact.
