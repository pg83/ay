# T-127 plan: PY23 COPY_FILE module_tag propagation

## Upstream mechanism

In upstream ymake every node a module produces inherits the module's
`MODULE_TAG` through `Node2Module` attribution. For a `PY23_LIBRARY` the
PY3 submodule's `MODULE_TAG` is `PY3` (set by `_ARCADIA_PYTHON3_ADDINCL`,
`build/conf/python.conf:1003`; multimodule submodule default tag in
`devtools/ymake/lang/confreader.cpp:847`). A `COPY_FILE` `.py` copy is a
node owned by that submodule, so its `target_properties.module_tag` is
`py3`. ay already reproduces this per-emitter: CC/AR/FL/PR nodes stamp
`moduleCCTag(d.moduleStmt.Name)` (`PY23_LIBRARY -> py3`). The CP emitter
is the only owned-node emitter that drops the tag.

## Divergence

The four reference `CP` pairs under
`$(B)/yabs/server/cs/pylibs/servant_app/` and `.../settings/` carry
`module_tag=py3`; ay's CP nodes carry only `module_dir`. T-119 classified
these as pure missing-tag residuals (commands/inputs/outputs/module_dir
already match).

## Files to touch

- `emit_cp.go`: `emitCPWithDeps` gains a `moduleTag STR` parameter; when
  non-zero it stamps `TargetProperties.ModuleTag`. `emitCP` passes `0`
  (LD plugin `.pyplugin` copies stay untagged — no live residual).
- `emit_copy.go`: `emitCopyFiles` passes the module's already-computed
  tag (`moduleInputs.ModuleTag`, == `moduleCCTag(d.moduleStmt.Name)`) to
  `emitCPWithDeps`. No recomputation — reuse the value gen.go already put
  on `ModuleCCInputs`.
- `emit_copy_test.go`: failing-first coverage.

`moduleInputs.ModuleTag` is `0` for ordinary `LIBRARY`, so plain copy
nodes gain no tag — the required guard holds for free.

## Tests

- `TestGen_Py23LibraryCopyFileCarriesPy3Tag`: a `PY23_LIBRARY` with
  `COPY_FILE(src.py copied.py)` — assert the CP node has `module_tag=py3`.
- Guard: a plain `LIBRARY` `COPY_FILE` CP node has empty `module_tag`.

- `CP target_properties` residual count drops 4 -> 0.
