# T-15 — Honor module-level BISON_FLAGS in the bison producer command

## Symptom (sg7, yt query base)

`$(B)/yt/yt/library/query/base/parser.h` and `parser.ypp.cpp` pair as
command-only/content drift: the reference bison invocation carries
`-Wcounterexamples` (the module's `BISON_FLAGS(-Wcounterexamples)`), ours does
not, so the yt query base bison subtree fails to collapse. T-3 fixed the output
shape and include propagation; the only residual difference on this pair is the
missing flag.

## Upstream mechanism (yatool/build/conf/bison_lex.conf)

- Line 10: `BISON_FLAGS=-v` — the default value of the `BISON_FLAGS` variable.
- Lines 57-59: `macro BISON_FLAGS(Flags...) { SET_APPEND(BISON_FLAGS $Flags) }`
  — the macro **appends** its args to that variable (repeated flags accumulate,
  declaration order preserved).
- Lines 140 / 147 (`_SRC("y")` / `_SRC("ypp")`, identical C++ bison family):
  `.CMD=${tool:"contrib/tools/bison"} $BISON_FLAGS ${env:_SRC_y_env} $_BISON_HEADER -o ${output} ${input:SRC} ${SRCFLAGS} …`

So `$BISON_FLAGS` expands inline right after the bison tool and before
`--defines` (`$_BISON_HEADER`). Default `-v`; with
`BISON_FLAGS(-Wcounterexamples)` it becomes `-v -Wcounterexamples`. Emitted
command: `bison -v -Wcounterexamples --defines=… -o <out> <src>`.

Our emitter hardcodes `-v` (the variable default) but drops the appended macro
flags entirely — `BISON_FLAGS` is currently in `acknowledgedMacros` (no-op).

## Changes

1. **modules.go** — add `bisonFlags []ARG` to `ModuleData`; add
   `case tokBisonFlags:` to `applyUnknownStmt` appending each expanded arg
   (`internArg`) — append semantics = upstream `SET_APPEND`, repeated flags and
   order preserved.
2. **gen.go** — drop `"BISON_FLAGS"` from `acknowledgedMacros` (now typed);
   populate `BisonFlags: d.bisonFlags` into `moduleInputs`.
3. **emit_cc.go** — add `BisonFlags []ARG` to `ModuleCCInputs`.
4. **emit_bison_y.go** `emitBisonProducer` — build the bison command head as
   `[bisonBin, -v] + in.BisonFlags + [--defines=…, -o, out, src]`, inserting the
   module flags between `-v` and `--defines`, matching upstream order.

## Test (test-first)

`emit_bison_y_test.go`: a `.ypp` module with `BISON_FLAGS(-Wcounterexamples)`;
assert the YC node's first cmd contains `-Wcounterexamples` positioned after
`-v` and before `--defines=`. Fails on current code (flag dropped).

## Out of scope

flex/ragel/protobuf, generated output naming, output placement / sibling
include registration (T-3).

## Expected gate effect

sg7: the parser.h/parser.ypp.cpp pair loses the `-Wcounterexamples` command
difference; yt query base bison subtree gains parity (or retains only
non-BISON_FLAGS diffs). sg2–sg6 carry no BISON_FLAGS → byte-exact unchanged.
No hot-path change → no perf regression.
