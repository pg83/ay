# T-15 — Internal sysincl files must be config-gated, not loaded wholesale

## Hypothesis (reproduced)

`./dev/gen_sg7.sh`'s graph carries Zephyr `smart_devices` headers
(`stdint.h`, `time.h`, `pthread.h`, `syscall.h`, …) on ordinary C++ compile
closures such as `$(B)/adfox/amacs/cache/borc/borc.cpp.o`. The upstream
reference (`/home/pg/monorepo/4/sg7.json`) contains **zero** `sdk/zephyr` or
`monocle_common/firmware` paths.

These mappings come from `build/internal/sysincl/actions_zephyr.yml`, whose
second record has a broad `source_filter: ^(alice|util|contrib|yandex_io|smart_devices|library)`
and remaps common libc headers (`stdint.h`, `time.h`, `pthread.h`, `syscall.h`)
to Zephyr headers. adfox's closure pulls in `util/…`, `contrib/…`, `library/…`
headers that `#include <stdint.h>` etc.; those includer paths legitimately match
`^(util|contrib|library|…)`, so the remap fires — **the source_filter matching
is correct**.

The real divergence is *which files are loaded*. Upstream's internal sysincl set
is a **curated, config-gated list** (`build/internal/conf/sysincl.conf` +
`build/internal/conf/project_specific/{smart_devices,maps/mapkit}.conf`).
`actions_zephyr.yml` is loaded only:

    when ($OS_ZEPHYR == "yes" && $ARCH_ARM_ATS3089P == "yes")   # smart_devices.conf:22

sg7 is a native linux/x86_64 build, so upstream never loads it. Our
`loadSysInclDir` loads **every** `*.yml` in `build/internal/sysincl/`
unconditionally — the comment claims absent-platform files are "inert", which is
false for `actions_zephyr.yml` (it maps `stdint.h`/`time.h`/… which appear
everywhere).

## Upstream mechanism to reproduce

The internal sysincl `SYSINCL+=…` appends are gated by `when` clauses on
`OPENSOURCE` / `OS_*` / `ARCH_*` / project flags. Restricted to the OSes this
model builds (`linux`/`darwin`/`windows` — there is no `OS_NONE`/`OS_FREERTOS`/
`OS_ZEPHYR`/`OS_ANDROID`), the curated internal set is:

| file | gate |
|------|------|
| macro, misc, sdc, smart_devices, speechkit, weird | `OPENSOURCE != yes` (the internal contour, already implied) |
| taxi | `!USE_STL_SYSTEM && !NORUNTIME && !OPENSOURCE` (unset → load) |
| misc-win | `OS_WINDOWS` |
| smart_devices_linux | `OS_LINUX` |
| smart_devices_darwin | `OS_DARWIN` |
| actions_zephyr, amlogic_rtos, smart_devices_{android,beken,esp32,telink}, sanitizers-to-nothing, mobile-metrica, qt, yx-account-manager | `OS_ZEPHYR/OS_NONE/OS_FREERTOS/OS_ANDROID/MAPSMOBI_BUILD_TARGET` — never built here |

## Test (fails first)

`TestSysInclInternalGating`: a MemFS internal sysincl dir with
`actions_zephyr.yml` (gated out on linux) and `smart_devices_linux.yml` (gated
in), each a `source_filter: ^util` record. After `loadSysInclSetForFS(os=linux,
opensource=false)`:

- `util/foo.cpp` header from `actions_zephyr.yml` → **absent** (file not loaded) — the fix.
- `util/foo.cpp` header from `smart_devices_linux.yml` → **present** (file loaded, filter matches).
- `adfox/bar.cpp` header from `smart_devices_linux.yml` → **absent** (filter `^util` does not match) — source_filter scope still works.

Fails on current code because `loadSysInclDir` loads `actions_zephyr.yml`, so its
header is present for `util/foo.cpp`.
