# T-35 plan: cpp_fbs module_tag on FlatBuffers producer/consumer nodes

## Upstream mechanism

`build/conf/fbs.conf` defines `multimodule FBS_LIBRARY` whose C++ variant is
`module CPP_FBS: LIBRARY { ENABLE(CPP_FBS) ENABLE(GEN_FBS) SET(PEERDIR_TAGS CPP_FBS) }`.
For a multimodule submodule ymake derives the node `module_tag` from the
submodule name lowercased — exactly as `CPP_PROTO` → `cpp_proto` (which our code
already reproduces via `tagCppProto`). `CPP_FBS` → `cpp_fbs`. Neither CPP_PROTO
nor CPP_FBS sets `MODULE_TAG` explicitly; the tag is the submodule identity.

The tag is a module-level property: every node owned by the CPP_FBS module
carries it — the flatc producer (`FL` `.fbs.h/.cpp/.bfbs`), the generated
`.fbs.cpp.o` compile, and the module archive.

## Reference evidence (sg7.ref)

- `$(B)/yabs/bscount/protocol/fbs/deploy.fbs.h` (FL producer):
  ref `{module_dir, module_tag:cpp_fbs}` vs ours `{module_dir}`.
- `$(B)/yabs/bscount/protocol/fbs/deploy.fbs.cpp.o` (CC consumer):
  ref `{module_dir, module_lang:cpp, module_tag:cpp_fbs, module_type:lib}`
  vs ours missing `module_tag`.
- `$(B)/yabs/bscount/protocol/fbs/libbscount-protocol-fbs.a` (archive):
  ref `{...,module_tag:cpp_fbs,...}` vs ours missing `module_tag` — same
  generic mechanism, tagged by the same module-level property (not separate
  drift).

## Files to touch

- `str_consts.go`: add `tagCppFbs = internStr("cpp_fbs")`.
- `gen.go`: extract the per-module CC tag switch (currently inline at the
  `perModuleCCTag` computation) into a single helper `moduleCCTag(name TOK) STR`
  and add `case tokFbsLibrary: tagCppFbs`. `perModuleCCTag` already flows into
  `ModuleCCInputs.ModuleTag` (→ `.fbs.cpp.o` consumer via emit_cc) and into the
  module archive (`emitARNamedTagged`).
- `emit_flatc.go`: thread the module's tag into `emitFL` so the FL producer's
  `TargetProperties.ModuleTag` is set. `emitFlatcProducer` computes it from
  `moduleCCTag(d.moduleStmt.Name)` — keeping producer/consumer/archive on one
  source of truth. A plain `LIBRARY()` with `.fbs` (test-only shape) yields tag
  `0` → untagged, so unrelated nodes never gain the tag.
