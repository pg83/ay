# T-114: EV `--cpp_out=proto_h=true` parity

## Upstream mechanism

`CPP_EV_CMDLINE` and `CPP_PROTO_CMDLINE` both expand the SAME
`_CPP_PROTO_CMDLINE_BASE` (ymake.core.conf:612,613,616). That base renders the
cpp_out arg as:

    --cpp_out=${CPP_PROTO_PLUGINS}:$ARCADIA_BUILD_ROOT/$PROTO_NAMESPACE

`CPP_PROTO_PLUGINS` is empty by default (`PROTOC_TRANSITIVE_HEADERS=yes`,
proto.conf:26). When a module does `SET(PROTOC_TRANSITIVE_HEADERS "no")`,
ymake.core.conf:656-657 sets `CPP_PROTO_PLUGINS=proto_h=true${CPP_PROTO_PLUGINS}`.
Because the EV (and `.cfgproto`, proto.conf:496 = `$CPP_EV_CMDLINE ...`) command
shares that same base variable, the EV/cfgproto `--cpp_out` carries the
`proto_h=true` token EXACTLY when the PB `--cpp_out` does.

Note: `proto_h=true` adds `.deps.pb.h` to `CPP_PROTO_OUTS` (ymake.core.conf:658)
but NOT to `CPP_EV_OUTS` (line 617). So the EV outputs are unchanged — only the
cpp_out token differs. This ticket therefore only touches the command flag.

## Defect

Our EV path (`emit_ev.go`) hardcodes the cpp_out token via the const
`evProtocConstTail`, which always uses `argCppOutB` = `--cpp_out=:$(B)/`. It
never reflects `PROTOC_TRANSITIVE_HEADERS == "no"`, unlike the PB path
(`emit_pb.go:366` / `emit_proto.go:275` `liteHeaders: !protoTransitiveHeadersEnabled(d)`).

Representative: `$(B)/yabs/server/proto/event_log/events.ev.pb.cc` — that module
does `SET(PROTOC_TRANSITIVE_HEADERS "no")`, so upstream emits
`--cpp_out=proto_h=true:$(B)/` while we emit `--cpp_out=:$(B)/`.

## Change

1. `arg_consts.go`: add `argCppOutProtoHB = internArg("--cpp_out=proto_h=true:$(B)/")`.
2. `emit_ev.go`: add `evProtocConstTailLite` (= `evProtocConstTail` with the
   lite cpp_out token). Thread a `liteHeaders bool` param through
   `emitProtoWrapperPBNode` and `emitEV`; select the tail by it.
3. Call sites compute `!protoTransitiveHeadersEnabled(d)`:
   - `emit_ev.go` `emitLibraryEvSource`
   - `emit_proto.go` (the evSrcs loop)
   - `emit_cfgproto.go` `emitLibraryCfgProtoSource`

## Tests (fail-first)

- `emit_ev_test.go`: new EV fixture with `SET(PROTOC_TRANSITIVE_HEADERS "no")`
  asserting the EV cmd carries `--cpp_out=proto_h=true:$(B)/` and NOT the bare
  `--cpp_out=:$(B)/`. Fails on current code.
- Control: existing `TestGen_ProtoLibrary_TransitivePROTONamespaceReachesEVCmd`
  (no SET) keeps `--cpp_out=:$(B)/`; add an explicit assertion it does not gain
  `proto_h=true`.

## Gate effect

clears; sg7 PB cmd total drops. sg2–sg6 byte-exact; no output/input change.
