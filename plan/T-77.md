# T-77 plan: R6 ragel6 output naming (defext/noext), not a flag default

## Hypothesis corrected by empirical dump

T-74 candidate 3 framed this as a `-CG2` vs `-CT0` default-flag mismatch on the
representative `$(B)/library/cpp/json/fast_sax/parser.rl6.cpp`. Dumping the
current artifacts disproves that framing:

- `library/cpp/json/fast_sax/parser.rl6.cpp` is emitted on BOTH sides as a pair:
  a tool/host node (`-CG2`, release host) and a target node (`-CT0`, debug
  target). The two normalized R6 lines are **byte-identical** between our output
  and the reference. The `dump diff --pair` "`-ours +-CG2 / -ref +-CT0`" line is
  a false positive of the diff tool's same-output node-pairing heuristic
  (it cross-pairs our CG2 with ref CT0), not a real divergence. Our
  `Ragel6Optimized = buildRelease && !buildSanitized` already matches upstream
  `Ragel(set_default_flags(optimized=build.is_release and not build.is_sanitized))`
  (ymake_conf.py:2314-2328): host is release→CG2, sg7 target is debug→CT0.

The single genuine R6 divergence in sg7 (the only R6 lines that differ between
our and ref normalized graphs) is the `library/cpp/config/markupfsm.h.rl6`
source:

```
ours: $(B)/library/cpp/config/markupfsm.h.rl6.cpp        (+ .cpp.o, .cpp.pic.o)
ref:  $(B)/library/cpp/config/markupfsm.h                (header only, no compile)
```

`markupfsm.h.rl6` is a ragel source whose generated artifact is a HEADER
(`markupfsm.h`), `#include`d by `markup.cpp` — upstream does not compile it.
Our `ragel6OutVFS` unconditionally appends `.cpp`, producing
`markupfsm.h.rl6.cpp`, and `emitLibraryRagel6Source` then unconditionally
compiles it, adding a spurious `.cpp.o`/`.cpp.pic.o` pair and naming the header
wrong (so `markup.cpp`'s `#include "markupfsm.h"` cannot resolve to it).

## Upstream mechanism

`_SRC("rl6", ...)` (ymake.core.conf:3303) writes the output via
`${output;defext=.rl6.cpp;nopath;noext:SRC}`. The modifier order in ymake
(macro_processor.cpp:1758-1792) is:

1. `noext` (EMF_CutExt): strip the LAST extension (`rfind('.')`).
2. `nopath` (EMF_CutPath): reduce to basename.
3. `defext=.rl6.cpp` (EMF_HasDefaultExt): append `.rl6.cpp` **only if the
   basename still has no extension** (`addSuffix=false` when a `.` remains after
   the last slash).

Worked examples:

- `parser.rl6`  → noext `parser`        → no ext → `parser.rl6.cpp`   (compiled)
- `datetime/parser.rl6` → noext `datetime/parser` → nopath `parser` → `parser.rl6.cpp`
- `markupfsm.h.rl6` → noext `markupfsm.h` → has ext `.h` → `markupfsm.h` (header, NOT compiled)

Whether the generated artifact is compiled is then driven purely by its
extension: a `.cpp` output is a buildable C++ source; a `.h` output is a header.

## Files touched

- `emit_r6.go`
  - `ragel6OutVFS`: replace the unconditional `+ ".cpp"` with the noext/defext
    rule applied to the basename (directory / `_/` infix handling unchanged).
  - `emitLibraryRagel6Source`: when the computed output is a C++ source, keep the
    current compile path verbatim; when it is a header, register the generated
    header + emit only the R6 node, return nil (the `.h.in` header-only pattern).
- `emit_r6_test.go`: fail-first coverage (below).
