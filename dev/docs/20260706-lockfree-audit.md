# Аудит lock-free свойств конвейера ген→executor (2026-07-06)

Статус: **закрыто; -race make -j прогон на реальной цели чистый.**

## Модель

PageVec — не lock-free структура общего вида, а SWSR-publish: один писатель
(ген-поток), читатели-воркеры видят только id, вручённые им через
happens-before ребро `go ex.fire(f)` (executor.go). Элементы пишутся ровно один
раз до hand-off; in-place обновлений нет. Кросс-воркерная синхронизация —
`NodeFuture.once` (uid зависимости читается после `once.Do` — упорядочено).

Элементы крупнее слова (internCell = 24 байта) безопасны не атомарностью
чтения, а отсутствием конкуренции на элементе. Это дисциплина, структурно не
принуждается: код воркера обязан ходить только через `shared*`-акцессоры
(cells/PageVec), никогда — во flat-массивы интерна (реаллоцируются под ногами).

## Найденные и закрытые дыры

1. **Торн-заголовок страницы** (`pages [32][]T`): make() писал 24-байтовый
   слайс-хедер неатомарно, читатель (`getSafe` незаписанных id — путь
   fs.contentHashes из воркеров) перечитывал его дважды → возможен nil-deref /
   bounds-panic. Закрыто атомарной публикацией: `[32]atomic.Pointer[[]T]`,
   хедер строится целиком (включая элемент-триггер) и публикуется одним словом
   (3401324).
2. **finish() без топопорядка**: pending-ноды доставлялись в порядке эмиссии,
   зависимый мог уйти воркеру раньше своей pending-зависимости (цепочки
   reserved-узлов PR→PR) → чтение незаписанного futs-слота. Закрыто волновой
   доставкой + throw на цикле (631da86). Репро:
   TestFinish_PendingDeliveredAfterTheirDeps.
3. **ex.fetchRefs** — найдена реальным -race прогоном (не аналитикой!):
   onNode переприсваивал поле на каждой ноде, воркеры конкурентно читали.
   Значение никогда не менялось, но это формальная гонка и первый репорт TSan.
   Закрыто write-once до первой горутины + throw на смену (4a2f582).

## Рецепт race-прогона

Нужна машина с cgo (libc-хедеры): `CGO_ENABLED=1 go build -race -o ay-race .`,
затем реальное исполнение с холодным кэшем: `ay-race make -j$(nproc) <target>`
(не -G и не -j0 — конкурентность живёт только в паре ген↔воркеры). Один чистый
прогон ≠ доказательство: тайминг-зависимо, гонять несколько раз. go test -race
почти бесполезен — тесты однопоточны.
