# Upstream over-emit: resource-objcopy node carries the producer's full source closure

## Summary

Upstream ymake attaches the **full transitive `$(SOURCE_ROOT)` compile closure of
the embedded resource's producer** to a resource-embedding `objcopy` node, even
though the `objcopy.py` action reads only the single resource file it embeds. The
extra inputs (libcxx headers, sanitizer headers, `cpp_proto_wrapper.py`, …) are
not referenced anywhere in the node's `cmd_args` — they are pure cache-key
dependencies, not action inputs. This is an over-emit (excess closure) specific
to this node kind.

We reproduce it for byte-exact parity (removing it diverges 2 nodes / sg5 by
~2978 inputs), via `prOutputInputs` → `prResourceExtraInputs`. This document
records that the source of the behavior is upstream, not our model.

## Affected node kind

Resource-embedding objcopy: `cmd_args` runs `build/scripts/objcopy.py
--compiler … --objcopy … --compressor … --rescompiler … --output_obj <out.o>
--inputs <one resource> --keys <b64>`. The node's `outputs` is a single
`objcopy_<hash>.o`.

In sg5 (`ydb/apps/ydbd`) two such nodes carry the over-emit:

- `$(B)/yt/yql/providers/yt/provider/objcopy_da3053cc1d35b909edac832dbd.o`
  (1356 inputs; the command embeds only `yql_yt_op_settings.json`)
- `$(B)/yt/yql/providers/yt/codec/codegen/llvm16/objcopy_eb223d69dbeb4201937297631d.o`

## The over-emit, precisely

For the provider node below: `cmd_args` names exactly one payload input —
`$(BUILD_ROOT)/yt/yql/providers/yt/provider/yql_yt_op_settings.json` — plus the
tools (`objcopy.py`, clang++, llvm-objcopy, rescompressor, rescompiler). Yet
`inputs` lists 1356 paths. None of the ~1350 `$(SOURCE_ROOT)/contrib/libs/...`
(libcxx, clang20-rt sanitizer) or `build/scripts/cpp_proto_wrapper.py` entries
appear in `cmd_args`. They are the transitive source closure of the node that
*produced* `yql_yt_op_settings.json` (a proto/codegen RUN step), folded onto the
objcopy node by upstream's flat input model.

## How to generate the upstream graph

The reference graph is the upstream `ya`/`ymake` `-G` dump for `ydb/apps/ydbd`,
native x86_64, `OS_SDK=local`, sandboxing — captured at
`/home/pg/monorepo/ydb/sg5.json`. Upstream invocation (run inside the `ydb`
source root):

```sh
ya make -G --sandboxing -DOS_SDK=local ydb/apps/ydbd > sg5.json
```

Our byte-exact mirror (this repo, `dev/gen_sg5.sh`):

```sh
env -u CFLAGS -u CXXFLAGS \
  ./ay make -j 0 -G --sandboxing \
    --source-root /home/pg/monorepo/ydb \
    -DOS_SDK=local --host-platform-flag OS_SDK=local \
    ydb/apps/ydbd > sg5.json
```

To pull the node below out of the raw upstream graph:

```sh
ay dump grep --in /home/pg/monorepo/ydb/sg5.json --raw \
  --substr 'objcopy_da3053cc1d35b909edac832dbd'
# returns two nodes: the AR (global.a, link_lib.py) and the objcopy node below
# (the one whose cmd_args use build/scripts/objcopy.py).
```
